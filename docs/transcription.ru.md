# Транскрипция изображений

Позвольте vision-модели описывать изображения, чтобы текстовые модели могли их понимать.

## Что это делает

Когда вы отправляете изображение текстовой модели, Agora может использовать отдельную vision-модель для предварительной генерации текстового описания изображения. Это описание затем включается в промпт, отправляемый вашей основной модели.

Это позволяет использовать изображения с любой моделью, даже с теми, которые нативно не поддерживают зрение.

## Настройка

1. Перейдите в **Настройки → Транскрипция изображений**
2. Выберите **Модель транскрипции** — это должна быть модель с поддержкой зрения (например, GPT-4o, Gemini Flash, Qwen-VL)
3. Добавьте модели в **Разрешённые модели** — это текстовые модели, которые будут получать описания изображений
4. Настройте **Размер пакета**, если вы отправляете много изображений сразу (сколько изображений описывать за один API-вызов)

!!! tip "Локальные vision-модели"
    Вы можете использовать локальную vision-модель (с mmproj) в качестве модели транскрипции. Это сохраняет обработку изображений на устройстве.

## Как это работает

1. Вы прикрепляете изображение к вашему сообщению
2. Agora определяет, что ваша текущая модель не поддерживает зрение
3. Изображение сначала отправляется модели транскрипции
4. Модель транскрипции генерирует текстовое описание
5. Это описание добавляется перед текстом вашего сообщения
6. Объединённый текст отправляется вашей основной модели

---

## Размер пакета

Управляет тем, сколько изображений описывается за один API-вызов модели транскрипции.

- **1** — Описывать по одному изображению за раз (больше вызовов API, точнее)
- **5–10** — Описывать несколько изображений за вызов (меньше вызовов API, может потерять детали)

По умолчанию зависит от устройства. Меньшие значения дают лучшие результаты, но стоят дороже.

## Выбор модели

### Модель транскрипции

Это vision-модель, которая генерирует описания изображений. Выберите самую способную vision-модель, доступную вам.

### Разрешённые модели

Это текстовые модели, которые будут использовать транскрипцию изображений. Только модели в этом списке будут получать транскрибированные описания изображений. Другие модели будут получать изображения напрямую (если они их поддерживают) или не получать вовсе.
