# Параметры генерации

Управляйте тем, как модели генерируют ответы — от длины контекста до настроек креативности.

## Контекстное окно

**Макс. сообщений контекста** устанавливает, сколько последних сообщений отправляется модели в качестве контекста. По умолчанию: **20**.

- **5–20** — Более короткий контекст, более быстрые ответы, меньшее использование токенов
- **20–50** — Более длинный контекст для сложных, многоходовых диалогов
- **50–100** — Максимальный контекст для очень длинных обсуждений (может достигнуть лимитов токенов)

Это применяется ко всем моделям. Фактический размер контекстного окна в токенах зависит от вашей модели и длины сообщений.

---

## Temperature

Управляет случайностью в выводе модели. Диапазон: **0.0 – 2.0**.

- **0.0 – 0.3** — Более детерминированный, последовательный, фактический
- **0.5 – 0.8** — Сбалансированная креативность (рекомендуется по умолчанию)
- **1.0 – 2.0** — Более случайный, креативный, непредсказуемый

Более высокая temperature означает, что модель с большей вероятностью выберет менее вероятные слова. Более низкая температура даёт более сфокусированные, повторяющиеся выводы.

!!! tip "Когда настраивать"
    - **Код / Факты**: Используйте низкую температуру (0.0 – 0.3)
    - **Творческое письмо**: Используйте высокую температуру (0.8 – 1.2)
    - **Общий чат**: Используйте среднюю температуру (0.5 – 0.7)

---

## Top P (Nucleus Sampling)

Управляет разнообразием выбора токенов. Диапазон: **0.0 – 1.0**.

Модель рассматривает только наименьший набор токенов, чья совокупная вероятность превышает `top_p`.

- **0.1** — Очень сфокусированный, только наиболее вероятные токены
- **0.5** — Умеренное разнообразие
- **0.9 – 1.0** — Полное разнообразие (рекомендуется по умолчанию)

Обычно вы настраиваете *либо* temperature, *либо* top P — не оба сразу.

---

## Максимум токенов по умолчанию

Устанавливает максимальный лимит токенов для ответов модели. При установке модель не сгенерирует больше этого количества токенов в одном ответе. Когда **не установлено** (по умолчанию), применяется собственный максимум модели.

Доступные предустановки:

```
256   512   1024   2048
4096  8192  16384  32768
```

!!! tip "Оставьте неустановленным для гибкости"
    Для большинства случаев использования оставьте это неустановленным. Устанавливайте лимит только когда вам нужна постоянная длина ответов (например, короткие резюме) или хотите ограничить затраты.

---

## Частотный штраф (Frequency Penalty)

Снижает склонность модели повторять одни и те же слова. Диапазон: **-2.0 – 2.0**.

- **Положительные значения** (0.1 – 1.0) — Препятствуют повторению
- **Ноль** (0.0) — Без штрафа (по умолчанию)
- **Отрицательные значения** (-1.0 – -0.1) — Поощряют повторение

---

## Штраф за присутствие (Presence Penalty)

Поощряет модель говорить на новые темы. Диапазон: **-2.0 – 2.0**.

- **Положительные значения** (0.1 – 1.0) — Поощряют разнообразие тем
- **Ноль** (0.0) — Без штрафа (по умолчанию)
- **Отрицательные значения** — Оставаться на текущей теме

---

## Мышление / Рассуждение (Thinking)

Включает цепочку рассуждений (chain-of-thought) для поддерживаемых моделей (например, DeepSeek R1, Qwen3, Claude).

При включении модель генерирует внутренние рассуждения перед созданием окончательного ответа. Это повышает точность для сложных задач, но занимает больше времени и использует больше токенов.

### Уровень мышления

- **Низкий** — Минимальные рассуждения, быстрее
- **Средний** — Сбалансированный (по умолчанию)
- **Высокий** — Максимальные рассуждения для сложных задач

!!! warning "Не все модели поддерживают мышление"
    Режим мышления требует модель, поддерживающую токены рассуждения. Если ваша модель не поддерживает это, настройка не имеет эффекта.

---

## Визуализация вытеснения контекста

При включении Agora визуально показывает, какие сообщения включены в текущее контекстное окно, а какие были вытеснены (исключены из-за ограничения контекстного окна). Это помогает понять:

- Какую часть вашего диалога модель может «видеть»
- Когда старые сообщения выпадают из контекста
- Нужно ли увеличить контекстное окно

Визуализация отображается как едва заметный маркер в представлении диалога.

---

## Как работают параметры

Все параметры генерации являются **nullable** — когда они явно не установлены, они не отправляются модели, и модель использует свои собственные значения по умолчанию. Каждый параметр имеет опцию сброса для очистки значения обратно в «не установлено».

---

## Переопределения для конкретного диалога

Вы можете переопределить параметры генерации для отдельных диалогов, используя диалог **Расширенные настройки** на экране чата (долгое нажатие кнопки отправки или использование меню ⋮).
