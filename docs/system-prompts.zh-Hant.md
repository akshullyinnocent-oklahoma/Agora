# 系統提示詞

系統提示詞定義模型的人格、行為和基本規則。Agora 讓您精細控制指令如何組裝並傳送給模型。

## 三段式編輯器

每個系統提示詞範本有三個可獨立編輯的區段：

```text
┌─────────────────────────────────┐
│ 系統提示詞                      │ ← 核心指令（人格、規則、語調）
├─────────────────────────────────┤
│ 使用者前綴                      │ ← 在每則使用者訊息之前添加
├─────────────────────────────────┤
│ 使用者後綴                      │ ← 在每則使用者訊息之後添加
└─────────────────────────────────┘
```

### 系統提示詞

主要的指令區塊。您在此定義：

- **人格**：「您是一位專注於簡潔架構的資深 Python 開發者。」
- **規則**：「始終使用中文回覆。使用項目符號列出清單。」
- **限制**：「永遠不要道歉。保持簡潔。優先使用程式碼而非解釋。」

### 使用者前綴與後綴

這些會包裹您傳送的每則訊息：

- **使用者前綴** — 新增在您的訊息文字之前。適用於提醒或上下文標籤。
- **使用者後綴** — 新增在您的訊息文字之後。適用於結束指示。

**範例**：如果您的前綴是 `[上下文：正在處理 Agora 文件]`，後綴是 `\n\n請使用 Markdown 回覆。`，模型會收到：

```text
[上下文：正在處理 Agora 文件]
如何設定網頁搜尋？
請使用 Markdown 回覆。
```

## 建立提示詞

1. 前往 **設定 → 系統提示詞**
2. 點選 **新增提示詞**
3. 輸入 **標題**（例如「翻譯器」、「程式碼審查者」、「中文助手」）
4. 填寫三個區段：
    - 點選 **新增文字** 撰寫靜態內容
    - 點選 **新增變數** 插入動態值
5. 點選 **儲存**

### 重新排序項目

在每個區段內，您可以有多個文字區塊和變數。長按項目可：

- **上移** / **下移** — 在區段內重新排序
- **移除** — 刪除該項目

## 變數替換

變數會在傳送訊息時替換為動態值：

| 變數 | 展開為 | 範例 | 解析時機 |
|----------|-----------|---------|---------------|
| `{time}` | 目前時間（HH:mm:ss） | `14:30:00` | 提示詞編譯時 |
| `{date}` | 目前日期（YYYY-MM-DD） | `2026-05-10` | 提示詞編譯時 |
| `{sent_time}` | 訊息傳送時間（HH:mm） | `10:05` | 每則訊息 |
| `{sent_date}` | 訊息傳送日期（YYYY-MM-DD） | `2026-05-11` | 每則訊息 |
| `{active_memory}` | 使用中記憶體的內容 | `[您儲存的記憶體內容]` | 提示詞編譯時 |
| `{model_id}` | 目前選擇的模型 ID | `gemini-1.5-flash` | 提示詞編譯時 |

**每則訊息變數**（`{sent_time}`、`{sent_date}`）在您每次傳送訊息時解析，因此它們反映確切的傳送時間。**提示詞層級變數**（`{time}`、`{date}`、`{active_memory}`、`{model_id}`）在系統提示詞編譯時解析。

!!! tip
    對於日期敏感的提示詞，使用 `{sent_date}`，例如「今天是 {sent_date}。當討論近期事件時，請注意您的知識可能已過時。」使用 `{active_memory}` 將模型的持續性記憶體注入系統指令。

### 新增變數

1. 在編輯器的任何區段中，點選 **新增變數**
2. 從選擇器中選取變數
3. 它會以藥丸/晶片形式顯示在區段中 — 拖曳以重新定位

## 管理提示詞

### 設為預設

點選提示詞旁的圓形按鈕將其設為 **全域預設值**。除非被覆寫，否則所有對話都使用此提示詞。

### 個別對話覆寫

每個對話可以使用不同的系統提示詞：

1. 開啟一個對話
2. 點選頂部列中的溢出選單（:material-dots-vertical:）
3. 選擇 **對話提示詞**
4. 從清單中選擇提示詞

個別對話設定僅對該對話覆寫全域預設值。

### 編輯或刪除

- 點選提示詞以 **編輯** 它
- 長按並選擇 **刪除** 以移除它

!!! warning
    刪除系統提示詞是永久性的。使用該提示詞的對話將退回使用全域預設值。

---

## 不使用系統提示詞

如果未選擇系統提示詞，模型不會收到特殊指令 — 它將根據其基礎訓練來運作。這有時適合用於測試，或對於在沒有系統指令的情況下表現更好的模型。

要使用無提示詞，從提示詞清單中選擇 **無**。

---

## 自動標題生成

Agora 可以在第一次回應後自動生成對話標題：

1. 前往 **設定 → 標題生成**
2. 啟用 **自動生成標題**
3. 選擇一個 **標題模型**：
    - **使用目前模型** — 使用對話中啟用的任何模型
    - **選擇標題模型** — 選擇一個特定快速/便宜的模型用於標題生成

啟用後，在第一次模型回應後會出現短暫的「正在生成標題...」Snackbar，對話會自動從「未命名」重新命名為描述性標題。

## 提示詞範例

### 翻譯器

```yaml
系統提示詞: |
  您是一位專業翻譯員。將使用者輸入翻譯成英文。
  保留格式、程式碼區塊和技術術語。不要新增解釋。
```

### 程式碼審查者

```yaml
系統提示詞: |
  您是一位資深程式碼審查者。當看到程式碼時：
  1. 找出錯誤和邊界情況
  2. 建議效能改進
  3. 檢查安全性問題
  請具體說明。盡可能引用行號。
```

### 中文助手

```yaml
系統提示詞: |
  你是一個樂於助人的中文助手。用簡潔、清晰的中文回答問題。
使用者後綴: |
  \n\n請用中文回答。
```
