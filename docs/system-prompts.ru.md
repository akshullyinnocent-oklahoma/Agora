# Системные промпты

Системные промпты определяют личность, поведение и основные правила модели. Agora предоставляет вам детальный контроль над тем, как инструкции собираются и отправляются модели.

## Трёхсекционный редактор

Каждый шаблон системного промпта имеет три независимо редактируемые секции:

```text
┌─────────────────────────────────┐
│ Системный промпт                │ ← Основные инструкции (личность, правила, тон)
├─────────────────────────────────┤
│ Пользовательский префикс        │ ← Добавляется перед каждым сообщением пользователя
├─────────────────────────────────┤
│ Пользовательский суффикс        │ ← Добавляется после каждого сообщения пользователя
└─────────────────────────────────┘
```

### Системный промпт

Основной блок инструкций. Здесь вы определяете:

- **Личность**: «Ты — старший Python-разработчик, ориентированный на чистую архитектуру.»
- **Правила**: «Всегда отвечай на китайском. Используй маркированные списки для перечислений.»
- **Ограничения**: «Никогда не извиняйся. Будь краток. Предпочитай код объяснениям.»

### Пользовательский префикс и суффикс

Они обрамляют каждое отправляемое вами сообщение:

- **Пользовательский префикс** — добавляется перед текстом вашего сообщения. Полезно для напоминаний или контекстных тегов.
- **Пользовательский суффикс** — добавляется после текста вашего сообщения. Полезно для заключительных инструкций.

**Пример**: Если ваш префикс — `[Контекст: работа над документацией Agora]`, а суффикс — `\n\nПожалуйста, ответь в Markdown.`, модель получает:

```text
[Контекст: работа над документацией Agora]
Как настроить веб-поиск?
Пожалуйста, ответь в Markdown.
```

## Создание промпта

1. Перейдите в **Настройки → Системные промпты**
2. Нажмите **Добавить новый промпт**
3. Введите **заголовок** (например, «Переводчик», «Ревьюер кода», «Китайский ассистент»)
4. Заполните три секции:
    - Нажмите **Добавить текст**, чтобы написать статическое содержимое
    - Нажмите **Добавить переменную**, чтобы вставить динамические значения
5. Нажмите **Сохранить**

### Изменение порядка элементов

В каждой секции у вас может быть несколько текстовых блоков и переменных. Долгое нажатие на элемент позволяет:

- **Переместить вверх** / **Переместить вниз** — изменить порядок в секции
- **Удалить** — удалить элемент

## Подстановка переменных

Переменные заменяются динамическими значениями при отправке сообщения:

| Переменная | Раскрывается в | Пример | Когда разрешается |
|----------|-----------|---------|---------------|
| `{time}` | Текущее время (ЧЧ:мм:сс) | `14:30:00` | Компиляция промпта |
| `{date}` | Текущая дата (ГГГГ-ММ-ДД) | `2026-05-10` | Компиляция промпта |
| `{sent_time}` | Время отправки сообщения (ЧЧ:мм) | `10:05` | На каждое сообщение |
| `{sent_date}` | Дата отправки сообщения (ГГГГ-ММ-ДД) | `2026-05-11` | На каждое сообщение |
| `{active_memory}` | Содержимое активной памяти | `[Содержимое вашей сохранённой памяти]` | Компиляция промпта |
| `{model_id}` | ID текущей выбранной модели | `gemini-1.5-flash` | Компиляция промпта |

**Переменные на каждое сообщение** (`{sent_time}`, `{sent_date}`) разрешаются каждый раз при отправке сообщения, поэтому они отражают точное время отправки. **Переменные уровня промпта** (`{time}`, `{date}`, `{active_memory}`, `{model_id}`) разрешаются при компиляции системного промпта.

!!! tip
    Используйте `{sent_date}` для промптов, чувствительных к дате, например «Сегодня {sent_date}. При обсуждении недавних событий учти, что твои знания могут быть устаревшими.» Используйте `{active_memory}` для внедрения постоянной памяти модели в системные инструкции.

### Добавление переменной

1. В любой секции редактора нажмите **Добавить переменную**
2. Выберите переменную из списка
3. Она появится как пилюля/чип в секции — перетащите для изменения позиции

## Управление промптами

### Установить по умолчанию

Нажмите радиокнопку рядом с промптом, чтобы сделать его **глобальным по умолчанию**. Все диалоги используют этот промпт, если не переопределено.

### Переопределение для конкретного диалога

Каждый диалог может использовать другой системный промпт:

1. Откройте диалог
2. Нажмите меню переполнения (:material-dots-vertical:) в верхней панели
3. Выберите **Промпт диалога**
4. Выберите промпт из списка

Настройка для конкретного диалога переопределяет глобальный по умолчанию только для этого диалога.

### Редактирование или удаление

- Нажмите на промпт, чтобы **редактировать** его
- Долгое нажатие и выбор **Удалить**, чтобы удалить его

!!! warning
    Удаление системного промпта необратимо. Диалоги, которые его использовали, вернутся к глобальному по умолчанию.

## Без системного промпта

Если системный промпт не выбран, модель не получает специальных инструкций — она ведёт себя в соответствии со своим базовым обучением. Это иногда желательно для тестирования или для моделей, которые работают лучше без системных инструкций.

Чтобы не использовать промпт, выберите **Нет** из списка промптов.

## Автоматическая генерация заголовков

Agora может автоматически генерировать заголовки диалогов после первого ответа:

1. Перейдите в **Настройки → Генерация заголовков**
2. Включите **Авто-генерация заголовка**
3. Выберите **Модель для заголовков**:
    - **Использовать текущую модель** — использует модель, активную в диалоге
    - **Выбрать модель для заголовков** — выберите конкретную быструю/дешёвую модель для генерации заголовков

При включении после первого ответа модели появляется краткий snackbar «Генерация заголовка...», и диалог автоматически переименовывается из «Без названия» в описательный заголовок.

## Примеры промптов

### Переводчик

```yaml
Системный промпт: |
  Ты — профессиональный переводчик. Переводи ввод пользователя на английский.
  Сохраняй форматирование, блоки кода и технические термины. Не добавляй объяснений.
```

### Ревьюер кода

```yaml
Системный промпт: |
  Ты — старший ревьюер кода. Когда тебе показывают код:
  1. Выявляй ошибки и граничные случаи
  2. Предлагай улучшения производительности
  3. Проверяй проблемы безопасности
  Будь конкретным. Ссылайся на номера строк, когда возможно.
```

### Китайский ассистент

```yaml
Системный промпт: |
  你是一个乐于助人的中文助手。用简洁、清晰的中文回答问题。
Пользовательский суффикс: |
  \n\n请用中文回答。
```
