# 이미지 트랜스크립션

비전 모델이 이미지를 설명하도록 하여 텍스트 전용 모델이 이미지를 이해할 수 있게 합니다.

## 기능

텍스트 전용 모델에 이미지를 보낼 때, Agora는 별도의 비전 모델을 사용하여 먼저 이미지의 텍스트 설명을 생성할 수 있습니다. 이 설명은 이후 주 모델에 전송되는 프롬프트에 포함됩니다.

이를 통해 네이티브 비전을 지원하지 않는 모델에서도 이미지를 사용할 수 있습니다.

## 설정

1. **설정 → 이미지 트랜스크립션**으로 이동
2. **트랜스크립션 모델** 선택 — 비전 지원 모델이어야 함 (예: GPT-4o, Gemini Flash, Qwen-VL)
3. **활성화된 모델**에 모델 추가 — 이미지 설명을 받을 텍스트 전용 모델
4. 여러 이미지를 한 번에 보내는 경우 **배치 크기** 조정 (API 호출당 설명할 이미지 수)

!!! tip "로컬 비전 모델"
    로컬 비전 모델(mmproj 포함)을 트랜스크립션 모델로 사용할 수 있습니다. 이렇게 하면 이미지 처리가 기기 내에서 유지됩니다.

## 작동 방식

1. 메시지에 이미지 첨부
2. Agora가 현재 모델이 비전을 지원하지 않음을 감지
3. 이미지가 먼저 트랜스크립션 모델로 전송됨
4. 트랜스크립션 모델이 텍스트 설명 생성
5. 이 설명이 메시지 텍스트 앞에 추가됨
6. 결합된 텍스트가 주 모델로 전송됨

---

## 배치 크기

트랜스크립션 모델에 대한 API 호출당 설명할 이미지 수를 제어합니다.

- **1** — 한 번에 하나의 이미지 설명 (더 많은 API 호출, 더 정확)
- **5–10** — 호출당 여러 이미지 설명 (더 적은 API 호출, 세부 사항 손실 가능)

기본값은 기기에 따라 다릅니다. 값이 낮을수록 더 나은 결과를 얻지만 비용이 더 듭니다.

---

## 모델 선택

### 트랜스크립션 모델

이미지 설명을 생성하는 비전 모델입니다. 사용 가능한 가장 유능한 비전 모델을 선택하세요.

### 활성화된 모델

이미지 트랜스크립션을 사용할 텍스트 전용 모델입니다. 이 목록에 있는 모델만 트랜스크립션된 이미지 설명을 받습니다. 다른 모델은 이미지를 직접 받거나(지원하는 경우) 전혀 받지 않습니다.
