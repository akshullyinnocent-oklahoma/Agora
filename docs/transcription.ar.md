# تفريغ الصور

دع نموذج رؤية يصف الصور بحيث يمكن للنماذج النصية فقط فهمها.

## ما يفعله

عند إرسال صورة إلى نموذج نصي فقط، يمكن لـ Agora استخدام نموذج رؤية منفصل لتوليد وصف نصي للصورة أولاً. هذا الوصف يُضمن بعد ذلك في التوجيه المرسل إلى نموذجك الرئيسي.

هذا يتيح لك استخدام الصور مع أي نموذج، حتى تلك التي لا تدعم الرؤية أصلاً.

## الإعداد

1. اذهب إلى **الإعدادات → تفريغ الصور**
2. اختر **نموذج التفريغ** — يجب أن يكون نموذجًا قادرًا على الرؤية (مثل GPT-4o، Gemini Flash، Qwen-VL)
3. أضف نماذج إلى **النماذج المفعلة** — هذه هي النماذج النصية فقط التي ستتلقى أوصاف الصور
4. عدّل **حجم الدفعة** إذا أرسلت صورًا متعددة دفعة واحدة (كم صورة توصف لكل استدعاء API)

!!! tip "نماذج الرؤية المحلية"
    يمكنك استخدام نموذج رؤية محلي (مع mmproj) كنموذج تفريغ. هذا يبقي معالجة الصور على الجهاز.

## كيف يعمل

1. ترفق صورة برسالتك
2. يكتشف Agora أن نموذجك الحالي لا يدعم الرؤية
3. الصورة ترسل إلى نموذج التفريغ أولاً
4. نموذج التفريغ يولد وصفًا نصيًا
5. هذا الوصف يضاف قبل نص رسالتك
6. النص المدمج يرسل إلى نموذجك الرئيسي

---

## حجم الدفعة

يتحكم في كم صورة توصف لكل استدعاء API لنموذج التفريغ.

- **1** — وصف صورة واحدة في كل مرة (استدعاءات API أكثر، أكثر دقة)
- **5–10** — وصف صور متعددة لكل استدعاء (استدعاءات API أقل، قد تفقد التفاصيل)

الافتراضي يعتمد على الجهاز. القيم المنخفضة تعطي نتائج أفضل لكنها تكلف أكثر.

---

## اختيار النموذج

### نموذج التفريغ

هذا هو نموذج الرؤية الذي يولد أوصاف الصور. اختر أكثر نموذج رؤية قدرة متاح لك.

### النماذج المفعلة

هذه هي النماذج النصية فقط التي ستستخدم تفريغ الصور. فقط النماذج في هذه القائمة ستتلقى أوصاف صور مفرغة. النماذج الأخرى ستتلقى الصور مباشرة (إذا كانت تدعمها) أو لا تتلقاها على الإطلاق.
