# 對話搜尋

Agora 可以搜尋您的整個對話記錄 — 可以透過關鍵字比對或使用嵌入模型的語義（基於含義）檢索。

## 搜尋方法

### 關鍵字搜尋

快速、精確的文字比對。搜尋會在訊息內容中尋找您查詢的字面出現。

**最適合：**
- 尋找特定片語或詞彙
- 當您記得確切措辭時快速查找
- 無需設定 — 立即生效

**限制：**
- 無法找到同義詞和相關概念
- 不理解含義

### 語義搜尋（RAG）

使用嵌入模型按 **含義** 尋找訊息，而非精確詞彙。查詢「如何設定資料庫」可以找到有關「Room 設定」的訊息，即使「資料庫」一詞從未出現。

**最適合：**
- 按主題或主旨尋找對話
- 當您不記得確切措辭時的廣泛查詢
- 在不同對話中發現相關討論

**需求：**
- 必須設定嵌入模型（請參閱 [嵌入 / RAG](embedding.md)）
- 訊息必須已快取（已生成嵌入向量）

---

## 設定

### 1. 新增嵌入模型

請參閱 [嵌入 / RAG](embedding.md) 了解詳細設定。您可以使用：
- **遠端模型**（OpenAI、Mistral、Voyage、Ollama 等）
- **本機模型**（GGUF 檔案，完全離線）

### 2. 選擇搜尋方法

在 **設定 → 對話搜尋** 中：

| 設定 | 描述 |
|---------|-------------|
| **模型搜尋方法** | 模型呼叫 `search_conversations` 工具時的搜尋方式 |
| **手動搜尋方法** | 對話抽屜中搜尋列的運作方式 |

將每項設定為 **關鍵字** 或 **語義（RAG）**。

### 3. 設定搜尋範圍

| 設定 | 範圍 | 描述 |
|---------|-------|-------------|
| **每個搜尋匹配的上下文訊息數** | 4–32 | 每個匹配結果要包含多少則周圍訊息（級距：4、8、12、16、20、24、28、32） |
| **最大搜尋結果數** | 5–30 | 要回傳的最大匹配數量（級距：5、10、15、20、25、30） |
| **相似度閾值** | 0.0–1.0 | 僅 RAG：匹配的最低相似度分數。越高 = 越嚴格。預設：0.5 |

### 4. 快取訊息

如果使用 RAG，點選 **快取** 為所有現有訊息生成嵌入向量。啟用 **自動快取** 以自動保持索引更新。

## 使用搜尋

### 手動搜尋（搜尋列）

1. 開啟 **對話抽屜**（漢堡選單 :material-menu: 或向右滑動）
2. 點選頂部的搜尋列
3. 輸入您的查詢
4. 結果顯示在下方 — 點選任何結果以在匹配訊息處開啟該對話

### 模型發起的搜尋

當啟用 **存取過去對話**（設定 → 記憶體）時，模型可以自主搜尋您的記錄：

```text
您：「我們上週對 API 設計做出了什麼決定？」
模型：[搜尋「API 設計決定」]
       「上週二，我們決定使用...」
```

搜尋會以工具卡片的形式顯示在對話中。

---

## 相似度閾值

**相似度閾值** 滑桿（0.0 到 1.0）控制訊息必須多接近才能包含在 RAG 結果中：

- **低（0.3–0.5）**：更多結果，可能包含鬆散相關的內容
- **中（0.5–0.7）**：平衡 — 良好的預設值
- **高（0.7–0.9）**：更少結果，僅非常接近的匹配

從預設值開始，根據您的結果進行調整。如果得到太多不相關的匹配，提高閾值。如果錯過相關對話，降低閾值。

---

## 搜尋結果顯示

在對話抽屜中，搜尋結果顯示：

- **對話標題**（或「未命名」）
- **匹配訊息** — 匹配的使用者或模型訊息
- **角色標籤** — 使用者或模型
- **上下文訊息** — 用於提供上下文的周圍訊息

點選結果以開啟對話，並捲動到匹配的訊息。
