# 画像文字起こし

視覚モデルに画像を説明させることで、テキスト専用モデルが画像を理解できるようにします。

## 機能

テキスト専用モデルに画像を送信すると、Agora は別の視覚モデルを使用して最初に画像のテキスト説明を生成できます。この説明はメインモデルに送信されるプロンプトに含まれます。

これにより、視覚をネイティブにサポートしていないモデルでも画像を使用できます。

## セットアップ

1. **設定 → 画像文字起こし** に移動します
2. **文字起こしモデル** を選択します — これは視覚対応モデルである必要があります（例：GPT-4o、Gemini Flash、Qwen-VL）
3. **有効なモデル** にモデルを追加します — これらは画像説明を受け取るテキスト専用モデルです
4. 一度に多くの画像を送信する場合は **バッチサイズ** を調整します（API 呼び出しあたりの画像説明数）

!!! tip "ローカル視覚モデル"
    ローカル視覚モデル（mmproj 付き）を文字起こしモデルとして使用できます。これにより画像処理がデバイス上で行われます。

## 仕組み

1. メッセージに画像を添付します
2. Agora は現在のモデルが視覚をサポートしていないことを検出します
3. 画像は最初に文字起こしモデルに送信されます
4. 文字起こしモデルがテキスト説明を生成します
5. この説明がメッセージテキストの前に付加されます
6. 結合されたテキストがメインモデルに送信されます

---

## バッチサイズ

文字起こしモデルへの API 呼び出しあたりの画像説明数を制御します。

- **1** — 一度に1枚の画像を説明（API 呼び出しが多いが、より正確）
- **5～10** — 呼び出しあたり複数枚の画像を説明（API 呼び出しが少ないが、詳細が失われる可能性あり）

デフォルトはデバイスに依存します。低い値はより良い結果をもたらしますが、コストが高くなります。

---

## モデル選択

### 文字起こしモデル

これは画像説明を生成する視覚モデルです。利用可能な中で最も高性能な視覚モデルを選択してください。

### 有効なモデル

これらは画像文字起こしを使用するテキスト専用モデルです。このリストにあるモデルのみが文字起こしされた画像説明を受け取ります。他のモデルは画像を直接受け取るか（サポートしている場合）、全く受け取りません。
