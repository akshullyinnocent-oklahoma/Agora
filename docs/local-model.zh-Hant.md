# 本機模型

使用 GGUF 模型檔案與 llama.cpp 直接在您的 Android 裝置上執行 LLM。無需網路、無需 API 金鑰、完全私密。

## 運作方式

Agora 透過 Android NDK (CMake) 捆綁 llama.cpp。當您匯入 GGUF 檔案時，模型完全在您裝置的 CPU 上執行——沒有任何資料離開裝置。

## 需求

- 僅支援 **GGUF 格式**模型（llama.cpp 的標準格式）
- **裝置記憶體**：模型必須能放入可用 RAM。經驗法則：
    - 1–3B 參數模型：4–6 GB RAM
    - 7–8B 參數模型：6–8 GB RAM
- **儲存空間**：GGUF 檔案大小範圍從約 500 MB（量化小型模型）到 5 GB 以上

!!! warning
    本機推理是 CPU 密集型的，比雲端 API 慢得多。它最適合離線使用、隱私敏感內容或實驗——不適合快速、大量的聊天。

---

## 匯入聊天模型

1. 下載 GGUF 模型檔案到您的裝置（請參閱下方的推薦來源）
2. 前往 **設定 → 提供者**
3. 選擇 **本機** 作為提供者
4. 點選 **匯入 GGUF 模型**
5. 從裝置中選擇 `.gguf` 檔案
6. 設定模型：

| 參數 | 說明 | 範例 |
|-----------|-------------|---------|
| **模型 ID** | 小寫識別碼，不含空格 | `qwen3-8b` |
| **別名** | 顯示名稱 | `Qwen 3 8B` |
| **上下文大小** | 最大上下文視窗（以 token 為單位） | `4096` |
| **溫度** | 隨機性 (0.0–2.0) | `0.7` |
| **Top P** | 核心取樣閾值 (0.0–1.0) | `0.9` |
| **最大 Token 數** | 最大生成長度 | `2048` |

7. 點選 **新增**

模型已匯入，可立即使用。

---

## 匯入嵌入模型

嵌入模型較小，用於語義搜尋：

1. 前往 **設定 → 對話搜尋**
2. 點選 **新增本機模型**
3. 選擇一個 `.gguf` 嵌入模型檔案
4. 為其命名
5. 點選 **新增**

請參閱[嵌入 / RAG](embedding.md) 了解搜尋設定。

---

## 選擇使用中的模型

匯入一個或多個模型後：

1. 前往 **設定 → 提供者 → 本機**
2. 您會看到所有已匯入的模型列出
3. 點選您想使用的模型旁的**單選按鈕**
4. 當**本機**被選為聊天提供者時，所選模型即成為使用中的模型

---

## 管理本機模型

### 重新命名

點選模型以變更其別名或調整參數（溫度、上下文大小等）。

### 刪除

長按模型並點選 **刪除**。這會從 Agora 中移除模型，並從儲存空間中刪除 GGUF 檔案。

## 推薦模型

### 聊天模型

| 模型 | 大小 | 所需 RAM | 備註 |
|-------|------|------------|-------|
| Qwen 3 1.7B | ~1 GB | 3–4 GB | 以體積而言品質優良 |
| Llama 3.2 3B | ~2 GB | 4–5 GB | 全方位的穩健選擇 |
| Qwen 3 8B | ~5 GB | 7–8 GB | 最佳品質，高 RAM 需求 |

### 嵌入模型

| 模型 | 大小 | 備註 |
|-------|------|-------|
| BGE Small EN v1.5 | ~130 MB | 良好的英文嵌入 |
| BGE Small ZH v1.5 | ~130 MB | 中文最佳化 |
| Nomic Embed Text v1.5 | ~270 MB | 良好的多語言支援 |

### 取得 GGUF 檔案的位置

- [Hugging Face](https://huggingface.co/models?library=gguf) — 搜尋「GGUF」
- [bartowski 的量化模型](https://huggingface.co/bartowski) — 選擇廣泛、組織良好

!!! tip
    尋找 `Q4_K_M` 量化——它為聊天模型提供了品質與大小之間的最佳權衡。

---

## 效能提示

- **較小的上下文 = 更快**：從 2048 開始，僅在需要時增加
- **較低的量化 = 更快**：Q4_K_M 比 Q6_K 或 Q8 更快
- **關閉其他應用程式**：本機推理需要盡可能多的 RAM
- **插上電源**：推理是 CPU 密集型的，長時間使用會耗盡電池電量
