# 시스템 프롬프트

시스템 프롬프트는 모델의 페르소나, 행동 및 기본 규칙을 정의합니다. Agora는 명령이 어떻게 조합되어 모델에 전송되는지에 대한 세밀한 제어를 제공합니다.

## 3섹션 편집기

각 시스템 프롬프트 템플릿에는 독립적으로 편집 가능한 세 개의 섹션이 있습니다:

```text
┌─────────────────────────────────┐
│ 시스템 프롬프트                  │ ← 핵심 명령 (페르소나, 규칙, 어조)
├─────────────────────────────────┤
│ 사용자 접두사                    │ ← 모든 사용자 메시지 앞에 추가
├─────────────────────────────────┤
│ 사용자 접미사                    │ ← 모든 사용자 메시지 뒤에 추가
└─────────────────────────────────┘
```

### 시스템 프롬프트

주요 명령 블록입니다. 여기에서 다음을 정의합니다:

- **페르소나**: "당신은 클린 아키텍처에 중점을 둔 시니어 Python 개발자입니다."
- **규칙**: "항상 중국어로 응답하세요. 목록에는 글머리 기호를 사용하세요."
- **제약 사항**: "절대 사과하지 마세요. 간결하게. 설명보다 코드를 선호하세요."

### 사용자 접두사 & 접미사

보내는 모든 메시지를 감쌉니다:

- **사용자 접두사** — 메시지 텍스트 앞에 추가됩니다. 알림이나 컨텍스트 태그에 유용합니다.
- **사용자 접미사** — 메시지 텍스트 뒤에 추가됩니다. 마무리 명령에 유용합니다.

**예시**: 접두사가 `[컨텍스트: Agora 문서 작업 중]`이고 접미사가 `\n\nMarkdown으로 답변해 주세요.`인 경우, 모델이 받는 내용:

```text
[컨텍스트: Agora 문서 작업 중]
웹 검색을 어떻게 구성하나요?
Markdown으로 답변해 주세요.
```

## 프롬프트 생성

1. **설정 → 시스템 프롬프트**로 이동
2. **새 프롬프트 추가** 탭
3. **제목** 입력 (예: "번역가", "코드 리뷰어", "중국어 도우미")
4. 세 섹션 채우기:
    - **텍스트 추가**를 탭하여 정적 콘텐츠 작성
    - **변수 추가**를 탭하여 동적 값 삽입
5. **저장** 탭

### 항목 재정렬

각 섹션 내에서 여러 텍스트 블록과 변수를 가질 수 있습니다. 항목을 길게 눌러:

- **위로 이동** / **아래로 이동** — 섹션 내 재정렬
- **제거** — 항목 삭제

## 변수 치환

변수는 메시지 전송 시 동적 값으로 대체됩니다:

| 변수 | 확장값 | 예시 | 해결 시점 |
|----------|-----------|---------|---------------|
| `{time}` | 현재 시간 (HH:mm:ss) | `14:30:00` | 프롬프트 컴파일 시 |
| `{date}` | 현재 날짜 (YYYY-MM-DD) | `2026-05-10` | 프롬프트 컴파일 시 |
| `{sent_time}` | 메시지 전송 시간 (HH:mm) | `10:05` | 메시지별 |
| `{sent_date}` | 메시지 전송 날짜 (YYYY-MM-DD) | `2026-05-11` | 메시지별 |
| `{active_memory}` | 활성 메모리 내용 | `[저장된 메모리 내용]` | 프롬프트 컴파일 시 |
| `{model_id}` | 현재 선택된 모델 ID | `gemini-1.5-flash` | 프롬프트 컴파일 시 |

**메시지별 변수** (`{sent_time}`, `{sent_date}`)는 메시지를 보낼 때마다 해결되므로 정확한 전송 시간을 반영합니다. **프롬프트 수준 변수** (`{time}`, `{date}`, `{active_memory}`, `{model_id}`)는 시스템 프롬프트가 컴파일될 때 해결됩니다.

!!! tip
    "오늘은 {sent_date}입니다. 최근 이벤트를 논의할 때, 지식이 오래되었을 수 있음을 유의하세요."와 같은 날짜 민감 프롬프트에는 `{sent_date}`를 사용하세요. `{active_memory}`를 사용하여 모델의 영구 메모리를 시스템 명령에 주입하세요.

### 변수 추가

1. 편집기의 아무 섹션에서 **변수 추가** 탭
2. 선택기에서 변수 선택
3. 섹션에 필/칩으로 표시됨 — 드래그하여 위치 조정

## 프롬프트 관리

### 기본값으로 설정

프롬프트 옆의 라디오 버튼을 탭하여 **전역 기본값**으로 만듭니다. 재정의하지 않는 한 모든 대화가 이 프롬프트를 사용합니다.

### 대화별 재정의

각 대화는 다른 시스템 프롬프트를 사용할 수 있습니다:

1. 대화 열기
2. 상단 바의 오버플로 메뉴 (:material-dots-vertical:) 탭
3. **대화 프롬프트** 선택
4. 목록에서 프롬프트 선택

대화별 설정은 해당 대화에 한해 전역 기본값을 재정의합니다.

### 편집 또는 삭제

- 프롬프트를 탭하여 **편집**
- 길게 누르고 **삭제**를 선택하여 제거

!!! warning
    시스템 프롬프트 삭제는 영구적입니다. 이를 사용했던 대화는 전역 기본값으로 폴백됩니다.

## 시스템 프롬프트 없음

시스템 프롬프트가 선택되지 않은 경우, 모델은 특별한 명령 없이 기본 학습에 따라 행동합니다. 이는 테스트 목적이나 시스템 명령 없이 더 잘 수행되는 모델에 바람직할 수 있습니다.

프롬프트 없이 사용하려면 프롬프트 목록에서 **없음**을 선택하세요.

---

## 자동 제목 생성

Agora는 첫 번째 응답 후 자동으로 대화 제목을 생성할 수 있습니다:

1. **설정 → 제목 생성**으로 이동
2. **제목 자동 생성** 활성화
3. **제목 모델** 선택:
    - **현재 모델 사용** — 대화에서 활성화된 모델을 사용
    - **제목 모델 선택** — 제목 생성용으로 특정 빠른/저렴한 모델 선택

활성화 시, 첫 모델 응답 후 "제목 생성 중..." 스낵바가 잠시 나타나며, 대화가 "제목 없음"에서 설명적인 제목으로 자동 변경됩니다.

## 프롬프트 예시

### 번역가

```yaml
시스템 프롬프트: |
  당신은 전문 번역가입니다. 사용자 입력을 영어로 번역하세요.
  서식, 코드 블록, 기술 용어를 보존하세요. 설명을 추가하지 마세요.
```

### 코드 리뷰어

```yaml
시스템 프롬프트: |
  당신은 시니어 코드 리뷰어입니다. 코드를 보여주면:
  1. 버그와 엣지 케이스 식별
  2. 성능 개선 제안
  3. 보안 문제 확인
  구체적으로. 가능하면 줄 번호를 참조하세요.
```

### 중국어 도우미

```yaml
시스템 프롬프트: |
  你是一个乐于助人的中文助手。用简洁、清晰的中文回答问题。
사용자 접미사: |
  \n\n请用中文回答。
```
