# Модели

Управляйте тем, какие AI-модели доступны, и установите модель по умолчанию для диалогов.

## Список моделей

Страница **Модели** показывает все модели, известные Agora, сгруппированные по провайдерам:

- **Модель по умолчанию** — модель, используемая для новых диалогов. Нажмите для изменения.
- **Доступные модели** — разверните каждого провайдера, чтобы увидеть его модели. Включите те, которые хотите использовать.

### Включение / Отключение моделей

Установите или снимите флажок рядом с моделью, чтобы переключить её доступность. Отключённые модели не будут отображаться в выборе модели в диалогах.

### Переименование моделей

Нажмите значок редактирования (ручка) рядом с моделью, чтобы дать ей пользовательский псевдоним. Этот псевдоним отображается во всём приложении вместо технического ID модели.

### Синхронизация моделей

Нажмите **Синхронизировать модели**, чтобы загрузить последние доступные модели от всех настроенных API-провайдеров. Для этого требуется подключение к интернету и действительные API-ключи.

!!! tip "Локальные модели"
    Локальные модели отображаются в разделе провайдера **Локальный**. Они управляются отдельно в **Настройки → Провайдеры → Локальный**.

---

## Модель по умолчанию

**Модель по умолчанию** используется для всех новых диалогов. Чтобы изменить её:

1. Нажмите строку модели по умолчанию вверху страницы Модели
2. Выберите модель из списка (отображаются только включённые модели)
3. Изменение вступает в силу немедленно

Вы можете переопределить модель для конкретного диалога из выбора модели на экране чата.

---

## Псевдонимы моделей

Псевдонимы моделей позволяют давать понятные имена моделям с длинными техническими ID. Например, вы можете переименовать `openai/gpt-4o-mini` просто в «GPT-4o Mini».

Псевдонимы отображаются везде: в выборе модели, заголовках диалогов и на страницах настроек.

Чтобы удалить псевдоним, очистите текстовое поле и сохраните.

---

## Устранение неполадок

### Модели не отображаются

- Нажмите **Синхронизировать модели**, чтобы обновить список
- Убедитесь, что у вас есть действительный API-ключ для провайдера в **Настройки → Провайдеры**
- Проверьте подключение к интернету
- Некоторые провайдеры могут быть временно недоступны

### Локальные модели не отображаются

- Импортируйте файл GGUF модели в **Настройки → Провайдеры → Локальный**
- Модель должна быть в допустимом формате GGUF
