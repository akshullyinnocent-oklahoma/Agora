# 圖片轉錄

讓視覺模型描述圖片，以便純文字模型能夠理解它們。

## 功能說明

當您向純文字模型發送圖片時，Agora 可以使用一個獨立的視覺模型先產生圖片的文字描述。此描述隨後會被包含在傳送給您主要模型的提示中。

這讓您可以對任何模型使用圖片，即使該模型本身不支援視覺功能。

## 設定

1. 前往 **設定 → 圖片轉錄**
2. 選擇一個**轉錄模型** — 這應該是一個具備視覺能力的模型（例如 GPT-4o、Gemini Flash、Qwen-VL）
3. 將模型新增到**已啟用模型** — 這些是將接收圖片描述的純文字模型
4. 如果您一次發送多張圖片，請調整**批次大小**（每次 API 呼叫要描述多少張圖片）

!!! tip "本機視覺模型"
    您可以使用本機視覺模型（搭配 mmproj）作為轉錄模型。這可以將圖片處理保留在裝置上。

## 運作方式

1. 您在訊息中附加一張圖片
2. Agora 偵測到您目前的模型不支援視覺功能
3. 圖片首先被傳送到轉錄模型
4. 轉錄模型產生文字描述
5. 此描述被附加到您的訊息文字之前
6. 合併後的文字被傳送給您的主要模型

---

## 批次大小

控制每次 API 呼叫向轉錄模型描述多少張圖片。

- **1** — 一次描述一張圖片（更多 API 呼叫，更準確）
- **5–10** — 每次呼叫描述多張圖片（較少 API 呼叫，可能遺失細節）

預設值取決於裝置。較低的值可獲得更好的結果，但成本較高。

---

## 模型選擇

### 轉錄模型

這是產生圖片描述的視覺模型。請選擇您可用的最強大視覺模型。

### 已啟用模型

這些是將使用圖片轉錄的純文字模型。只有在此清單中的模型才會收到轉錄後的圖片描述。其他模型將直接接收圖片（如果它們支援）或完全不接收。
