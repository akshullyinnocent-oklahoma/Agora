# メモリ & キャッシュ

Agora には、モデルが会話をまたいで情報を記憶できる永続メモリシステムがあります。自動埋め込みベースのキャッシュと組み合わせることで、使用とともに成長するナレッジベースを提供します。

## メモリタイプ

### アクティブメモリ

モデルへの **すべての API 呼び出し** に含まれる単一の常時オンのメモリコンテキストです。モデルが常に目にする付箋のようなものと考えてください。

**アクティブメモリの用途：**
- 名前、好み、背景
- モデルが常に知っておくべきプロジェクトコンテキスト
- すべての会話に適用される継続的な指示
- 繰り返し伝えるのに疲れた事実

**アクティブメモリの内容例：**
```text
ユーザー: Newo Ether
好み: カジュアルなチャットは中国語、技術的なトピックは英語を好む。
プロジェクト: Agora — BYOK Android LLM クライアントを構築中。
コーディングスタイル: Kotlin、Jetpack Compose、MVVM アーキテクチャ。
```

#### アクティブメモリの編集

1. **設定 → メモリ** に移動します
2. **アクティブメモリ** までスクロールします
3. **アクティブメモリを編集** をタップします
4. 内容を入力します
5. **保存** をタップします

**アクティブメモリへのアクセス** が有効な場合、モデルはツール呼び出しを通じてアクティブメモリを更新することもできます。

---

### 保存メモリ

モデルが検索、読み取り、作成、編集、削除できる名前付きメモリファイルのコレクションです。アクティブメモリ（常に送信される）とは異なり、保存メモリは必要に応じて取得されます。

**保存メモリの用途：**
- 参考資料（API ドキュメント、設定詳細、コマンド）
- プロジェクト固有のメモ
- 過去の会話からの学びと洞察
- 関連性があるときにモデルに思い出してほしいもの

#### 手動でのメモリ作成

1. **設定 → メモリ** に移動します
2. **メモリを追加** をタップします
3. 以下を入力します：
    - **タイトル** — 説明的な名前
    - **説明** — 簡単な概要（検索マッチングに使用）
    - **内容** — 完全なメモリ内容
4. **作成** をタップします

#### モデルが作成するメモリ

**保存メモリへのアクセス** が有効な場合、モデルはツール呼び出しを通じてメモリファイルを作成、読み取り、更新、削除できます。これによりモデルは：

- 伝えられた事実を記憶する
- 便利なコードスニペットや設定を保存する
- 時間をかけてナレッジベースを構築する
- 古い情報をクリーンアップする

---

## メモリ権限

モデルがアクセスできるものを制御します：

| 設定 | 場所 | 有効にするタイミング |
|---------|----------|----------------|
| **保存メモリへのアクセス** | 設定 → メモリ | モデルにメモリファイルの読み取り/書き込みを許可したい場合 |
| **アクティブメモリへのアクセス** | 設定 → メモリ | モデルに永続コンテキストを更新させたい場合 |
| **過去の会話へのアクセス** | 設定 → 会話検索 | モデルにチャット履歴を検索させたい場合 |

3つすべてデフォルトで **オフ** です。必要なものだけ有効にしてください。

## 自動キャッシュ

自動キャッシュは、新しいメッセージが到着すると自動的に埋め込みを生成します。これにより、手動操作なしで会話検索インデックスが最新に保たれます。

### 自動キャッシュを有効にする

1. **設定 → 会話検索** に移動します
2. 埋め込みモデルを選択します（まだの場合は [埋め込み / RAG](embedding.md) を参照）
3. **キャッシュ** の下で **新しいメッセージを自動キャッシュ** をオンにします

有効にすると、すべての新しいメッセージ（ユーザーとモデル）が自動的に埋め込まれ、セマンティック検索用にインデックスされます。

### 手動キャッシュ

自動キャッシュがオフの場合、手動でメッセージをキャッシュできます：

1. **設定 → 会話検索** に移動します
2. **キャッシュ** をタップします — すべての未キャッシュメッセージの埋め込みを計算します
3. 進捗は円形インジケーターで表示されます

**再キャッシュ** をタップしてインデックス全体をゼロから再構築します。これによりすべてのキャッシュされた埋め込みが削除され、すべてのメッセージが再処理されます。以下の場合に使用します：
- 埋め込みモデルを変更した場合
- キャッシュが破損または古くなっていると思われる場合
- 検索結果が予想外に悪い場合

!!! warning
    再キャッシュは元に戻せず、メッセージ数と埋め込みモデルの速度によっては時間がかかることがあります。

### キャッシュステータス

埋め込みモデル設定は、キャッシュされたメッセージ数と未キャッシュのメッセージ数を表示します：
- **「すべての N 件のメッセージがキャッシュされました」** — 最新の状態
- **「Y 件中 X 件のメッセージがキャッシュされていません」** — 処理待ちのバックログ

## チャットでのメモリツール呼び出し

モデルがメモリツールを使用すると、インラインカードが表示されます：

| ツール | カードテキスト |
|------|-----------|
| 検索 | 「N 件の保存メモリを検索しました」 |
| 読み取り | 「[メモリ名] を読み取りました」 |
| 保存 | 「[メモリ名] を保存しました」 |
| 編集 | 「[メモリ名] を更新しました」 |
| 削除 | 「[メモリ名] を削除しました」 |
| アクティブ更新 | 「アクティブメモリを更新しました」 |

カードをタップすると、読み取りまたは書き込みされた完全な内容が表示されます。

---

## ベストプラクティス

- **アクティブメモリは簡潔に保つ** — すべての API 呼び出しに含まれるため、冗長な内容はトークンを浪費します
- **保存メモリには説明的なタイトルを使用する** — タイトルはモデルが適切なメモリを見つけるのに役立ちます
- **会話検索を定期的に使用する場合は自動キャッシュを有効にする**
- **埋め込みモデルを切り替えた後は再キャッシュする** — 異なるモデルは互換性のない埋め込みを生成します
