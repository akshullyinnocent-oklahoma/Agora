# 모델

사용 가능한 AI 모델을 관리하고 대화의 기본 모델을 설정하세요.

## 모델 목록

**모델** 페이지는 Agora가 알고 있는 모든 모델을 프로바이더별로 표시합니다:

- **기본 모델** — 새 대화에 사용되는 모델. 탭하여 변경.
- **사용 가능한 모델** — 각 프로바이더를 확장하여 모델 확인. 사용하려는 모델을 활성화하세요.

### 모델 활성화 / 비활성화

모델 옆의 체크박스를 선택 또는 해제하여 가용성을 토글합니다. 비활성화된 모델은 대화의 모델 선택기에 표시되지 않습니다.

### 모델 이름 변경

모델 옆의 편집 아이콘(펜)을 탭하여 사용자 지정 별칭을 지정합니다. 이 별칭은 기술 모델 ID 대신 앱 전체에 표시됩니다.

### 모델 동기화

**모델 동기화**를 탭하여 구성된 모든 API 프로바이더의 최신 사용 가능한 모델을 가져옵니다. 인터넷 연결과 유효한 API 키가 필요합니다.

!!! tip "로컬 모델"
    로컬 모델은 **로컬** 프로바이더 섹션에 표시됩니다. **설정 → 프로바이더 → 로컬**에서 별도로 관리됩니다.

---

## 기본 모델

**기본 모델**은 모든 새 대화에 사용됩니다. 변경 방법:

1. 모델 페이지 상단의 기본 모델 행 탭
2. 목록에서 모델 선택 (활성화된 모델만 표시됨)
3. 변경 사항이 즉시 적용됩니다

채팅 화면의 모델 선택기에서 대화별로 모델을 재정의할 수 있습니다.

---

## 모델 별칭

모델 별칭을 사용하면 긴 기술 ID를 가진 모델에 친숙한 이름을 지정할 수 있습니다. 예를 들어, `openai/gpt-4o-mini`를 "GPT-4o Mini"로 이름 변경할 수 있습니다.

별칭은 모델 선택기, 대화 헤더, 설정 페이지 등 모든 곳에 표시됩니다.

별칭을 제거하려면 텍스트 필드를 비우고 저장하세요.

---

## 문제 해결

### 모델이 표시되지 않음

- **모델 동기화**를 탭하여 목록 새로 고침
- **설정 → 프로바이더**에서 해당 프로바이더의 유효한 API 키가 있는지 확인
- 인터넷 연결 확인
- 일부 프로바이더가 일시적으로 사용 불가할 수 있음

### 로컬 모델이 표시되지 않음

- **설정 → 프로바이더 → 로컬**에서 GGUF 모델 파일 가져오기
- 모델이 유효한 GGUF 형식이어야 함
