# 대화

Agora의 대화 시스템은 **비선형 분기**를 중심으로 구축되었습니다 — 대부분의 채팅 앱과 달리, 과거 메시지를 편집하고 원래 대화를 잃지 않고 대체 응답 경로를 탐색할 수 있습니다.

## 대화 생성

대화 서랍에서 **새 채팅**을 탭하거나 채팅 화면에서 바로 입력을 시작하세요. 첫 메시지와 함께 새 대화가 자동으로 생성됩니다.

대화는 첫 응답 후 자동으로 제목이 지정되거나([제목 생성](system-prompts.ko.md#auto-title-generation)이 활성화된 경우), 수동으로 이름을 변경할 수 있습니다.

## 대화 관리

### 대화 전환

**대화 서랍**을 열고 (햄버거 메뉴 :material-menu: 또는 오른쪽으로 스와이프) 대화를 탭하여 엽니다.

### 이름 변경

1. 서랍에서 대화 길게 누르기
2. **이름 변경** 탭
3. 새 제목 입력 후 저장

### 삭제

1. 서랍에서 대화 길게 누르기
2. **삭제** 탭
3. 삭제 확인 — 이 작업은 되돌릴 수 없습니다

---

## 비선형 분기

이것이 Agora의 시그니처 기능입니다. 모든 메시지는 분기점이 될 수 있습니다.

### 과거 메시지 편집

1. 아무 메시지 버블 길게 누르기 (사용자 또는 모델)
2. **편집** 탭
3. 메시지 내용 수정
4. 전송 — Agora가 이 지점에서 **새 분기**를 생성합니다

원래 분기는 보존됩니다. 언제든지 분기 간 전환이 가능합니다.

### 분기 작동 방식

각 메시지는 **트리 구조**로 존재합니다:

```text
메시지 1 (사용자)
├── 메시지 2 (모델) ← 원래 응답
└── 메시지 3 (모델) ← 메시지 1 편집 후 생성된 분기
    ├── 메시지 4 (사용자)
    └── ...
```

메시지를 편집하고 재생성하면, 새 응답은 원본의 형제가 됩니다 — 둘 다 동일한 부모 메시지 아래에 존재합니다.

### 분기 전환

메시지에 여러 자식(분기)이 있는 경우, UI가 분기 간 전환을 위한 내비게이션 컨트롤을 표시합니다. 컨텍스트를 잃지 않고 대체 경로를 탐색할 수 있습니다.

### 왜 분기하나요?

- **대안 탐색** — 다른 표현으로 같은 질문하기
- **A/B 테스트 프롬프트** — 다른 시스템 프롬프트나 모델의 응답 비교
- **실수 수정** — 원래 스레드를 잃지 않고 질문의 오타 수정
- **반복** — 여러 버전을 통해 프롬프트 개선, 모든 시도 유지

## 메시지 작업

메시지를 길게 눌러 다음 작업에 접근:

| 작업 | 설명 |
|--------|-------------|
| **복사** | 메시지 텍스트를 클립보드에 복사 |
| **편집** | 메시지 편집 및 분기 생성 |
| **정보** | 메타데이터 보기: 타임스탬프, 사용된 모델, 토큰 수 |
| **삭제** | 이 메시지와 모든 후속 답장 삭제 |

!!! warning "메시지 삭제"
    메시지를 삭제하면 이어지는 모든 응답도 제거됩니다. 되돌릴 수 없습니다.

## 하단 바

채팅 입력 영역은 필수 컨트롤에 빠르게 접근할 수 있게 해줍니다:

### 모델 선택기

하단 바 왼쪽의 모델 이름을 탭하여 **모델 선택기**를 엽니다. 대화 중간에도 언제든지 모델을 전환할 수 있습니다. 같은 대화의 다른 메시지가 다른 모델을 사용할 수 있습니다.

### 첨부 파일

**+** (:material-plus:)를 탭하여 파일 첨부:

- **사진** — 갤러리의 이미지
- **동영상** — 비디오 파일 (프레임 추출 지원 포함)
- **파일** — PDF를 포함한 모든 파일 유형

지원되는 이미지 형식은 비전 지원 모델로 직접 전송됩니다. PDF 파일은 페이지 선택 대화 상자를 엽니다.

### 전송

메시지를 입력하고 **보내기** (:material-send:)를 탭합니다. 모델이 토큰별로 응답을 스트리밍합니다.

## 스트리밍 & 표시

### 실시간 스트리밍

응답은 모델이 생성함에 따라 단어별로 나타납니다. Agora는 최신 내용을 볼 수 있도록 자동 스크롤합니다. **하단으로 스크롤** 버튼(위로 스크롤 시 나타남)을 탭하여 실시간 응답으로 돌아갑니다.

### 마크다운 렌더링

모델 응답은 전체 마크다운 지원으로 렌더링됩니다:

- **헤더**, **굵게**, *기울임*, `인라인 코드`
- 구문 하이라이트가 포함된 **코드 블록** (````` ``` ````` 사용)
- **표**, 인용구, 목록
- **LaTeX 수식** — 인라인 `$E=mc^2$` 및 블록 `$$\int_a^b f(x)dx$$`

### Thinking 표시

추론을 지원하는 모델(OpenAI o-시리즈, Anthropic 확장 thinking, Gemini thinking, DeepSeek-R1)의 경우, 모델의 사고 과정이 최종 응답 전에 **접을 수 있는 패널**에 표시됩니다:

- 패널은 추론 단계에서 "생각 중..."을 표시
- 완료되면 사고 시간 표시 (예: "12초 동안 생각함")
- 탭하여 사고 내용 확장/축소
- 사고 중 이루어진 도구 호출이 집계됨 (예: "8초 생각, 2개 도구 호출")

## 대화별 설정

각 대화는 전역 기본값을 재정의할 수 있습니다:

- **모델** — 이 대화에 다른 모델 선택
- **시스템 프롬프트** — 다른 시스템 명령 사용
- **생성 매개변수** — temperature, 최대 토큰, thinking 수준

이러한 재정의는 상단 바의 대화 오버플로 메뉴에서 설정합니다.

---

## 컨텍스트 윈도우

Agora는 실시간으로 토큰 사용량을 추적합니다. 대화가 모델의 컨텍스트 윈도우를 초과하면, 오래된 메시지가 시각적으로 **희미해져** 활성 컨텍스트 밖에 있음을 나타냅니다. 모델은 더 이상 희미해진 메시지를 "보지" 못하지만, UI에는 계속 표시됩니다.

**설정 → 생성 → 컨텍스트 윈도우**에서 컨텍스트 윈도우 크기를 조정하세요.
