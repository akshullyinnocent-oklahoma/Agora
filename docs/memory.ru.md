# Память и кэш

Agora имеет постоянную систему памяти, которая позволяет модели запоминать информацию между диалогами. В сочетании с автоматическим кэшированием на основе эмбеддингов она предоставляет базу знаний, которая растёт с вашим использованием.

## Типы памяти

### Активная память

Единый, всегда активный контекст памяти, который включается в **каждый API-вызов** к модели. Представьте это как заметку, которую модель всегда видит.

**Используйте Активную память для:**
- Вашего имени, предпочтений и контекста
- Контекста проекта, который модель всегда должна знать
- Постоянных инструкций, применимых ко всем диалогам
- Фактов, которые вам надоело повторять

**Пример содержимого Активной памяти:**
```text
Пользователь: Newo Ether
Предпочтения: Предпочитает китайский для повседневного общения, английский для технических тем.
Проект: Разработка Agora — BYOK Android LLM-клиента.
Стиль кода: Kotlin, Jetpack Compose, архитектура MVVM.
```

#### Редактирование Активной памяти

1. Перейдите в **Настройки → Память**
2. Прокрутите до **Активная память**
3. Нажмите **Редактировать активную память**
4. Введите ваше содержимое
5. Нажмите **Сохранить**

Модель также может обновлять активную память через вызовы инструментов, если включён **Доступ к активной памяти**.

---

### Сохранённые воспоминания

Коллекция именованных файлов памяти, которые модель может искать, читать, создавать, редактировать и удалять. В отличие от Активной памяти (всегда отправляемой), сохранённые воспоминания извлекаются по требованию.

**Используйте Сохранённые воспоминания для:**
- Справочных материалов (документация API, детали конфигурации, команды)
- Заметок по конкретным проектам
- Знаний и инсайтов из прошлых диалогов
- Всего, что вы хотите, чтобы модель вспоминала, когда это уместно

#### Создание воспоминаний вручную

1. Перейдите в **Настройки → Память**
2. Нажмите **Добавить воспоминание**
3. Введите:
    - **Заголовок** — описательное имя
    - **Описание** — краткое резюме (используется для сопоставления при поиске)
    - **Содержимое** — полное содержимое воспоминания
4. Нажмите **Создать**

#### Воспоминания, созданные моделью

Когда включён **Доступ к сохранённым воспоминаниям**, модель может создавать, читать, обновлять и удалять файлы памяти через вызовы инструментов. Это позволяет модели:

- Запоминать факты, которые вы ей сообщаете
- Сохранять полезные фрагменты кода или конфигурации
- Со временем строить базу знаний
- Очищать устаревшую информацию

## Разрешения памяти

Управляйте тем, к чему модель может получить доступ:

| Настройка | Расположение | Когда включать |
|---------|----------|----------------|
| **Доступ к сохранённым воспоминаниям** | Настройки → Память | Вы хотите, чтобы модель читала/записывала файлы памяти |
| **Доступ к активной памяти** | Настройки → Память | Вы хотите, чтобы модель обновляла постоянный контекст |
| **Доступ к прошлым диалогам** | Настройки → Поиск по диалогам | Вы хотите, чтобы модель искала в истории чата |

Все три по умолчанию **отключены**. Включайте только то, что вам нужно.

## Авто-кэш

Авто-кэширование автоматически генерирует эмбеддинги для новых сообщений по мере их поступления. Это поддерживает ваш индекс поиска по диалогам в актуальном состоянии без ручного вмешательства.

### Включение авто-кэша

1. Перейдите в **Настройки → Поиск по диалогам**
2. Выберите модель эмбеддингов (если ещё не выбрали — см. [Эмбеддинги / RAG](embedding.ru.md))
3. В разделе **Кэширование** переключите **Авто-кэшировать новые сообщения**

При включении каждое новое сообщение (пользователя и модели) автоматически эмбеддируется и индексируется для семантического поиска.

### Ручное кэширование

Если авто-кэш отключён, вы можете кэшировать сообщения вручную:

1. Перейдите в **Настройки → Поиск по диалогам**
2. Нажмите **Кэшировать** — вычисляет эмбеддинги для всех некэшированных сообщений
3. Прогресс отображается в виде кругового индикатора

Нажмите **Перекэшировать**, чтобы перестроить весь индекс с нуля. Это удаляет все закэшированные эмбеддинги и заново обрабатывает каждое сообщение. Используйте, когда:
- Вы сменили модель эмбеддингов
- Кэш кажется повреждённым или устаревшим
- Результаты поиска неожиданно плохие

!!! warning
    Перекэширование необратимо и может занять некоторое время в зависимости от количества сообщений и скорости модели эмбеддингов.

### Статус кэша

Настройки модели эмбеддингов показывают, сколько сообщений закэшировано и сколько нет:
- **«Все N сообщений закэшированы»** — актуально
- **«X из Y сообщений не закэшированы»** — есть очередь на обработку

## Карточки вызовов инструментов памяти в чате

Когда модель использует инструменты памяти, вы увидите встроенные карточки:

| Инструмент | Текст карточки |
|------|-----------|
| Поиск | «Просмотрено N сохранённых воспоминаний» |
| Чтение | «Прочитано [имя воспоминания]» |
| Сохранение | «Сохранено [имя воспоминания]» |
| Редактирование | «Обновлено [имя воспоминания]» |
| Удаление | «Удалено [имя воспоминания]» |
| Обновление активной | «Обновлена активная память» |

Нажмите на любую карточку, чтобы увидеть полное содержимое, которое было прочитано или записано.

## Лучшие практики

- **Делайте Активную память краткой** — она включается в каждый API-вызов, поэтому многословное содержимое тратит токены
- **Используйте описательные заголовки для Сохранённых воспоминаний** — заголовки помогают модели найти нужное воспоминание
- **Включите авто-кэш**, если вы регулярно используете поиск по диалогам
- **Перекэшируйте после смены модели эмбеддингов** — разные модели создают несовместимые эмбеддинги
