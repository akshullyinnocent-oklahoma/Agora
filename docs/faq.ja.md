# よくある質問

## API とプロバイダー

### API キーはどうやって入手しますか？

- **Google Gemini**: [Google AI Studio](https://aistudio.google.com/apikey) — 無料枠あり
- **OpenAI**: [Platform API Keys](https://platform.openai.com/api-keys)
- **Anthropic**: [Console API Keys](https://console.anthropic.com/)
- **DeepSeek**: [プラットフォーム](https://platform.deepseek.com/)
- **OpenRouter**: [Keys ページ](https://openrouter.ai/keys)
- **Brave Search**: [Brave Search API](https://api.search.brave.com/)

### 同じプロバイダーに複数の API キーを使用できますか？

はい。各プロバイダーは複数の名前付きキーをサポートしています。ラジオボタンをタップしてアクティブなキーを選択します。仕事用/個人用キーの切り替えやバックアップキーの準備に便利です。[API プロバイダー](provider.md#api-keys) を参照してください。

### カスタムプロバイダーを追加するには？

設定 → プロバイダー → **+ カスタムプロバイダーを追加** に移動します。名前とベース URL を入力します。OpenAI 互換のエンドポイントであれば動作します。[カスタムプロバイダー](provider.md#custom-providers) を参照してください。

## ローカルモデル

### どの GGUF モデルが動作しますか？

Agora はチャットと埋め込みの両方で GGUF 形式をサポートしています。チャットモデルはデバイスメモリに収まる必要があります（RAM に応じて 1～8B パラメータ）。埋め込みモデルははるかに小さいです（100～500 MB）。[ローカルモデル](local-model.md) を参照してください。

### オフラインでモデルを実行するには？

設定 → プロバイダー → ローカル → **GGUF モデルをインポート** から GGUF チャットモデルをインポートします。完全オフラインのセマンティック検索を行うには、GGUF 埋め込みモデルもインポートします。ネットワーク接続は不要です。

### ローカルモデルが非常に遅いのはなぜですか？

ローカル推論はデバイスの CPU で実行されます。クラウド API よりも本質的に遅くなります。ヒント：より小さいモデル（1～3B パラメータ）を使用し、低い量子化（Q4_K_M）を選び、コンテキストウィンドウを短くし、バックグラウンドアプリを閉じてください。

## 埋め込みと検索

### 埋め込みモデルのテストが失敗するのはなぜですか？

よくある原因：

- **モデル名が間違っている** — Ollama タグを含めて正確なスペルを確認してください（例：`qwen3-embedding:8b` であって `qwen3-embedding` ではありません）
- **ベース URL が間違っている** — エンドポイントが `/v1/embeddings` をサポートしていることを確認してください
- **API キーがない** — 一部のプロバイダーは埋め込みにも認証が必要です
- **ネットワーク** — エンドポイントへの接続を確認してください

### キーワード検索と RAG 検索の違いは何ですか？

キーワード検索は正確なテキストに一致します。RAG（セマンティック検索）は意味で一致します — 「データベース設定」という検索で「Room 設定」という会話を見つけることができます（共通の単語がなくても）。RAG には埋め込みモデルとキャッシュされたメッセージが必要です。[会話検索](search.md) を参照してください。

### Ollama を埋め込みに使用するには？

1. マシンに Ollama をインストールします
2. 埋め込みモデルをプルします：`ollama pull qwen3-embedding:8b`
3. Agora で、**Ollama** プリセットを使用してリモート埋め込みモデルを追加します
4. ベース URL として `http://<host>:11434/v1` を使用します
5. タグを含む正確なモデル名を入力します（例：`qwen3-embedding:8b`）
6. API キーは空白のままにします

## メモリ

### アクティブメモリと保存メモリの違いは何ですか？

**アクティブメモリ** は、すべての API 呼び出しに含まれる単一の永続コンテキストです — モデルは常にそれを認識します。**保存メモリ** は、モデルが必要に応じて検索して取得する名前付きファイルのコレクションです。永続的な事実にはアクティブメモリを、参考資料には保存メモリを使用してください。[メモリ & キャッシュ](memory.md) を参照してください。

### モデルは私のメモリを変更できますか？

はい、設定 → メモリで **保存メモリへのアクセス** および/または **アクティブメモリへのアクセス** を有効にした場合に可能です。モデルはツール呼び出しを通じてメモリを作成、読み取り、編集、削除できます。すべての権限はデフォルトでオフです。

## シェルとツール

### リモートシェルアクセスを設定するには？

ターゲットマシンに [Conch](https://github.com/newo-ether/conch) サーバーをデプロイし、設定 → シェルで URL と API キーを使ってデバイスを追加します。[リモートシェル](shell.md) を参照してください。

### シェル接続は暗号化されていますか？

はい。Conch は ECDH 鍵交換 + AES-256-GCM 暗号化 + HMAC-SHA256 署名を使用しています。Agora と Conch サーバー間のすべてのトラフィックはエンドツーエンドで暗号化されています。

## データ

### データをバックアップするには？

設定 → データ管理 → **データをエクスポート** に移動します。カテゴリを選択して `.agora` ファイルにエクスポートします。安全な場所に保管してください。[データポータビリティ](import-export.md) を参照してください。

### ChatGPT や Claude からインポートできますか？

はい。ChatGPT または Claude からデータをエクスポートし（`.zip` ファイルが提供されます）、設定 → データ管理 → **サードパーティ** でインポートします。マージと置換の両方の戦略がサポートされています。[データポータビリティ](import-export.md#third-party-import) を参照してください。

### API キーはエクスポートに含まれますか？

含めることは可能ですが、オプションです。エクスポート画面では API キーの含有を切り替えられます。有効にすると警告が表示されます。キーは `.agora` ファイル内に平文で保存されるため、信頼できる宛先への完全なデバイス移行時にのみ含めてください。

## 全般

### データはどこに保存されていますか？

すべてのデータは Android デバイス上の Room データベースにローカル保存されます。Agora にはサーバーもクラウド同期もテレメトリもありません。メッセージはデバイスから設定した AI プロバイダーに直接送信されます。

### Agora は複数言語をサポートしていますか？

はい。アプリの UI は **English** と **中文（中国語）** をサポートしています。設定 → 言語。切り替え後に再起動が必要です。
