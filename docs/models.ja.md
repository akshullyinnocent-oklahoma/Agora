# モデル

利用可能な AI モデルを管理し、会話のデフォルトモデルを設定します。

## モデルリスト

**モデル** ページには、Agora が認識しているすべてのモデルがプロバイダーごとに整理されて表示されます：

- **デフォルトモデル** — 新しい会話に使用されるモデル。タップして変更します。
- **利用可能なモデル** — 各プロバイダーを展開してモデルを表示します。使用したいものを有効にします。

### モデルの有効化 / 無効化

モデルの横のチェックボックスをオン/オフにして、利用可能性を切り替えます。無効化されたモデルは会話のモデルピッカーに表示されません。

### モデルの名前変更

モデルの横の編集アイコン（ペン）をタップして、カスタムエイリアスを付けます。このエイリアスは技術的なモデル ID の代わりにアプリ全体で表示されます。

### モデルの同期

**モデルを同期** をタップして、設定されたすべての API プロバイダーから最新の利用可能モデルを取得します。インターネット接続と有効な API キーが必要です。

!!! tip "ローカルモデル"
    ローカルモデルは **ローカル** プロバイダーセクションに表示されます。これらは **設定 → プロバイダー → ローカル** で個別に管理されます。

---

## デフォルトモデル

**デフォルトモデル** はすべての新しい会話に使用されます。変更するには：

1. モデルページの上部にあるデフォルトモデルの行をタップします
2. リストからモデルを選択します（有効なモデルのみ表示されます）
3. 変更はすぐに反映されます

チャット画面のモデルピッカーから会話ごとにモデルを上書きすることもできます。

---

## モデルエイリアス

モデルエイリアスを使用すると、長い技術的 ID を持つモデルにわかりやすい名前を付けることができます。例えば、`openai/gpt-4o-mini` を「GPT-4o Mini」に名前変更できます。

エイリアスはすべての場所に表示されます：モデルピッカー、会話ヘッダー、設定ページ。

エイリアスを削除するには、テキストフィールドをクリアして保存します。

---

## トラブルシューティング

### モデルが表示されない

- **モデルを同期** をタップしてリストを更新します
- **設定 → プロバイダー** でプロバイダーの有効な API キーがあることを確認します
- インターネット接続を確認します
- 一部のプロバイダーは一時的に利用できない場合があります

### ローカルモデルが表示されない

- **設定 → プロバイダー → ローカル** で GGUF モデルファイルをインポートします
- モデルは有効な GGUF 形式である必要があります
