# 자주 묻는 질문

## API & 프로바이더

### API 키는 어떻게 얻나요?

- **Google Gemini**: [Google AI Studio](https://aistudio.google.com/apikey) — 무료 티어 이용 가능
- **OpenAI**: [Platform API Keys](https://platform.openai.com/api-keys)
- **Anthropic**: [Console API Keys](https://console.anthropic.com/)
- **DeepSeek**: [Platform](https://platform.deepseek.com/)
- **OpenRouter**: [Keys page](https://openrouter.ai/keys)
- **Brave Search**: [Brave Search API](https://api.search.brave.com/)

### 같은 프로바이더에 여러 API 키를 사용할 수 있나요?

네. 각 프로바이더는 여러 개의 이름 지정된 키를 지원합니다. 라디오 버튼을 탭하여 활성 키를 선택하세요. 업무/개인 키 간 전환이나 백업 키 준비에 유용합니다. [API 프로바이더](provider.ko.md#api-keys)를 참조하세요.

### 사용자 지정 프로바이더는 어떻게 추가하나요?

설정 → 프로바이더 → **+ 사용자 지정 프로바이더 추가**로 이동합니다. 이름과 기본 URL을 입력하세요. 모든 OpenAI 호환 엔드포인트가 작동합니다. [사용자 지정 프로바이더](provider.ko.md#custom-providers)를 참조하세요.

## 로컬 모델

### 어떤 GGUF 모델이 작동하나요?

Agora는 채팅 및 임베딩 모두에 GGUF 형식을 지원합니다. 채팅 모델은 기기 메모리에 맞아야 합니다 (RAM에 따라 1–8B 매개변수). 임베딩 모델은 훨씬 작습니다 (100–500 MB). [로컬 모델](local-model.ko.md)을 참조하세요.

### 모델을 오프라인으로 실행하려면 어떻게 하나요?

설정 → 프로바이더 → 로컬 → **GGUF 모델 가져오기**를 통해 GGUF 채팅 모델을 가져옵니다. 완전 오프라인 시맨틱 검색을 위해서는 GGUF 임베딩 모델도 가져오세요. 네트워크 연결이 필요하지 않습니다.

### 로컬 모델이 너무 느린 이유는 무엇인가요?

로컬 추론은 기기의 CPU에서 실행됩니다. 클라우드 API보다 본질적으로 느립니다. 팁: 더 작은 모델 (1–3B 매개변수), 낮은 양자화 (Q4_K_M), 짧은 컨텍스트 윈도우, 백그라운드 앱 종료를 사용하세요.

## 임베딩 & 검색

### 임베딩 모델 테스트가 실패하는 이유는 무엇인가요?

일반적인 원인:

- **잘못된 모델 이름** — Ollama 태그를 포함한 정확한 철자 확인 (예: `qwen3-embedding:8b`, `qwen3-embedding` 아님)
- **잘못된 기본 URL** — 엔드포인트가 `/v1/embeddings`를 지원하는지 확인
- **API 키 누락** — 일부 프로바이더는 임베딩에도 인증이 필요합니다
- **네트워크** — 엔드포인트 연결 확인

### 키워드 검색과 RAG 검색의 차이점은 무엇인가요?

키워드 검색은 정확한 텍스트를 일치시킵니다. RAG(시맨틱 검색)는 의미로 일치시킵니다 — "데이터베이스 설정"이 공유 단어 없이도 "Room 구성"을 찾을 수 있습니다. RAG는 임베딩 모델과 캐시된 메시지가 필요합니다. [대화 검색](search.ko.md)을 참조하세요.

### Ollama를 임베딩에 어떻게 사용하나요?

1. 머신에 Ollama 설치
2. 임베딩 모델 풀: `ollama pull qwen3-embedding:8b`
3. Agora에서 **Ollama** 프리셋으로 원격 임베딩 모델 추가
4. 기본 URL로 `http://<host>:11434/v1` 사용
5. 태그를 포함한 정확한 모델 이름 입력 (예: `qwen3-embedding:8b`)
6. API 키는 비워 두기

## 메모리

### 활성 메모리와 저장된 메모리의 차이점은 무엇인가요?

**활성 메모리**는 모든 API 호출에 포함되는 단일 영구 컨텍스트입니다 — 모델이 항상 볼 수 있습니다. **저장된 메모리**는 모델이 필요에 따라 검색하고 조회하는 이름 지정된 파일 모음입니다. 영구적인 사실에는 활성 메모리를, 참조 자료에는 저장된 메모리를 사용하세요. [메모리 & 캐시](memory.ko.md)를 참조하세요.

### 모델이 내 메모리를 수정할 수 있나요?

네, 설정 → 메모리에서 **저장된 메모리 접근** 및/또는 **활성 메모리 접근**을 활성화한 경우 가능합니다. 모델은 도구 호출을 통해 메모리를 생성, 읽기, 편집, 삭제할 수 있습니다. 모든 권한은 기본적으로 꺼져 있습니다.

## 셸 & 도구

### 원격 셸 접근은 어떻게 설정하나요?

대상 머신에 [Conch](https://github.com/newo-ether/conch) 서버를 배포한 후, 설정 → 셸에서 URL과 API 키로 기기를 추가하세요. [원격 셸](shell.ko.md)을 참조하세요.

### API 키 없이 웹 검색을 할 수 있나요?

네. **DuckDuckGo Lite**는 기본 웹 검색 프로바이더이며 API 키가 필요하지 않습니다. 설정 → 웹 검색에서 웹 검색을 활성화하기만 하면 바로 작동합니다. 더 높은 신뢰성을 원하면 API 기반 프로바이더(Brave, Serper, Tavily, SearXNG)를 구성하세요. [웹 검색](web-search.ko.md)을 참조하세요.

### 셸 연결은 암호화되나요?

네. Conch는 ECDH 키 교환 + AES-256-GCM 암호화 + HMAC-SHA256 서명을 사용합니다. Agora와 Conch 서버 간의 모든 트래픽은 종단 간 암호화됩니다.

## 데이터

### 데이터를 어떻게 백업하나요?

설정 → 데이터 제어 → **데이터 내보내기**로 이동하여 수동 `.agora` 백업을 만듭니다. 번거로움 없는 보호를 위해 설정 → 데이터 제어 → 자동 백업에서 **자동 백업**을 활성화하세요 — 백그라운드에서 주기적으로 데이터를 백업합니다. [데이터 이식성](import-export.ko.md)을 참조하세요.

### ChatGPT나 Claude에서 가져올 수 있나요?

네. ChatGPT나 Claude에서 데이터를 내보내고 (`.zip` 파일 제공), 설정 → 데이터 제어 → **서드파티**에서 가져옵니다. 병합 및 교체 전략이 모두 지원됩니다. [데이터 이식성](import-export.ko.md#third-party-import)을 참조하세요.

### 내보내기에 API 키가 포함되나요?

포함될 수 있지만 선택 사항입니다. 내보내기 화면에서 API 키 포함을 토글할 수 있습니다. 활성화 시 경고가 표시됩니다. 키는 `.agora` 파일 내에 평문으로 저장되므로, 신뢰할 수 있는 대상으로의 전체 기기 마이그레이션에만 포함하세요.

## 일반

### 데이터는 어디에 저장되나요?

모든 것은 Android 기기의 Room 데이터베이스에 로컬로 저장됩니다. Agora는 서버, 클라우드 동기화, 원격 측정이 없습니다. 메시지는 기기에서 사용자가 구성한 AI 프로바이더로 직접 전송됩니다.

### Agora는 여러 언어를 지원하나요?

네. 앱 UI는 **English**, **中文 (중국어)**, **繁體中文 (번체 중국어)** 를 지원합니다. 설정 → 언어. 전환 후 재시작이 필요합니다.

### 버그 신고나 기능 요청은 어떻게 하나요?

[GitHub](https://github.com/newo-ether/Agora/issues)에서 이슈를 열어주세요. 기여에 관해서는 README의 [Contributing](https://github.com/newo-ether/Agora#contributing) 섹션을 참조하세요.
