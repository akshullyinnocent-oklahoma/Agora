# 임베딩 / RAG

임베딩 모델은 텍스트를 의미를 포착하는 숫자 벡터로 변환합니다. Agora는 이 벡터를 사용하여 대화 기록에 대한 시맨틱 검색(RAG)을 수행합니다 — 포함된 단어가 아닌 의미로 메시지를 찾습니다.

## 작동 방식

1. 각 메시지가 임베딩 모델로 전송됨
2. 모델이 메시지의 의미를 나타내는 벡터(숫자 목록) 반환
3. 검색 시, 쿼리도 임베딩됨
4. Agora가 쿼리 벡터와 모든 메시지 벡터 간의 **코사인 유사도** 계산
5. 임계값 이상의 유사도를 가진 메시지가 일치 항목으로 반환됨

## 지원되는 프로바이더

| 프로바이더 | 기본 URL | API 키 필요 | 참고 |
|----------|----------|------------------|-------|
| **OpenAI** | `https://api.openai.com/v1` | 예 | `text-embedding-3-small`, `text-embedding-3-large` |
| **Mistral** | `https://api.mistral.ai/v1` | 예 | `mistral-embed` |
| **Voyage AI** | `https://api.voyageai.com/v1` | 예 | `voyage-3`, `voyage-3-lite` |
| **SiliconFlow** | `https://api.siliconflow.cn/v1` | 예 | `BAAI/bge-large-zh-v1.5` (중국어 최적화) |
| **Ollama** | `http://localhost:11434/v1` | 아니요 | `qwen3-embedding`, `nomic-embed-text` 등 |
| **사용자 지정** | 모두 | 선택 사항 | 모든 OpenAI 호환 임베딩 엔드포인트 |
| **로컬** | N/A | 아니요 | llama.cpp 경유 GGUF 임베딩 모델 |

## 임베딩 모델 추가

### 원격 (API)

1. **설정 → 대화 검색**으로 이동
2. **원격 모델 추가** 탭
3. 구성:

| 필드 | 설명 |
|-------|-------------|
| **프로바이더** | 드롭다운에서 선택 (OpenAI, Mistral, Voyage, SiliconFlow, Ollama, 사용자 지정) |
| **모델 이름** | 정확한 모델 ID (예: `text-embedding-3-small`) |
| **기본 URL** | 알려진 프로바이더는 자동 입력됨; 프록시용 편집 가능 |
| **API 키** | 비워 두면 채팅 프로바이더 키에서 자동 해결, 또는 전용 키 입력 |
| **배치 크기** | API 요청당 임베딩할 메시지 수 (1–100) |

4. **추가** 탭 — 저장 전 연결 테스트 실행

!!! tip
    API 키 필드는 채팅용으로 이미 동일한 프로바이더를 구성한 경우 선택 사항입니다. 비워 두면 Agora가 채팅 API 키를 자동으로 해결합니다.

### 로컬 (GGUF)

1. **설정 → 대화 검색**으로 이동
2. **로컬 모델 추가** 탭
3. `.gguf` 임베딩 모델 파일 가져오기 (예: `bge-small-en-v1.5-q4_k.gguf`)
4. 이름 지정
5. **추가** 탭

임베딩 모델은 일반적으로 채팅 모델보다 훨씬 작습니다 — 많아야 수백 MB.

### Ollama

1. 머신에 Ollama 설치
2. 임베딩 모델 풀: `ollama pull qwen3-embedding:8b`
3. Agora에서 원격 모델 추가:
    - 프로바이더: **Ollama**
    - 기본 URL: `http://<host>:11434/v1`
    - 모델 이름: `qwen3-embedding:8b` (`:tag` 포함)
    - API 키: 비워 두기
4. **추가** 탭

!!! note
    `:8b`, `:latest`와 같은 Ollama 접미사 태그는 모델 이름의 일부입니다. `ollama list`의 정확한 이름을 사용하세요.

## 캐싱

모델 추가 후, 메시지를 캐시해야 합니다 (임베딩 생성):

1. 임베딩 모델에서 **캐시** 탭
2. Agora가 모든 캐시되지 않은 메시지를 배치로 처리
3. 원형 진행 표시기가 현재 진행 상황 표시
4. 완료: "N개 메시지 모두 캐시됨"

### 자동 캐시

**자동 캐시**를 활성화하여 새 메시지가 도착할 때 자동으로 임베딩합니다. 이를 통해 검색 인덱스가 항상 최신 상태로 유지됩니다.

### 재캐시

**재캐시**를 탭하여 모든 기존 임베딩을 삭제하고 처음부터 다시 빌드합니다. 사용 시기:

- 다른 임베딩 모델로 전환할 때
- 임베딩 품질이 저하된 것 같을 때
- 캐시가 불일치할 때

!!! warning
    재캐싱은 되돌릴 수 없으며 대규모 메시지 기록의 경우 오랜 시간이 걸릴 수 있습니다.

---

## 배치 크기

**배치 크기** 설정 (1–100)은 캐싱 중 API 요청당 전송되는 메시지 수를 제어합니다:

- **높을수록**: 더 빠른 캐싱, 그러나 더 큰 API 페이로드
- **낮을수록**: 더 작은 요청, 느리지만 느린 연결에서 더 안정적

기본값으로 시작하여 타임아웃이 발생하면 낮추거나, 더 빠른 캐싱을 원하면 높이세요.

---

## 설정 테스트

원격 모델 추가 시, Agora가 자동 연결 테스트를 실행합니다. 실패 시:

1. 모델 이름 확인 — Ollama의 경우 태그 포함 (`:8b`, `:latest`)
2. 기본 URL이 기기에서 접근 가능한지 확인
3. API 키가 유효한지 확인 (필요한 경우)
4. 해당 프로바이더의 알려진 모델 이름 시도

일반적인 오류:
- **"잘못된 모델 이름"** — 태그를 포함한 정확한 철자 확인
- **"잘못된 기본 URL"** — 엔드포인트가 `/v1/embeddings`를 지원하는지 확인
- **"API 키 누락"** — 일부 프로바이더는 인증이 필요
- **"네트워크 오류"** — 연결 확인

|----------|---------------------|
