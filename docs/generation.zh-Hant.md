# 生成參數

控制模型如何生成回應 — 從上下文長度到創意設定。

## 上下文視窗

**最大上下文訊息數** 設定有多少則最近的訊息會作為上下文傳送給模型。預設：**20**。

- **5–20** — 較短的上下文、更快的回應、較少的 token 使用量
- **20–50** — 較長的上下文，適用於複雜的多輪對話
- **50–100** — 最大上下文，用於非常長的討論（可能達到 token 限制）

這適用於所有模型。實際的 token 上下文視窗取決於您的模型和訊息長度。

---

## 溫度（Temperature）

控制模型輸出的隨機性。範圍：**0.0 – 2.0**。

- **0.0 – 0.3** — 更確定性、一致、事實性
- **0.5 – 0.8** — 平衡的創意（建議預設值）
- **1.0 – 2.0** — 更隨機、有創意、不可預測

溫度越高，模型越有可能選擇機率較低的詞彙。較低的溫度會產生更聚焦、重複性較高的輸出。

!!! tip "何時調整"
    - **程式碼 / 事實**：使用低溫度（0.0 – 0.3）
    - **創意寫作**：使用高溫度（0.8 – 1.2）
    - **一般聊天**：使用中等溫度（0.5 – 0.7）

---

## Top P（核取樣）

控制 token 選擇的多樣性。範圍：**0.0 – 1.0**。

模型僅考慮累積機率超過 `top_p` 的最小 token 集合。

- **0.1** — 非常聚焦，僅最可能的 token
- **0.5** — 中等多樣性
- **0.9 – 1.0** — 完全多樣性（建議預設值）

通常您調整溫度 *或* Top P 其中之一 — 而非兩者同時。

---

## 預設最大 Token 數

設定模型回應的最大 token 限制。設定後，模型在單一回應中不會生成超過此數量的 token。當 **未設定** 時（預設），則使用模型自身的最大值。

可用預設值：

```
256   512   1024   2048
4096  8192  16384  32768
```

!!! tip "保持未設定以獲得彈性"
    對於大多數使用案例，保持此項未設定。僅在您需要一致的回應長度（例如簡短摘要）或想控制成本時才設定限制。

---

## 頻率懲罰（Frequency Penalty）

降低模型重複相同詞彙的傾向。範圍：**-2.0 – 2.0**。

- **正值**（0.1 – 1.0）— 抑制重複
- **零**（0.0）— 無懲罰（預設）
- **負值**（-1.0 – -0.1）— 鼓勵重複

---

## 存在懲罰（Presence Penalty）

鼓勵模型談論新主題。範圍：**-2.0 – 2.0**。

- **正值**（0.1 – 1.0）— 鼓勵主題多樣性
- **零**（0.0）— 無懲罰（預設）
- **負值** — 停留在目前主題

---

## 思考 / 推理

為支援的模型（例如 DeepSeek R1、Qwen3、Claude）啟用思維鏈推理。

啟用後，模型會在產生最終回應之前生成內部推理。這可提高複雜任務的準確性，但需要更長時間並使用更多 token。

### 思考等級

- **低** — 最少推理，較快
- **中** — 平衡（預設）
- **高** — 最大推理，用於複雜問題

!!! warning "並非所有模型都支援思考"
    思考模式需要支援推理 token 的模型。如果您的模型不支援，此設定不會產生效果。

---

## 視覺化上下文捲出

啟用後，Agora 會以視覺方式指出哪些訊息包含在目前上下文視窗中，哪些已被捲出（因上下文視窗限制而被排除）。這有助於您了解：

- 模型能「看到」多少對話內容
- 舊訊息何時脫離上下文
- 是否需要增加上下文視窗

視覺化效果會以對話檢視中的微妙標記呈現。

---

## 參數運作方式

所有生成參數都是 **可為空的** — 當未明確設定時，它們不會被傳送給模型，模型將使用其自身的預設值。每個參數都有重設選項，可將值清除回「未設定」。

---

## 個別對話覆寫

您可以使用聊天畫面上的 **進階設定** 對話框（長按傳送按鈕或使用 ⋮ 選單），為個別對話覆寫生成參數。
