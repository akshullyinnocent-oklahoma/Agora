# Эмбеддинги / RAG

Модели эмбеддингов преобразуют текст в числовые векторы, capturing смысл. Agora использует эти векторы для семантического поиска (RAG) по истории ваших диалогов — находя сообщения по их значению, а не только по содержащимся в них словам.

## Как это работает

1. Каждое сообщение отправляется модели эмбеддингов
2. Модель возвращает вектор (список чисел), представляющий смысл сообщения
3. Когда вы ищете, ваш запрос также эмбеддируется
4. Agora вычисляет **косинусное сходство** между вектором запроса и всеми векторами сообщений
5. Сообщения со сходством выше вашего порога возвращаются как совпадения

## Поддерживаемые провайдеры

| Провайдер | Базовый URL | Требует API-ключ | Примечания |
|----------|----------|------------------|-------|
| **OpenAI** | `https://api.openai.com/v1` | Да | `text-embedding-3-small`, `text-embedding-3-large` |
| **Mistral** | `https://api.mistral.ai/v1` | Да | `mistral-embed` |
| **Voyage AI** | `https://api.voyageai.com/v1` | Да | `voyage-3`, `voyage-3-lite` |
| **SiliconFlow** | `https://api.siliconflow.cn/v1` | Да | `BAAI/bge-large-zh-v1.5` (оптимизирован для китайского) |
| **Ollama** | `http://localhost:11434/v1` | Нет | `qwen3-embedding`, `nomic-embed-text` и др. |
| **Пользовательский** | Любой | Опционально | Любая конечная точка эмбеддингов, совместимая с OpenAI |
| **Локальный** | Н/Д | Нет | GGUF модели эмбеддингов через llama.cpp |

## Добавление модели эмбеддингов

### Удалённая (API)

1. Перейдите в **Настройки → Поиск по диалогам**
2. Нажмите **Добавить удалённую модель**
3. Настройте:

| Поле | Описание |
|-------|-------------|
| **Провайдер** | Выберите из выпадающего списка (OpenAI, Mistral, Voyage, SiliconFlow, Ollama, Пользовательский) |
| **Имя модели** | Точный ID модели (например, `text-embedding-3-small`) |
| **Базовый URL** | Автозаполняется для известных провайдеров; редактируется для прокси |
| **API-ключ** | Оставьте пустым для авто-разрешения из ключа чат-провайдера или введите выделенный ключ |
| **Размер пакета** | Сообщений для эмбеддинга за один API-запрос (1–100) |

4. Нажмите **Добавить** — перед сохранением выполняется проверка подключения

!!! tip
    Поле API-ключа необязательно, если вы уже настроили того же провайдера для чата. Оставьте его пустым, и Agora автоматически определит ваш чат API-ключ.

### Локальная (GGUF)

1. Перейдите в **Настройки → Поиск по диалогам**
2. Нажмите **Добавить локальную модель**
3. Импортируйте файл модели эмбеддингов `.gguf` (например, `bge-small-en-v1.5-q4_k.gguf`)
4. Дайте ей имя
5. Нажмите **Добавить**

Модели эмбеддингов обычно значительно меньше чат-моделей — не более нескольких сотен МБ.

### Ollama

1. Установите Ollama на машину
2. Загрузите модель эмбеддингов: `ollama pull qwen3-embedding:8b`
3. В Agora добавьте удалённую модель:
    - Провайдер: **Ollama**
    - Базовый URL: `http://<хост>:11434/v1`
    - Имя модели: `qwen3-embedding:8b` (включая `:tag`)
    - API-ключ: оставьте пустым
4. Нажмите **Добавить**

!!! note
    Суффиксные теги Ollama, такие как `:8b`, `:latest`, являются частью имени модели. Используйте точное имя из `ollama list`.

## Кэширование

После добавления модели вам нужно закэшировать ваши сообщения (сгенерировать эмбеддинги):

1. Нажмите **Кэшировать** на модели эмбеддингов
2. Agora обрабатывает все некэшированные сообщения пакетами
3. Круговой индикатор выполнения показывает текущий прогресс
4. Завершение: «Все N сообщений закэшированы»

### Авто-кэш

Включите **Авто-кэш**, чтобы автоматически эмбеддировать новые сообщения по мере их поступления. Это поддерживает ваш поисковый индекс всегда в актуальном состоянии.

### Перекэширование

Нажмите **Перекэшировать**, чтобы удалить все существующие эмбеддинги и перестроить с нуля. Используйте, когда:

- Переключаетесь на другую модель эмбеддингов
- Качество эмбеддингов кажется ухудшенным
- Кэш неконсистентен

!!! warning
    Перекэширование нельзя отменить, и оно может занять много времени для больших историй сообщений.

## Размер пакета

Настройка **Размер пакета** (1–100) управляет тем, сколько сообщений отправляется за один API-запрос при кэшировании:

- **Выше**: Быстрее кэширование, но больше полезная нагрузка API
- **Ниже**: Меньше запросы, медленнее, но надёжнее на медленных соединениях

Начните со значения по умолчанию и корректируйте, если сталкиваетесь с таймаутами (уменьшите) или хотите более быстрого кэширования (увеличьте).

## Тестирование настройки

При добавлении удалённой модели Agora выполняет автоматическую проверку подключения. Если она не проходит:

1. Проверьте имя модели — включите теги для Ollama (`:8b`, `:latest`)
2. Убедитесь, что базовый URL достижим с вашего устройства
3. Подтвердите, что API-ключ действителен (если требуется)
4. Попробуйте известное имя модели для этого провайдера

Распространённые ошибки:
- **«Неверное имя модели»** — проверьте точное написание, включая теги
- **«Неверный базовый URL»** — убедитесь, что конечная точка поддерживает `/v1/embeddings`
- **«Отсутствует API-ключ»** — некоторые провайдеры требуют аутентификацию
- **«Сетевая ошибка»** — проверьте подключение

| **Лучшее качество (английский)** | Voyage AI `voyage-3` |
