# リモートシェル（Conch）

Agora は [Conch](https://github.com/newo-ether/conch) プロトコルを通じてリモートマシン上でコマンドを実行できます — AI エージェント向けに設計されたエンドツーエンド暗号化のセキュアシェルです。

## 仕組み

```text
Agora (Android)  ──ECDH + AES-256-GCM──▶  Conch サーバー (Linux/macOS/Windows)
                                           │
                                           ├── コマンド実行
                                           ├── ファイルの読み取り/書き込み/編集
                                           ├── Glob および Grep 検索
                                           └── 結果を返す
```

モデルがシェルを使用するタイミングを決定します — サーバーステータスの確認、ファイル管理、スクリプト実行、問題のトラブルシューティングを自律的に行えます。

## セキュリティ

Conch は強力な暗号化と悪用防止策を使用しています：

- **ECDH 鍵交換** — セッションごとにエフェメラル鍵
- **AES-256-GCM 暗号化** — すべてのトラフィックが暗号化
- **HMAC-SHA256 署名** — メッセージの整合性を検証
- **トークンバケットレート制限** — 悪用を防止
- **ノンスベースのアンチリプレイ** — 各リクエストが一意

!!! note
    コマンドは Conch サーバーを実行しているユーザーの権限で実行されます。機密環境では制限付きユーザーアカウントを使用してください。

## セットアップ

### ステップ 1: Conch サーバーのデプロイ

ターゲットマシンに Conch サーバーをデプロイします。セットアップ手順については [Conch リポジトリ](https://github.com/newo-ether/conch) を参照してください。

### ステップ 2: Agora にデバイスを追加

1. **設定 → シェル** に移動します
2. **シェルツール** を有効にします
3. **デバイスを追加** をタップします
4. デバイスタイプを選択します：**Conch** または **SSH**
5. デバイスの詳細を入力します：

=== "Conch"

    | 項目 | 説明 | 例 |
    |-------|-------------|---------|
    | **名前** | このデバイスの表示名 | `ビルドサーバー` |
    | **説明** | このマシンに関する任意のメモ | `オフィスの Ubuntu マシン` |
    | **サーバー URL** | Conch サーバーのエンドポイント（host:port） | `http://192.168.1.100:14216` |
    | **API キー** | 認証トークン | Conch サーバー設定から |
    | **タイムアウト** | コマンドタイムアウト（秒） | `30` |

=== "SSH"

    | 項目 | 説明 | 例 |
    |-------|-------------|---------|
    | **名前** | このデバイスの表示名 | `VPS サーバー` |
    | **説明** | このマシンに関する任意のメモ | `本番ウェブサーバー` |
    | **ホスト** | SSH ホスト名または IP アドレス | `192.168.1.200` |
    | **ポート** | SSH ポート | `22` |
    | **ユーザー** | SSH ユーザー名 | `root` |
    | **パスワード** | SSH パスワード | SSH パスワード |

**追加** をタップして保存します。

### ステップ 3: 使用

設定が完了すると、モデルはデバイスにアクセスできます。手動トリガーはありません — モデルは利用可能なシェルデバイスを自動検出し、適切なタイミングで呼び出します。

## マルチデバイス対応

複数のシェルデバイスを追加して、モデルに複数のマシンでの作業を許可します：

- **ビルドサーバー** — コードのコンパイルとテスト
- **ホームラボ** — セルフホストサービスの管理
- **開発 VM** — コードの編集とスクリプトの実行

各デバイスは独自の名前、URL、認証情報で個別に設定されます。モデルはそれらを区別し、各タスクに適切なデバイスを選択できます。

## 利用可能な操作

### コマンド実行（`shell_execute`）

任意のシェルコマンドを実行し、stdout、stderr、終了コードを受け取ります。

### ファイル操作

| ツール | 機能 |
|------|----------|
| `file_read` | リモートファイルシステムからファイルを読み取り |
| `file_write` | ファイルを書き込みまたは上書き |
| `file_edit` | ファイル内の正確な文字列置換を実行 |
| `file_glob` | Glob パターンに一致するファイルを検索 |
| `file_grep` | 正規表現でファイル内容を検索 |

すべてのファイル操作は暗号化された Conch チャネルを通じて行われます。

## MCP 連携

Conch は **Claude Desktop MCP サーバー** としても機能します。Claude Code や他の MCP クライアントを使用している場合、デスクトップからのリモートファイルおよびシェルアクセス用のツールプロバイダーとして Conch を設定できます。

MCP セットアップ手順については [Conch ドキュメント](https://github.com/newo-ether/conch) を参照してください。

## トラブルシューティング

### デバイスが利用不可と表示される
- Conch サーバーが実行中であることを確認
- Android デバイスから URL に到達可能であることを確認
- サーバーのファイアウォールルールを確認

### コマンドがタイムアウトする
- デバイス設定でタイムアウトを延長
- コマンドがハングしていないか確認（ユーザー入力の要求など）

### 認証に失敗する
- API キーがサーバー設定と一致することを確認
- 必要に応じてキーを再生成
