# الطرفية البعيدة (Conch)

يمكن لـ Agora تنفيذ أوامر على أجهزة بعيدة عبر بروتوكول [Conch](https://github.com/newo-ether/conch) — طرفية آمنة مشفرة من الطرف إلى الطرف مصممة لوكلاء الذكاء الاصطناعي.

## كيف يعمل

```text
Agora (Android)  ──ECDH + AES-256-GCM──▶  خادم Conch (Linux/macOS/Windows)
                                           │
                                           ├── تنفيذ الأوامر
                                           ├── قراءة/كتابة/تعديل الملفات
                                           ├── بحث glob و grep
                                           └── إرجاع النتائج
```

النموذج يقرر متى يستخدم الطرفية — يمكنه التحقق من حالة الخادم، إدارة الملفات، تشغيل السكربتات، أو استكشاف المشاكل ذاتيًا.

## الأمان

Conch يستخدم تشفيرًا قويًا وحمايات ضد الإساءة:

- **تبادل مفاتيح ECDH** — مفاتيح مؤقتة لكل جلسة
- **تشفير AES-256-GCM** — كل حركة المرور مشفرة
- **توقيع HMAC-SHA256** — سلامة الرسالة متحققة
- **تحديد معدل دلو الرموز** — يمنع الإساءة
- **مكافحة الإعادة بناءً على nonce** — كل طلب فريد

!!! note
    الأوامر تنفذ بصلاحيات المستخدم المشغل لخادم Conch. استخدم حساب مستخدم مقيد للبيئات الحساسة.

## الإعداد

### الخطوة 1: نشر خادم Conch

انشر خادم Conch على جهازك المستهدف. راجع [مستودع Conch](https://github.com/newo-ether/conch) لتعليمات الإعداد.

### الخطوة 2: إضافة جهاز في Agora

1. اذهب إلى **الإعدادات → الطرفية**
2. فعّل **أداة الطرفية**
3. اضغط **إضافة جهاز**
4. اختر نوع الجهاز: **Conch** أو **SSH**
5. املأ تفاصيل الجهاز:

=== "Conch"

    | الحقل | الوصف | مثال |
    |-------|-------------|---------|
    | **الاسم** | اسم عرض لهذا الجهاز | `خادم البناء` |
    | **الوصف** | ملاحظة اختيارية عن هذا الجهاز | `جهاز Ubuntu المكتبي` |
    | **URL الخادم** | نقطة نهاية خادم Conch (host:port) | `http://192.168.1.100:14216` |
    | **مفتاح API** | رمز التوثيق | من إعدادات خادم Conch |
    | **المهلة** | مهلة الأمر بالثواني | `30` |

=== "SSH"

    | الحقل | الوصف | مثال |
    |-------|-------------|---------|
    | **الاسم** | اسم عرض لهذا الجهاز | `خادم VPS` |
    | **الوصف** | ملاحظة اختيارية عن هذا الجهاز | `خادم ويب إنتاجي` |
    | **المضيف** | اسم مضيف SSH أو عنوان IP | `192.168.1.200` |
    | **المنفذ** | منفذ SSH | `22` |
    | **المستخدم** | اسم مستخدم SSH | `root` |
    | **كلمة المرور** | كلمة مرور SSH | كلمة مرور SSH الخاصة بك |

اضغط **إضافة** للحفظ.

### الخطوة 3: الاستخدام

بمجرد الإعداد، يمكن للنموذج الوصول إلى الجهاز. لا يوجد تشغيل يدوي — النموذج يكتشف تلقائيًا أجهزة الطرفية المتاحة ويستدعيها عندما يكون مناسبًا.

---

## دعم الأجهزة المتعددة

أضف أجهزة طرفية متعددة لتتيح للنموذج العمل عبر الأجهزة:

- **خادم البناء** — ترجمة واختبار الكود
- **مختبر منزلي** — إدارة الخدمات المستضافة ذاتيًا
- **VM تطوير** — تعديل الكود وتشغيل السكربتات

كل جهاز معد بشكل مستقل باسمه و URL وبيانات اعتماده الخاصة. النموذج يمكنه التمييز بينها واختيار الجهاز الصحيح لكل مهمة.

---

## العمليات المتاحة

### تنفيذ الأوامر (`shell_execute`)

تشغيل أي أمر طرفية واستقبال stdout، stderr، ورمز الخروج.

### عمليات الملفات

| الأداة | الوظيفة |
|------|----------|
| `file_read` | قراءة ملف من نظام الملفات البعيد |
| `file_write` | كتابة أو استبدال ملف |
| `file_edit` | تنفيذ استبدالات نصية دقيقة في ملف |
| `file_glob` | إيجاد ملفات تطابق نمط glob |
| `file_grep` | البحث في محتويات الملفات بـ regex |

كل عمليات الملفات تمر عبر قناة Conch المشفرة.

## تكامل MCP

يمكن لـ Conch أيضًا العمل **كخادم MCP لـ Claude Desktop**. إذا كنت تستخدم Claude Code أو عميل MCP آخر، يمكنك إعداد Conch كمزود أدوات للوصول إلى الملفات والطرفية عن بُعد من سطح مكتبك.

راجع [توثيق Conch](https://github.com/newo-ether/conch) لتعليمات إعداد MCP.

## استكشاف المشاكل

### الجهاز يظهر كغير متاح
- تحقق أن خادم Conch قيد التشغيل
- تحقق أن URL قابل للوصول من جهاز Android الخاص بك
- تحقق من قواعد جدار الحماية على الخادم

### الأوامر تنتهي مهلتها
- زد المهلة في إعدادات الجهاز
- تحقق أن الأمر ليس معلقًا (يتطلب إدخال مستخدم، إلخ.)

### فشل التوثيق
- تحقق أن مفتاح API يطابق إعدادات الخادم
- أعد توليد المفاتيح إذا لزم الأمر
