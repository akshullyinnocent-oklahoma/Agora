# 遠端 Shell (Conch)

Agora 可以透過 [Conch](https://github.com/newo-ether/conch) 協定在遠端機器上執行指令 — 一個專為 AI 代理設計的端對端加密安全 Shell。

## 運作方式

```text
Agora (Android)  ──ECDH + AES-256-GCM──▶  Conch 伺服器 (Linux/macOS/Windows)
                                           │
                                           ├── 執行指令
                                           ├── 讀取/寫入/編輯檔案
                                           ├── Glob 與 grep 搜尋
                                           └── 回傳結果
```

模型會自行決定何時使用 Shell — 它可以自主檢查伺服器狀態、管理檔案、執行腳本或排除問題。

## 安全性

Conch 使用強加密與防濫用保護：

- **ECDH 金鑰交換** — 每個工作階段使用臨時金鑰
- **AES-256-GCM 加密** — 所有流量皆加密
- **HMAC-SHA256 簽章** — 驗證訊息完整性
- **Token bucket 速率限制** — 防止濫用
- **基於 Nonce 的抗重送攻擊** — 每個請求都是唯一的

!!! note
    指令會以執行 Conch 伺服器的使用者權限執行。對於敏感環境，請使用受限的使用者帳號。

## 設定

### 步驟 1：部署 Conch 伺服器

在您的目標機器上部署 Conch 伺服器。請參閱 [Conch 儲存庫](https://github.com/newo-ether/conch) 了解設定說明。

### 步驟 2：在 Agora 中新增裝置

1. 前往 **設定 → Shell**
2. 啟用 **Shell 工具**
3. 點選 **新增裝置**
4. 選擇裝置類型：**Conch** 或 **SSH**
5. 填寫裝置詳細資料：

=== "Conch"

    | 欄位 | 說明 | 範例 |
    |-------|-------------|---------|
    | **名稱** | 此裝置的顯示名稱 | `Build Server` |
    | **描述** | 關於此機器的選填備註 | `Office Ubuntu box` |
    | **伺服器 URL** | Conch 伺服器端點 (host:port) | `http://192.168.1.100:14216` |
    | **API 金鑰** | 驗證權杖 | 來自 Conch 伺服器設定 |
    | **逾時** | 指令逾時秒數 | `30` |

=== "SSH"

    | 欄位 | 說明 | 範例 |
    |-------|-------------|---------|
    | **名稱** | 此裝置的顯示名稱 | `VPS Server` |
    | **描述** | 關於此機器的選填備註 | `Production web server` |
    | **主機** | SSH 主機名稱或 IP 位址 | `192.168.1.200` |
    | **連接埠** | SSH 連接埠 | `22` |
    | **使用者** | SSH 使用者名稱 | `root` |
    | **密碼** | SSH 密碼 | 您的 SSH 密碼 |

點選 **新增** 以儲存。

### 步驟 3：使用

設定完成後，模型即可存取該裝置。無需手動觸發 — 模型會自動探索可用的 Shell 裝置，並在適當時機呼叫它們。

## 多裝置支援

新增多個 Shell 裝置，讓模型可以跨機器工作：

- **建置伺服器** — 編譯與測試程式碼
- **家庭實驗室** — 管理自託管服務
- **開發虛擬機** — 編輯程式碼與執行腳本

每個裝置皆獨立設定，擁有各自的名稱、URL 與憑證。模型可以區分它們，並為每個任務選擇正確的裝置。

## 可用操作

### 指令執行 (`shell_execute`)

執行任何 Shell 指令，並接收 stdout、stderr 與退出碼。

### 檔案操作

| 工具 | 功能 |
|------|----------|
| `file_read` | 從遠端檔案系統讀取檔案 |
| `file_write` | 寫入或覆寫檔案 |
| `file_edit` | 在檔案中執行精確的字串替換 |
| `file_glob` | 尋找符合 glob 模式的檔案 |
| `file_grep` | 使用正規表示式搜尋檔案內容 |

所有檔案操作皆透過加密的 Conch 通道進行。

## MCP 整合

Conch 也可以作為 **Claude Desktop MCP 伺服器**。如果您使用 Claude Code 或其他 MCP 客戶端，您可以將 Conch 設定為工具提供者，以便從您的桌面進行遠端檔案與 Shell 存取。

請參閱 [Conch 文件](https://github.com/newo-ether/conch) 了解 MCP 設定說明。

## 疑難排解

### 裝置顯示為不可用
- 檢查 Conch 伺服器是否正在執行
- 確認從您的 Android 裝置可以連線到該 URL
- 檢查伺服器上的防火牆規則

### 指令逾時
- 在裝置設定中增加逾時時間
- 檢查指令是否卡住（需要使用者輸入等）

### 驗證失敗
- 確認 API 金鑰與伺服器設定相符
- 如有需要，重新產生金鑰
