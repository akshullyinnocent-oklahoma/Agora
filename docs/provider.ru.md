# API Провайдеры

Agora подключается напрямую к AI-провайдерам — без посредников, без подписки, без телеметрии. Вы предоставляете свои API-ключи, и всё работает с вашего устройства.

## Встроенные провайдеры

| Провайдер | Базовый URL | Модели | Примечания |
|----------|----------|--------|-------|
| **Google** | `https://generativelanguage.googleapis.com/v1beta` | Серия Gemini | Доступен бесплатный тариф через Google AI Studio |
| **OpenAI** | `https://api.openai.com/v1` | GPT-4, GPT-4o, o-серия | Поддерживаются модели рассуждения |
| **Anthropic** | `https://api.anthropic.com/v1` | Серия Claude | Поддерживается расширенное мышление |
| **DeepSeek** | `https://api.deepseek.com/v1` | DeepSeek-V3, DeepSeek-R1 | Поддерживаются модели рассуждения |
| **Qwen** | `https://dashscope-intl.aliyuncs.com/compatible-mode/v1` | Серия Qwen | Через Alibaba DashScope |
| **Ollama** | `http://localhost:11434/v1` | Любая загруженная модель | Самостоятельный хостинг, API-ключ не требуется |
| **OpenRouter** | `https://openrouter.ai/api/v1` | Мульти-провайдер | Доступ ко многим моделям через один API |
| **Локальный** | Н/Д | GGUF модели | На устройстве через llama.cpp, полностью автономно |

## Переключение провайдеров

Нажмите переключатель провайдера в Настройках для переключения между провайдерами. Каждый провайдер хранит отдельно:

- API-ключи
- Базовый URL (редактируемый для прокси/собственного хостинга)
- Список моделей

## API-ключи

### Несколько ключей на провайдера

Каждый провайдер поддерживает несколько именованных API-ключей. Это позволяет:

- **Ротация** — переключение между ключами для разных тарифов использования
- **Организация** — разделение рабочего и личного использования
- **Резерв** — наличие запасного ключа наготове

### Управление ключами

1. Перейдите в **Настройки → Провайдер**
2. Выберите провайдера
3. В разделе **API-ключи** нажмите **Добавить новый ключ**
4. Введите **имя** (например, «Рабочий», «Личный», «Общий командный») и **значение ключа**
5. Нажмите **Добавить**

Нажмите радиокнопку, чтобы установить активный ключ. Долгое нажатие на ключ — **Редактировать** или **Удалить**.

### Безопасность ключей

!!! warning
    API-ключи хранятся локально в зашифрованной базе данных Room. Они никогда не отправляются на серверы Agora (их нет). Однако они экспортируются в открытом виде, если вы включите их в файл экспорта `.agora`.

## Пользовательские провайдеры

Добавьте любую конечную точку API, совместимую с OpenAI:

1. Перейдите в **Настройки → Провайдер**
2. Нажмите **+ Добавить пользовательского провайдера** внизу списка провайдеров
3. Введите:
    - **Имя провайдера** — любое отображаемое имя
    - **Базовый URL** — конечная точка API
4. Нажмите **Добавить**

Agora загружает список моделей из `{base_url}/v1/models`. После добавления пользовательские провайдеры работают точно так же, как встроенные: добавляйте API-ключи, синхронизируйте модели и общайтесь.

### Варианты использования

- **Собственный хостинг** — подключение к vLLM, LocalAI, text-generation-webui или другим серверам, совместимым с OpenAI
- **Прокси** — маршрутизация через корпоративный прокси или API-шлюз
- **Альтернативные конечные точки** — использование Azure OpenAI, Cloudflare AI Gateway или других совместимых сервисов

### Переименование или удаление

Долгое нажатие на пользовательского провайдера — **Переименовать** или **Удалить**. Удаление удаляет провайдера и все его ключи.

!!! warning
    Встроенные провайдеры не могут быть переименованы или удалены.

## Переопределение базового URL

Каждый провайдер (включая встроенные) имеет редактируемый **Базовый URL**. Это полезно для:

- **Прокси**: Маршрутизация через `https://my-proxy.example.com/v1`
- **Собственный хостинг**: Указание на собственный экземпляр
- **Региональная маршрутизация**: Использование конечных точек для конкретных регионов

---

## Синхронизация моделей

После добавления API-ключей синхронизируйте список моделей:

1. Перейдите в **Настройки → Модели**
2. Нажмите **Синхронизировать от всех провайдеров**
3. Agora загружает доступные модели от каждого настроенного провайдера

Snackbar показывает прогресс и результаты синхронизации. Затем вы можете включить/отключить отдельные модели и установить модель по умолчанию.

## Заметки по конкретным провайдерам

### Google Gemini

- API-ключи из [Google AI Studio](https://aistudio.google.com/apikey)
- Доступен бесплатный тариф с ограничениями по частоте
- Поддерживает выполнение кода и поисковую привязку (встроенные инструменты)

### OpenAI

- API-ключи из [Platform](https://platform.openai.com/api-keys)
- Модели рассуждения (o1, o3) требуют специального доступа к API
- Поддерживаются потоковая передача, инструменты и зрение

### Anthropic

- API-ключи из [Console](https://console.anthropic.com/)
- Расширенное мышление с настраиваемыми бюджетами токенов
- Поддерживается использование инструментов с параллельными вызовами

### Ollama

- API-ключ не требуется (локальная сеть)
- Базовый URL обычно `http://<хост>:11434/v1`
- Список моделей загружается из API Ollama
- См. [FAQ](faq.ru.md) по устранению неполадок, специфичных для Ollama

### OpenRouter

- Один API-ключ для 200+ моделей
- Цена за токен варьируется в зависимости от модели
- Хорошо подходит для опробования разных моделей без отдельных аккаунтов у провайдеров

### Локальный (llama.cpp)

- Сеть не требуется
- Файлы GGUF моделей хранятся на устройстве
- См. [Локальные модели](local-model.ru.md) для настройки
