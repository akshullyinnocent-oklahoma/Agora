# 원격 셸 (Conch)

Agora는 [Conch](https://github.com/newo-ether/conch) 프로토콜을 통해 원격 머신에서 명령을 실행할 수 있습니다 — AI 에이전트를 위해 설계된 종단 간 암호화 보안 셸입니다.

## 작동 방식

```text
Agora (Android)  ──ECDH + AES-256-GCM──▶  Conch 서버 (Linux/macOS/Windows)
                                           │
                                           ├── 명령 실행
                                           ├── 파일 읽기/쓰기/편집
                                           ├── Glob 및 grep 검색
                                           └── 결과 반환
```

모델이 셸을 사용할 시기를 결정합니다 — 서버 상태 확인, 파일 관리, 스크립트 실행, 문제 해결을 자율적으로 수행할 수 있습니다.

## 보안

Conch는 강력한 암호화와 남용 방지 보호를 사용합니다:

- **ECDH 키 교환** — 세션별 임시 키
- **AES-256-GCM 암호화** — 모든 트래픽 암호화
- **HMAC-SHA256 서명** — 메시지 무결성 검증
- **토큰 버킷 속도 제한** — 남용 방지
- **Nonce 기반 재전송 방지** — 각 요청은 고유함

!!! note
    명령은 Conch 서버를 실행하는 사용자의 권한으로 실행됩니다. 민감한 환경에서는 제한된 사용자 계정을 사용하세요.

## 설정

### 1단계: Conch 서버 배포

대상 머신에 Conch 서버를 배포합니다. 설정 지침은 [Conch 저장소](https://github.com/newo-ether/conch)를 참조하세요.

### 2단계: Agora에 기기 추가

1. **설정 → 셸**로 이동
2. **셸 도구** 활성화
3. **기기 추가** 탭
4. 기기 유형 선택: **Conch** 또는 **SSH**
5. 기기 세부 정보 입력:

=== "Conch"

    | 필드 | 설명 | 예시 |
    |-------|-------------|---------|
    | **이름** | 이 기기의 표시 이름 | `빌드 서버` |
    | **설명** | 이 머신에 대한 선택적 메모 | `사무실 Ubuntu 박스` |
    | **서버 URL** | Conch 서버 엔드포인트 (host:port) | `http://192.168.1.100:14216` |
    | **API 키** | 인증 토큰 | Conch 서버 설정에서 |
    | **타임아웃** | 명령 타임아웃(초) | `30` |

=== "SSH"

    | 필드 | 설명 | 예시 |
    |-------|-------------|---------|
    | **이름** | 이 기기의 표시 이름 | `VPS 서버` |
    | **설명** | 이 머신에 대한 선택적 메모 | `프로덕션 웹 서버` |
    | **호스트** | SSH 호스트명 또는 IP 주소 | `192.168.1.200` |
    | **포트** | SSH 포트 | `22` |
    | **사용자** | SSH 사용자명 | `root` |
    | **비밀번호** | SSH 비밀번호 | SSH 비밀번호 |

**추가**를 탭하여 저장.

### 3단계: 사용

구성 후, 모델이 기기에 접근할 수 있습니다. 수동 트리거는 없습니다 — 모델이 사용 가능한 셸 기기를 자동으로 발견하고 적절할 때 호출합니다.

---

## 다중 기기 지원

여러 셸 기기를 추가하여 모델이 여러 머신에서 작업할 수 있도록 합니다:

- **빌드 서버** — 코드 컴파일 및 테스트
- **홈 랩** — 자체 호스팅 서비스 관리
- **개발 VM** — 코드 편집 및 스크립트 실행

각 기기는 고유한 이름, URL, 자격 증명으로 독립적으로 구성됩니다. 모델은 이를 구분하고 각 작업에 적합한 기기를 선택할 수 있습니다.

---

## 사용 가능한 작업

### 명령 실행 (`shell_execute`)

모든 셸 명령을 실행하고 stdout, stderr, 종료 코드를 수신합니다.

### 파일 작업

| 도구 | 기능 |
|------|----------|
| `file_read` | 원격 파일 시스템에서 파일 읽기 |
| `file_write` | 파일 쓰기 또는 덮어쓰기 |
| `file_edit` | 파일에서 정확한 문자열 교체 수행 |
| `file_glob` | Glob 패턴과 일치하는 파일 찾기 |
| `file_grep` | 정규식으로 파일 내용 검색 |

모든 파일 작업은 암호화된 Conch 채널을 통해 이루어집니다.

## MCP 통합

Conch는 **Claude Desktop MCP 서버**로도 작동할 수 있습니다. Claude Code 또는 다른 MCP 클라이언트를 사용하는 경우, 데스크톱에서 원격 파일 및 셸 접근을 위한 도구 제공자로 Conch를 구성할 수 있습니다.

MCP 설정 지침은 [Conch 문서](https://github.com/newo-ether/conch)를 참조하세요.

## 문제 해결

### 기기가 사용 불가로 표시됨
- Conch 서버가 실행 중인지 확인
- Android 기기에서 URL에 접근 가능한지 확인
- 서버의 방화벽 규칙 확인

### 명령 타임아웃
- 기기 설정에서 타임아웃 증가
- 명령이 멈춰 있지 않은지 확인 (사용자 입력 요구 등)

### 인증 실패
- API 키가 서버 설정과 일치하는지 확인
- 필요시 키 재생성
