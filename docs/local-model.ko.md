# 로컬 모델

GGUF 모델 파일과 llama.cpp를 사용하여 Android 기기에서 직접 LLM을 실행하세요. 네트워크 불필요, API 키 불필요, 완전한 개인정보 보호.

## 작동 방식

Agora는 Android NDK(CMake)를 통해 llama.cpp를 번들로 제공합니다. GGUF 파일을 가져오면 모델이 기기의 CPU에서 완전히 실행됩니다 — 데이터가 기기를 벗어나지 않습니다.

## 요구 사항

- **GGUF 형식** 모델만 지원 (llama.cpp 표준)
- **기기 메모리**: 모델이 사용 가능한 RAM에 맞아야 합니다. 경험상:
    - 1–3B 매개변수 모델: 4–6 GB RAM
    - 7–8B 매개변수 모델: 6–8 GB RAM
- **저장소**: GGUF 파일은 약 500 MB(양자화된 소형 모델)에서 5+ GB 범위

!!! warning
    로컬 추론은 CPU 집약적이며 클라우드 API보다 훨씬 느립니다. 빠른 대량 채팅보다는 오프라인 사용, 개인정보 민감 콘텐츠, 실험에 적합합니다.

---

## 채팅 모델 가져오기

1. GGUF 모델 파일을 기기에 다운로드 (아래 권장 소스 참조)
2. **설정 → 프로바이더**로 이동
3. 프로바이더로 **로컬** 선택
4. **GGUF 모델 가져오기** 탭
5. 기기에서 `.gguf` 파일 선택
6. 모델 구성:

| 매개변수 | 설명 | 예시 |
|-----------|-------------|---------|
| **모델 ID** | 소문자 식별자, 공백 없음 | `qwen3-8b` |
| **별칭** | 표시 이름 | `Qwen 3 8B` |
| **컨텍스트 크기** | 토큰 단위의 최대 컨텍스트 윈도우 | `4096` |
| **Temperature** | 무작위성 (0.0–2.0) | `0.7` |
| **Top P** | 핵 샘플링 임계값 (0.0–1.0) | `0.9` |
| **최대 토큰** | 최대 생성 길이 | `2048` |

7. **추가** 탭

모델이 가져와지고 즉시 사용할 준비가 됩니다.

---

## 임베딩 모델 가져오기

임베딩 모델은 더 작고 시맨틱 검색에 사용됩니다:

1. **설정 → 대화 검색**으로 이동
2. **로컬 모델 추가** 탭
3. `.gguf` 임베딩 모델 파일 선택
4. 이름 지정
5. **추가** 탭

검색 설정은 [임베딩 / RAG](embedding.ko.md)를 참조하세요.

---

## 활성 모델 선택

하나 이상의 모델을 가져온 후:

1. **설정 → 프로바이더 → 로컬**로 이동
2. 가져온 모든 모델이 나열됨
3. 사용하려는 모델 옆의 **라디오 버튼** 탭
4. 채팅 프로바이더로 **로컬**이 선택되면 선택한 모델이 활성화됨

---

## 로컬 모델 관리

### 이름 변경

모델을 탭하여 별칭 변경 또는 매개변수 조정 (temperature, 컨텍스트 크기 등).

### 삭제

모델을 길게 누르고 **삭제** 탭. Agora에서 모델이 제거되고 저장소에서 GGUF 파일이 삭제됩니다.

## 권장 모델

### 채팅 모델

| 모델 | 크기 | 필요 RAM | 참고 |
|-------|------|------------|-------|
| Qwen 3 1.7B | ~1 GB | 3–4 GB | 크기 대비 좋은 품질 |
| Llama 3.2 3B | ~2 GB | 4–5 GB | 전반적으로 견고 |
| Qwen 3 8B | ~5 GB | 7–8 GB | 최고 품질, 높은 RAM |

### 임베딩 모델

| 모델 | 크기 | 참고 |
|-------|------|-------|
| BGE Small EN v1.5 | ~130 MB | 좋은 영어 임베딩 |
| BGE Small ZH v1.5 | ~130 MB | 중국어 최적화 |
| Nomic Embed Text v1.5 | ~270 MB | 좋은 다국어 지원 |

### GGUF 파일 다운로드 위치

- [Hugging Face](https://huggingface.co/models?library=gguf) — "GGUF" 검색
- [bartowski의 양자화 모델](https://huggingface.co/bartowski) — 다양한 선택, 체계적 구성

!!! tip
    `Q4_K_M` 양자화를 찾으세요 — 채팅 모델용으로 품질과 크기 간 최상의 균형을 제공합니다.

---

## 성능 팁

- **더 작은 컨텍스트 = 더 빠름**: 2048로 시작하여 필요할 때만 증가
- **더 낮은 양자화 = 더 빠름**: Q4_K_M이 Q6_K나 Q8보다 빠름
- **다른 앱 닫기**: 로컬 추론은 최대한 많은 RAM 필요
- **전원 연결**: 추론은 CPU 집약적이며 장시간 사용 시 배터리 소모
