# 代理工具

Agora 的模型可以自主使用工具 — 它們會自行決定要搜尋、執行、讀取或記住什麼，無需您手動觸發每個動作。工具以**多輪迴圈**方式運作：模型可以呼叫工具、讀取結果，然後決定呼叫另一個工具或作出回應。

## 工具呼叫的運作方式

1. 您發送一則訊息
2. 模型判斷它需要外部資訊或動作
3. 它發出一個**工具呼叫** — 一個包含工具名稱與參數的結構化請求
4. Agora 在裝置上或遠端伺服器上執行該工具
5. 結果被送回給模型
6. 模型可以呼叫另一個工具或產生最終回應

此迴圈可以在單一訊息回合中重複多次。

## 可用工具

### 網頁搜尋

搜尋網際網路並擷取網頁。模型可以查詢最新資訊、驗證事實或擷取文件。

- **提供者**：Brave、Serper、Tavily、SearXNG
- **設定**：設定 → 網頁搜尋
- **指南**：[網頁搜尋](web-search.md)

### 程式碼執行

在沙盒環境中執行程式碼片段（取決於提供者）。適用於計算、資料分析或內聯測試邏輯。

- **提供者**：Google Gemini（內建程式碼執行）
- **設定**：無需設定 — Gemini 模型會自動啟用

### 遠端 Shell

透過 [Conch](https://github.com/newo-ether/conch) 協定在遠端機器上執行指令。模型可以檢查伺服器狀態、管理檔案或執行腳本。

- **協定**：端對端加密（ECDH + AES-256-GCM）
- **設定**：設定 → Shell
- **指南**：[遠端 Shell](shell.md)

### 檔案操作

透過 Conch 協定在遠端裝置上讀取、寫入、編輯、glob 搜尋及 grep 搜尋檔案。模型可以直接操作遠端檔案系統。

!!! note
    檔案操作需要已設定的 Conch Shell 裝置。請參閱[遠端 Shell](shell.md) 了解設定方式。

### 記憶

跨對話持續存在的知識儲存：

- **活躍記憶** — 始終包含在每個 API 呼叫中。適用於模型應始終記住的事實、偏好或上下文。
- **已儲存記憶** — 一系列具名記憶檔案，模型可以透過工具呼叫來搜尋、讀取、寫入和編輯。

詳情請參閱[記憶與快取](memory.md)。

### 對話搜尋

模型可以使用關鍵字或語義（RAG）方法搜尋您過去的對話記錄。這讓它能夠參考先前的討論，而無需您手動尋找並分享它們。

請參閱[對話搜尋](search.md) 了解設定方式。

## 聊天中的工具 UI

當工具被呼叫時，您會在對話中看到內聯顯示：

<div class="grid cards" markdown>

- **:material-progress-wrench: 工具呼叫橫幅**

    ---

    顯示工具名稱與簡短狀態（例如 :material-magnify: "正在網路上搜尋 'latest AI news'"）。

- **:material-check-circle: 工具結果**

    ---

    執行後，顯示格式化的結果或摘要（例如 "找到 5 個關於 'latest AI news' 的結果"）。

</div>

### 可展開的詳細資料

點選工具呼叫以展開並查看：

- **參數** — 傳送給工具的確切參數
- **結果** — 工具執行的原始輸出
- **狀態** — 成功、失敗或部分結果

### 失敗的呼叫

如果工具呼叫失敗，模型會收到錯誤通知，並可以重試或調整。您會看到一個帶有錯誤訊息的紅色橫幅。

## 工具權限

您可以控制模型可以存取哪些工具：

| 設定 | 位置 | 預設值 |
|---------|----------|---------|
| 網頁搜尋 | 設定 → 網頁搜尋 | 關閉 |
| Shell | 設定 → Shell | 關閉 |
| 記憶（已儲存） | 設定 → 記憶 → 存取已儲存記憶 | 關閉 |
| 記憶（活躍） | 設定 → 記憶 → 存取活躍記憶 | 關閉 |
| 過往對話 | 設定 → 記憶 → 存取過往對話 | 關閉 |
| 對話搜尋 | 設定 → 對話搜尋 | 開啟* |

*模型搜尋對話的能力取決於是否已設定 embedding 模型。若無設定，則僅能使用關鍵字搜尋。

## 多輪工具迴圈

模型可以串連多個工具呼叫。例如：

1. 使用者："最新的 Linux 核心版本是什麼？我的伺服器正在執行它嗎？"
2. 模型呼叫 `web_search("latest Linux kernel version")`
3. 模型呼叫 `shell_execute("uname -r", device="my-server")`
4. 模型比較結果並回應

每個工具呼叫及其結果會在最終文字回應之前，作為單獨的內聯項目顯示在對話中。
