# 会話検索

Agora は会話履歴全体を検索できます — キーワードマッチングまたは埋め込みモデルを使用したセマンティック（意味ベース）検索によって。

## 検索方法

### キーワード検索

高速で正確なテキストマッチング。メッセージ内容にクエリの文字通りの出現を検索します。

**適している場合：**
- 特定のフレーズや用語を見つける
- 正確な文言を覚えている場合の素早い検索
- セットアップ不要 — すぐに使用可能

**制限：**
- 同義語や関連概念を見逃す
- 意味の理解がない

### セマンティック検索（RAG）

埋め込みモデルを使用して、正確な単語ではなく **意味** でメッセージを見つけます。「データベースのセットアップ方法」というクエリで、「データベース」という単語が一度も出てこなくても「Room 設定」に関するメッセージを見つけることができます。

**適している場合：**
- トピックやテーマで会話を見つける
- 正確な文言を覚えていない広範なクエリ
- 異なる会話にまたがる関連ディスカッションの発見

**要件：**
- 埋め込みモデルが設定されていること（[埋め込み / RAG](embedding.md) を参照）
- メッセージがキャッシュされていること（埋め込みが生成済み）

## セットアップ

### 1. 埋め込みモデルを追加

詳細なセットアップについては [埋め込み / RAG](embedding.md) を参照してください。以下を使用できます：
- **リモートモデル**（OpenAI、Mistral、Voyage、Ollama など）
- **ローカルモデル**（GGUF ファイル、完全オフライン）

### 2. 検索方法を選択

**設定 → 会話検索** で：

| 設定 | 説明 |
|---------|-------------|
| **モデル検索方法** | モデルが `search_conversations` ツールを呼び出すときの検索方法 |
| **手動検索方法** | 会話ドロワーの検索バーの動作方法 |

それぞれを **キーワード** または **セマンティック（RAG）** に設定します。

### 3. 検索範囲を設定

| 設定 | 範囲 | 説明 |
|---------|-------|-------------|
| **検索ヒットあたりのコンテキストメッセージ数** | 4～32 | 各一致に含める周辺メッセージの数（ステップ: 4、8、12、16、20、24、28、32） |
| **最大検索結果数** | 5～30 | 返す最大一致数（ステップ: 5、10、15、20、25、30） |
| **類似度閾値** | 0.0～1.0 | RAG のみ：一致の最小類似度スコア。高いほど厳密。デフォルト：0.5 |

### 4. メッセージをキャッシュ

RAG を使用する場合は、**キャッシュ** をタップしてすべての既存メッセージの埋め込みを生成します。**自動キャッシュ** を有効にしてインデックスを自動的に最新に保ちます。

## 検索の使用

### 手動検索（検索バー）

1. **会話ドロワー** を開きます（ハンバーガーメニュー :material-menu: または右にスワイプ）
2. 上部の検索バーをタップします
3. クエリを入力します
4. 結果が下に表示されます — 結果をタップすると、一致するメッセージの位置でその会話が開きます

### モデルによる検索

**過去の会話へのアクセス** が有効な場合（設定 → メモリ）、モデルは自律的に履歴を検索できます：

```text
あなた: 「先週 API 設計について何を決めましたか？」
モデル: [「API 設計 決定」を検索]
       「先週の火曜日に、...を使用することに決めました」
```

検索は会話内にツールカードとして表示されます。

## 類似度閾値

**類似度閾値** スライダー（0.0 ～ 1.0）は、メッセージが RAG 結果に含まれるためにどれだけ密接に一致する必要があるかを制御します：

- **低（0.3～0.5）**: より多くの結果、緩く関連するコンテンツを含む可能性あり
- **中（0.5～0.7）**: バランス — 適切なデフォルト
- **高（0.7～0.9）**: より少ない結果、非常に近い一致のみ

デフォルトから始めて、結果に基づいて調整します。無関係な一致が多すぎる場合は閾値を上げます。関連する会話を見逃す場合は下げます。

---

## 検索結果の表示

会話ドロワーでは、検索結果に以下が表示されます：

- **会話タイトル**（または「無題」）
- **一致するメッセージ** — 一致したユーザーまたはモデルのメッセージ
- **ロールラベル** — ユーザーまたはモデル
- **コンテキストメッセージ** — コンテキストのための周辺メッセージ

結果をタップすると、一致するメッセージまでスクロールされた状態で会話が開きます。
