# Веб-поиск

Позвольте модели искать в интернете и загружать веб-страницы в реальном времени. При включении модель может искать актуальную информацию, проверять факты, извлекать документацию или исследовать темы — всё автономно через вызов инструментов.

## Поддерживаемые провайдеры

| Провайдер | Описание | Бесплатный тариф | Настройка |
|----------|-------------|-----------|-------|
| **Brave** | Приватно-ориентированный поисковый API | Да (2 000 запросов/мес) | [api.search.brave.com](https://api.search.brave.com/) |
| **Serper** | Быстрый API поиска Google | Да (2 500 запросов/мес) | [serper.dev](https://serper.dev) |
| **Tavily** | AI-оптимизированный поиск, создан для LLM-агентов | Да (1 000 запросов/мес) | [tavily.com](https://tavily.com) |
| **SearXNG** | Метапоисковая система на собственном хостинге | Свой хостинг (безлимитно) | Ваш собственный экземпляр |

## Настройка

### Brave

1. Получите API-ключ на [Brave Search API](https://api.search.brave.com/)
2. В Agora перейдите в **Настройки → Веб-поиск**
3. Выберите **Brave** как поискового провайдера
4. Вставьте ваш API-ключ

### Serper

1. Получите API-ключ на [serper.dev](https://serper.dev)
2. В Agora перейдите в **Настройки → Веб-поиск**
3. Выберите **Serper**
4. Вставьте ваш API-ключ

### Tavily

1. Получите API-ключ на [tavily.com](https://tavily.com)
2. В Agora перейдите в **Настройки → Веб-поиск**
3. Выберите **Tavily**
4. Вставьте ваш API-ключ

### SearXNG

1. Настройте экземпляр SearXNG (свой хостинг) или используйте публичный экземпляр
2. В Agora перейдите в **Настройки → Веб-поиск**
3. Выберите **SearXNG**
4. Введите **Базовый URL** вашего экземпляра (например, `https://searx.be`)
5. API-ключ необязателен (нужен только если ваш экземпляр требует аутентификацию)

!!! warning "Публичные экземпляры"
    Публичные экземпляры SearXNG часто имеют ограничения по частоте или ненадёжны. Для постоянного использования рекомендуется собственный хостинг.

## Конфигурация

### Максимум результатов

Установите, сколько результатов поиска получать за запрос: **1–10**. По умолчанию зависит от устройства. Больше результатов дают модели больше контекста, но стоят больше токенов.

### Включение/Отключение

Переключите **Включить веб-поиск** на странице настроек Веб-поиска. При отключении модель не может вызывать инструмент веб-поиска.

## Как модель использует поиск

Когда вы задаёте вопрос, требующий актуальной или внешней информации, модель автоматически вызывает веб-поиск:

1. **Поиск**: Модель вызывает поисковый API с запросом, который она формулирует
2. **Загрузка**: Модель может опционально загрузить полное содержимое страниц из URL результатов
3. **Синтез**: Модель читает результаты и интегрирует их в свой ответ

Вы увидите каждый поиск и загрузку как встроенные карточки инструментов в диалоге.

### Пример

```text
Вы: «Какая последняя версия Python?»
Модель: [Ищет «последняя версия Python 2026»]
       [Читает результат]
       «Python 3.14.0 был выпущен в октябре 2025 года...»
```

## Веб-загрузка (Web Fetch)

Помимо поиска, модель может загружать и читать конкретные веб-страницы. Когда модель встречает URL в результатах поиска, она может вызвать `web_fetch` для получения полного содержимого страницы:

- Загруженное содержимое конвертируется в markdown
- Модель обрабатывает его и извлекает релевантную информацию
- Результаты загрузки отображаются как карточки инструментов

---

## Вопросы конфиденциальности

При использовании веб-поиска:

- Ваши запросы отправляются поисковому провайдеру (Brave, Serper и т.д.), а не в Agora
- Agora не логирует и не хранит ваши поисковые запросы (кроме как в самом диалоге)
- Собственный хостинг SearXNG даёт наибольшую конфиденциальность — запросы остаются на вашей инфраструктуре
