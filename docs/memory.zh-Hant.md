# 記憶體與快取

Agora 具有持續性記憶體系統，讓模型能夠跨對話記住資訊。搭配自動嵌入式快取，它提供了一個隨使用量成長的知識庫。

## 記憶體類型

### 使用中記憶體（Active Memory）

單一、永遠啟用的記憶體上下文，會隨 **每次 API 呼叫** 傳送給模型。可將其視為模型始終看到的便利貼。

**使用「使用中記憶體」存放：**
- 您的姓名、偏好和背景
- 模型應始終知道的專案上下文
- 適用於所有對話的持續性指示
- 您不想再重複說明的事實

**使用中記憶體內容範例：**
```text
使用者：Newo Ether
偏好：日常對話偏好中文，技術主題使用英文。
專案：開發 Agora — 一個 BYOK Android LLM 客戶端。
程式風格：Kotlin、Jetpack Compose、MVVM 架構。
```

#### 編輯使用中記憶體

1. 前往 **設定 → 記憶體**
2. 捲動到 **使用中記憶體**
3. 點選 **編輯使用中記憶體**
4. 輸入您的內容
5. 點選 **儲存**

如果啟用了 **存取使用中記憶體**，模型也可以透過工具呼叫來更新使用中記憶體。

---

### 已儲存記憶體（Saved Memories）

一組具名記憶體檔案，模型可以搜尋、讀取、建立、編輯和刪除。與使用中記憶體（總是傳送）不同，已儲存記憶體是按需擷取的。

**使用「已儲存記憶體」存放：**
- 參考資料（API 文件、設定詳細資訊、指令）
- 專案特定筆記
- 從過去對話中學到的見解
- 您希望模型在相關時能回想起來的任何內容

#### 手動建立記憶體

1. 前往 **設定 → 記憶體**
2. 點選 **新增記憶體**
3. 輸入：
    - **標題** — 描述性名稱
    - **描述** — 簡短摘要（用於搜尋比對）
    - **內容** — 完整的記憶體內容
4. 點選 **建立**

#### 模型建立的記憶體

當啟用 **存取已儲存記憶體** 時，模型可以透過工具呼叫來建立、讀取、更新和刪除記憶體檔案。這讓模型能夠：

- 記住您告訴它的事實
- 儲存有用的程式碼片段或設定
- 隨著時間建立知識庫
- 清理過時的資訊

## 記憶體權限

控制模型可以存取的內容：

| 設定 | 位置 | 何時啟用 |
|---------|----------|----------------|
| **存取已儲存記憶體** | 設定 → 記憶體 | 您希望模型讀取/寫入記憶體檔案 |
| **存取使用中記憶體** | 設定 → 記憶體 | 您希望模型更新持續性上下文 |
| **存取過去對話** | 設定 → 對話搜尋 | 您希望模型搜尋聊天記錄 |

這三項預設皆為 **關閉**。僅啟用您需要的項目。

## 自動快取

自動快取會在新訊息到達時自動為其生成嵌入向量。這讓您的對話搜尋索引保持最新，無需手動介入。

### 啟用自動快取

1. 前往 **設定 → 對話搜尋**
2. 選擇一個嵌入模型（如果您尚未選擇 — 請參閱 [嵌入 / RAG](embedding.md)）
3. 在 **快取** 下，切換 **自動快取新訊息**

啟用後，每則新訊息（使用者和模型）都會自動進行嵌入並索引以供語義搜尋使用。

### 手動快取

如果自動快取關閉，您可以手動快取訊息：

1. 前往 **設定 → 對話搜尋**
2. 點選 **快取** — 為所有未快取的訊息計算嵌入向量
3. 進度以圓形指示器顯示

點選 **重新快取** 從頭重建整個索引。這會刪除所有已快取的嵌入向量並重新處理每則訊息。使用時機：
- 您更換了嵌入模型
- 快取似乎損壞或過時
- 搜尋結果異常不佳

!!! warning
    重新快取是不可逆的，且可能需要一段時間，取決於您的訊息數量和嵌入模型速度。

### 快取狀態

嵌入模型設定顯示已快取與未快取的訊息數量：
- **「所有 N 則訊息已快取」** — 最新狀態
- **「Y 則訊息中有 X 則未快取」** — 有待處理的積壓

## 聊天中的記憶體工具呼叫

當模型使用記憶體工具時，您會看到內嵌卡片：

| 工具 | 卡片文字 |
|------|-----------|
| 查找 | 「查閱了 N 個已儲存記憶體」 |
| 讀取 | 「已讀取 [記憶體名稱]」 |
| 儲存 | 「已儲存 [記憶體名稱]」 |
| 編輯 | 「已更新 [記憶體名稱]」 |
| 刪除 | 「已移除 [記憶體名稱]」 |
| 更新使用中 | 「已更新使用中記憶體」 |

點選任何卡片可查看已讀取或寫入的完整內容。

---

## 最佳實踐

- **保持使用中記憶體簡潔** — 它包含在每次 API 呼叫中，冗長的內容會浪費 token
- **為已儲存記憶體使用描述性標題** — 標題有助於模型找到正確的記憶體
- **如果您經常使用對話搜尋，請啟用自動快取**
- **切換嵌入模型後重新快取** — 不同模型會產生不相容的嵌入向量
