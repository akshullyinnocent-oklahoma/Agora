# 對話

Agora 的對話系統建立在 **非線性分支** 之上 — 與大多數聊天應用程式不同，您可以編輯任何過去的訊息並探索替代回應路徑，而不會遺失原始對話。

## 建立對話

在對話抽屜中點選 **新聊天**，或直接在聊天畫面中開始輸入。新的對話會在您輸入第一則訊息時自動建立。

對話會在第一則回應後自動命名（如果已啟用 [標題生成](system-prompts.md#自動標題生成)），或者您可以手動重新命名。

## 管理對話

### 切換對話

開啟 **對話抽屜**（漢堡選單 :material-menu: 或向右滑動）並點選任何對話以開啟。

### 重新命名

1. 在抽屜中長按對話
2. 點選 **重新命名**
3. 輸入新標題並儲存

### 刪除

1. 在抽屜中長按對話
2. 點選 **刪除**
3. 確認刪除 — 此動作無法復原

---

## 非線性分支

這是 Agora 的標誌性功能。每則訊息都可以是分支點。

### 編輯過去的訊息

1. 長按任何訊息泡泡（使用者或模型）
2. 點選 **編輯**
3. 修改訊息內容
4. 傳送 — Agora 會從此點建立一個 **新分支**

原始分支會被保留。您可以隨時在分支之間切換。

### 分支如何運作

每則訊息存在於 **樹狀結構** 中：

```text
訊息 1（使用者）
├── 訊息 2（模型）← 原始回應
└── 訊息 3（模型）← 編輯訊息 1 後建立的分支
    ├── 訊息 4（使用者）
    └── ...
```

當您編輯訊息並重新生成時，新回應會成為原始回應的同級 — 兩者都存在於同一個父訊息之下。

### 切換分支

當訊息有多個子項（分支）時，UI 會顯示導航控制項以在它們之間切換。您可以探索替代路徑而不會遺失上下文。

### 為何使用分支？

- **探索替代方案** — 用不同的措辭問同一個問題
- **A/B 測試提示詞** — 比較來自不同系統提示詞或模型的回應
- **修正錯誤** — 更正問題中的錯字，同時保留原始討論串
- **迭代** — 透過多個版本精煉提示詞，同時保留所有嘗試

## 訊息操作

長按任何訊息以存取以下動作：

| 動作 | 描述 |
|--------|-------------|
| **複製** | 將訊息文字複製到剪貼簿 |
| **編輯** | 編輯訊息並建立分支 |
| **資訊** | 檢視中繼資料：時間戳記、使用的模型、token 數量 |
| **刪除** | 刪除此訊息及其所有後續回覆 |

!!! warning "刪除訊息"
    刪除訊息也會移除其後的所有回覆。此動作無法復原。

## 底部列

聊天輸入區域提供快速存取基本控制項：

### 模型選擇器

點選底部列左側的模型名稱以開啟 **模型選擇器**。您可以隨時切換模型 — 即使在對話中途。同一對話中的不同訊息可以使用不同的模型。

### 附件

點選 **+**（:material-plus:）來附加檔案：

- **照片** — 來自您相簿的圖片
- **影片** — 影片檔案（支援幀擷取）
- **檔案** — 任何檔案類型，包括 PDF

支援的圖片格式會直接傳送給支援視覺的模型。PDF 檔案會開啟頁面選擇對話框。

### 傳送

輸入您的訊息並點選 **傳送**（:material-send:）。模型會逐個 token 串流其回應。

## 串流與顯示

### 即時串流

回應會隨著模型生成逐字顯示。Agora 會自動捲動以保持最新內容可見。點選 **捲動到底部** 按鈕（當您向上捲動時出現）可跳回即時回應。

### Markdown 渲染

模型回應以完整的 Markdown 支援進行渲染：

- **標題**、**粗體**、*斜體*、`內嵌程式碼`
- **程式碼區塊** 附帶語法高亮（使用 ````` ``` `````）
- **表格**、引用區塊、清單
- **LaTeX 數學** — 內嵌 `$E=mc^2$` 和區塊 `$$\int_a^b f(x)dx$$`

### 思考顯示

對於支援推理的模型（OpenAI o 系列、Anthropic 延伸思考、Gemini 思考、DeepSeek-R1），模型的思考過程會顯示在最終回應之前的 **可摺疊面板** 中：

- 面板在推理階段顯示「思考中...」
- 完成後，顯示思考持續時間（例如「思考了 12 秒」）
- 點選以展開/摺疊思考內容
- 思考期間進行的工具呼叫會被計數（例如「思考了 8 秒，呼叫了 2 個工具」）

## 個別對話設定

每個對話都可以覆寫全域預設值：

- **模型** — 為此對話選擇不同的模型
- **系統提示詞** — 使用不同的系統指令
- **生成參數** — 溫度、最大 token 數、思考等級

這些覆寫可以從頂部列的對話溢出選單中設定。

---

## 上下文視窗

Agora 即時追蹤 token 使用量。當對話超出模型的上下文視窗時，較舊的訊息會以視覺方式 **變暗**，以表示它們不在使用中的上下文內。模型不再「看到」變暗的訊息，但它們在您的 UI 中仍然可見。

在 **設定 → 生成 → 上下文視窗** 中調整上下文視窗大小。
