# ウェブ検索

モデルにリアルタイムでインターネットを検索させ、ウェブページを取得させます。有効にすると、モデルは最新情報の検索、事実確認、ドキュメントの取得、トピックの調査を — すべてツール呼び出しを通じて自律的に行えます。

## 対応プロバイダー

| プロバイダー | 説明 | 無料枠 | セットアップ |
|----------|-------------|-----------|-------|
| **DuckDuckGo Lite** | 匿名、API キー不要 | あり（無制限、ベストエフォート） | セットアップ不要 — すぐに動作 |
| **Brave** | プライバシー重視の検索 API | あり（2,000 クエリ/月） | [api.search.brave.com](https://api.search.brave.com/) |
| **Serper** | 高速 Google 検索 API | あり（2,500 クエリ/月） | [serper.dev](https://serper.dev) |
| **Tavily** | AI 最適化検索、LLM エージェント向け | あり（1,000 クエリ/月） | [tavily.com](https://tavily.com) |
| **SearXNG** | セルフホストメタ検索エンジン | セルフホスト（無制限） | 自分のインスタンス |

## セットアップ

### DuckDuckGo Lite

DuckDuckGo Lite は**デフォルト**の検索プロバイダーです — API キー不要で、すぐに動作します。

1. Agora で、**設定 → ウェブ検索** に移動します
2. 検索プロバイダーとして **DuckDuckGo Lite** を選択します
3. キーや URL は不要です — すぐに検索を開始できます

!!! note "ベストエフォートサービス"
    DuckDuckGo Lite は `lite.duckduckgo.com` の HTML スクレイピングを使用します。DDG はレイアウトを変更したり、レート制限をかけたり、自動リクエストをブロックする場合があります。明示的にベストエフォートの、キー不要のオプションとして提供されています。信頼性が必要な場合は、以下の API ベースのプロバイダーを設定してください。

### Brave

1. [Brave Search API](https://api.search.brave.com/) から API キーを取得します
2. Agora で、**設定 → ウェブ検索** に移動します
3. 検索プロバイダーとして **Brave** を選択します
4. API キーを貼り付けます

### Serper

1. [serper.dev](https://serper.dev) から API キーを取得します
2. Agora で、**設定 → ウェブ検索** に移動します
3. **Serper** を選択します
4. API キーを貼り付けます

### Tavily

1. [tavily.com](https://tavily.com) から API キーを取得します
2. Agora で、**設定 → ウェブ検索** に移動します
3. **Tavily** を選択します
4. API キーを貼り付けます

### SearXNG

1. SearXNG インスタンスをセットアップ（セルフホスト）するか、公開インスタンスを使用します
2. Agora で、**設定 → ウェブ検索** に移動します
3. **SearXNG** を選択します
4. インスタンスの **ベース URL** を入力します（例：`https://searx.be`）
5. API キーはオプションです（インスタンスが認証を必要とする場合のみ）

!!! warning "公開インスタンス"
    公開 SearXNG インスタンスはレート制限されたり信頼性が低いことがよくあります。一貫した使用にはセルフホスティングをお勧めします。

## 設定

### 最大結果数

クエリごとに取得する検索結果の数を設定します：**1～10**。デフォルトはデバイスに依存します。結果が多いほどモデルにより多くのコンテキストを提供しますが、トークンコストが増加します。

### 有効/無効

ウェブ検索設定ページで **ウェブ検索を有効にする** を切り替えます。無効にすると、モデルはウェブ検索ツールを呼び出せません。

---

## モデルが検索を使用する方法

最新情報や外部情報が必要な質問をすると、モデルは自動的にウェブ検索を呼び出します：

1. **検索**: モデルが独自に作成したクエリで検索 API を呼び出します
2. **取得**: モデルは結果 URL からオプションでページ全体のコンテンツを取得できます
3. **統合**: モデルは結果を読み取り、応答に統合します

各検索と取得は、会話内でインラインツールカードとして表示されます。

### 例

```text
あなた: 「Python の最新バージョンは何ですか？」
モデル: [「最新の Python バージョン 2026」を検索]
       [結果を読み取り]
       「Python 3.14.0 は 2025 年 10 月にリリースされました...」
```

---

## ウェブ取得

検索を超えて、モデルは特定のウェブページを取得して読み取ることができます。モデルが検索結果で URL に遭遇すると、`web_fetch` を呼び出してページ全体のコンテンツを取得できます：

- 取得されたコンテンツはマークダウンに変換されます
- モデルはそれを処理して関連情報を抽出します
- 取得結果はツールカードとして表示されます

---

## プライバシーに関する考慮事項

ウェブ検索を使用する場合：

- クエリは Agora ではなく検索プロバイダー（Brave、Serper など）に送信されます
- Agora は検索クエリをログに記録または保存しません（会話自体を除く）
- SearXNG のセルフホスティングは最大のプライバシーを提供します — クエリは自身のインフラストラクチャ内に留まります
