# エージェントツール

Agora のモデルは自律的にツールを使用できます — 各アクションを手動でトリガーすることなく、何を検索、実行、読み取り、記憶するかをモデルが決定します。ツールは **マルチラウンドループ** で動作します：モデルはツールを呼び出し、結果を読み取り、その後別のツールを呼び出すか応答するかを決定できます。

## ツール呼び出しの仕組み

1. あなたがメッセージを送信します
2. モデルは外部情報やアクションが必要だと判断します
3. ツール名と引数を含む構造化されたリクエストである **ツール呼び出し** を発行します
4. Agora がデバイス上またはリモートサーバーでツールを実行します
5. 結果がモデルにフィードバックされます
6. モデルは別のツールを呼び出すか、最終応答を生成できます

このループは単一のメッセージターン内で複数回繰り返すことができます。

## 利用可能なツール

### ウェブ検索

インターネットを検索してウェブページを取得します。モデルは最新情報の検索、事実確認、ドキュメントの取得を行えます。

- **プロバイダー**: DuckDuckGo Lite（デフォルト、キー不要）、Brave、Serper、Tavily、SearXNG
- **設定**: 設定 → ウェブ検索
- **ガイド**: [ウェブ検索](web-search.md)

### 画像生成

テキストプロンプトから専用のテキストから画像へのモデルを使用して画像を生成します。画像は会話内にインラインで表示され、フルスクリーンで表示できます。

- **プロバイダー**: BYOK — 独自の API キーとベース URL を使用し、チャットモデルとは分離
- **設定**: 設定 → 画像生成
- **ガイド**: [画像生成](image-generation.md)

### コード実行

隔離された環境でコードを実行します：

- **Gemini コード実行** — Gemini モデル向けの組み込みコード実行（セットアップ不要）
- **Sandbox** — PRoot 経由のローカル Alpine Linux 環境、パッケージ管理と SAF ファイルアクセス付き

### リモートシェル

[Conch](https://github.com/newo-ether/conch) プロトコルを介してリモートマシン上でコマンドを実行します。モデルはサーバーステータスの確認、ファイル管理、スクリプトの実行を行えます。

- **プロトコル**: エンドツーエンド暗号化（ECDH + AES-256-GCM）
- **設定**: 設定 → シェル
- **ガイド**: [リモートシェル](shell.md)

### ファイル操作

Conch プロトコルを通じてリモートデバイス上のファイルの読み取り、書き込み、編集、glob 検索、grep 検索を行います。モデルはリモートファイルシステムを直接操作できます。

!!! note
    ファイル操作には設定済みの Conch シェルデバイスが必要です。セットアップについては [リモートシェル](shell.md) を参照してください。

### メモリ

会話をまたぐ永続的な知識ストレージ：

- **アクティブメモリ** — すべての API 呼び出しに常に含まれます。モデルが常に覚えておくべき事実、好み、コンテキストに使用します。
- **保存メモリ** — モデルがツール呼び出しを通じて検索、読み取り、書き込み、編集できる名前付きメモリファイルのコレクション。

詳細は [メモリ & キャッシュ](memory.md) を参照してください。

### 会話検索

モデルはキーワードまたはセマンティック（RAG）メソッドを使用して過去の会話履歴を検索できます。これにより、手動で見つけて共有することなく、以前の議論を参照できます。

セットアップについては [会話検索](search.md) を参照してください。

## チャットでのツール UI

ツールが呼び出されると、会話内にインラインで表示されます：

<div class="grid cards" markdown>

- **:material-progress-wrench: ツール呼び出しバナー**

    ---

    ツール名と簡単なステータスを表示します（例：:material-magnify: 「ウェブで『最新の AI ニュース』を検索中」）。

- **:material-check-circle: ツール結果**

    ---

    実行後、フォーマットされた結果または概要を表示します（例：「『最新の AI ニュース』の検索結果 5 件」）。

</div>

### 展開可能な詳細

ツール呼び出しをタップして展開し、以下を表示します：

- **引数** — ツールに送信された正確なパラメータ
- **結果** — ツール実行からの生の出力
- **ステータス** — 成功、失敗、または部分的な結果

### 失敗した呼び出し

ツール呼び出しが失敗した場合、モデルはエラーを通知され、再試行または調整できます。エラーメッセージとともに赤いバナーが表示されます。

## ツール権限

モデルがアクセスできるツールを制御します：

| 設定 | 場所 | デフォルト |
|---------|----------|---------|
| ウェブ検索 | 設定 → ウェブ検索 | オフ |
| シェル | 設定 → シェル | オフ |
| メモリ（保存） | 設定 → メモリ → 保存メモリへのアクセス | オフ |
| メモリ（アクティブ） | 設定 → メモリ → アクティブメモリへのアクセス | オフ |
| 過去の会話 | 設定 → メモリ → 過去の会話へのアクセス | オフ |
| 会話検索 | 設定 → 会話検索 | オン* |

*モデルが会話を検索する能力は、埋め込みモデルが設定されているかどうかに依存します。設定されていない場合、キーワード検索のみが利用可能です。

## マルチラウンドツールループ

モデルは複数のツール呼び出しを連鎖できます。例：

1. ユーザー: 「最新の Linux カーネルバージョンは何ですか？また、私のサーバーはそれを実行していますか？」
2. モデルが `web_search("最新の Linux カーネルバージョン")` を呼び出し
3. モデルが `shell_execute("uname -r", device="my-server")` を呼び出し
4. モデルが結果を比較して応答

各ツール呼び出しとその結果は、最終的なテキスト応答の前に、会話内で個別のインライン項目として表示されます。
