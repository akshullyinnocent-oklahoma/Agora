# Удалённая оболочка (Conch)

Agora может выполнять команды на удалённых машинах через протокол [Conch](https://github.com/newo-ether/conch) — сквозную зашифрованную безопасную оболочку, разработанную для AI-агентов.

## Как это работает

```text
Agora (Android)  ──ECDH + AES-256-GCM──▶  Сервер Conch (Linux/macOS/Windows)
                                           │
                                           ├── Выполнение команд
                                           ├── Чтение/запись/редактирование файлов
                                           ├── Glob и grep поиск
                                           └── Возврат результатов
```

Модель сама решает, когда использовать оболочку — она может проверять статус сервера, управлять файлами, запускать скрипты или автономно устранять неполадки.

## Безопасность

Conch использует надёжное шифрование и защиту от злоупотреблений:

- **Обмен ключами ECDH** — эфемерные ключи на сессию
- **Шифрование AES-256-GCM** — весь трафик зашифрован
- **Подпись HMAC-SHA256** — целостность сообщений проверена
- **Ограничение скорости Token Bucket** — предотвращает злоупотребления
- **Защита от повторов на основе nonce** — каждый запрос уникален

!!! note
    Команды выполняются с разрешениями пользователя, запустившего сервер Conch. Используйте ограниченную учётную запись для чувствительных окружений.

## Настройка

### Шаг 1: Разверните сервер Conch

Разверните сервер Conch на вашей целевой машине. Инструкции по настройке см. в [репозитории Conch](https://github.com/newo-ether/conch).

### Шаг 2: Добавьте устройство в Agora

1. Перейдите в **Настройки → Оболочка**
2. Включите **Инструмент оболочки**
3. Нажмите **Добавить устройство**
4. Выберите тип устройства: **Conch** или **SSH**
5. Заполните детали устройства:

=== "Conch"

    | Поле | Описание | Пример |
    |-------|-------------|---------|
    | **Имя** | Отображаемое имя устройства | `Сборочный сервер` |
    | **Описание** | Необязательная заметка об этой машине | `Офисный Ubuntu` |
    | **URL сервера** | Конечная точка сервера Conch (хост:порт) | `http://192.168.1.100:14216` |
    | **API-ключ** | Токен аутентификации | Из конфигурации сервера Conch |
    | **Таймаут** | Таймаут команды в секундах | `30` |

=== "SSH"

    | Поле | Описание | Пример |
    |-------|-------------|---------|
    | **Имя** | Отображаемое имя устройства | `VPS Сервер` |
    | **Описание** | Необязательная заметка об этой машине | `Продакшн веб-сервер` |
    | **Хост** | Имя хоста SSH или IP-адрес | `192.168.1.200` |
    | **Порт** | Порт SSH | `22` |
    | **Пользователь** | Имя пользователя SSH | `root` |
    | **Пароль** | Пароль SSH | Ваш пароль SSH |

Нажмите **Добавить** для сохранения.

### Шаг 3: Использование

После настройки модель может получить доступ к устройству. Ручной триггер не требуется — модель автоматически обнаруживает доступные устройства оболочки и вызывает их, когда это уместно.

## Поддержка нескольких устройств

Добавьте несколько устройств оболочки, чтобы модель могла работать на разных машинах:

- **Сборочный сервер** — компиляция и тестирование кода
- **Домашняя лаборатория** — управление сервисами на собственном хостинге
- **ВМ для разработки** — редактирование кода и запуск скриптов

Каждое устройство настраивается независимо с собственным именем, URL и учётными данными. Модель может различать их и выбирать подходящее устройство для каждой задачи.

## Доступные операции

### Выполнение команд (`shell_execute`)

Запуск любой команды оболочки и получение stdout, stderr и кода выхода.

### Файловые операции

| Инструмент | Функция |
|------|----------|
| `file_read` | Чтение файла из удалённой файловой системы |
| `file_write` | Запись или перезапись файла |
| `file_edit` | Выполнение точных строковых замен в файле |
| `file_glob` | Поиск файлов по glob-шаблону |
| `file_grep` | Поиск в содержимом файлов по регулярному выражению |

Все файловые операции проходят через зашифрованный канал Conch.

## Интеграция с MCP

Conch также может служить **MCP-сервером Claude Desktop**. Если вы используете Claude Code или другой MCP-клиент, вы можете настроить Conch как провайдера инструментов для удалённого доступа к файлам и оболочке с вашего рабочего стола.

Инструкции по настройке MCP см. в [документации Conch](https://github.com/newo-ether/conch).

## Устранение неполадок

### Устройство отображается как недоступное
- Проверьте, что сервер Conch запущен
- Убедитесь, что URL достижим с вашего Android-устройства
- Проверьте правила брандмауэра на сервере

### Команды завершаются по таймауту
- Увеличьте таймаут в настройках устройства
- Проверьте, что команда не зависает (требует ввода пользователя и т.д.)

### Ошибка аутентификации
- Убедитесь, что API-ключ совпадает с конфигурацией сервера
- При необходимости перегенерируйте ключи
