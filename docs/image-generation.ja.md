# 画像生成

会話内でテキストプロンプトから直接、テキストから画像へのモデルを使用して画像を生成します。

## 機能

画像生成が有効な場合、Agora は専用のテキストから画像へのモデル（DALL·E、GPT-Image、Imagen、FLUX、Stable Diffusion、Seedream、Qwen-Image など多数）を使用して、プロンプトを画像に変換できます。生成された画像は会話に返されるため、他の返信と同様に反復できます。

画像生成は **独自のモデル選択** を使用し、チャットするモデルとは独立しています — そのため、あるモデルとチャットしながら別のモデルで画像を生成できます。

## セットアップ

1. **設定 → 画像生成** に移動します
2. **画像生成を有効にする** をオンにします
3. **モデル** をタップしてテキストから画像へのモデルを選択します
4. オプションで **デフォルトサイズ**（幅 × 高さ）を設定します

!!! note "プロバイダー認証情報を再利用"
    画像モデルはそのプロバイダーの API キーとベース URL を再利用します — **設定 → API プロバイダー** で既に設定したものと同じ認証情報です。個別のキーを入力する必要はありません。ローカルモデル（適切な重みを持つもの）も使用して、生成をデバイス上に保つことができます。

## モデル選択

**モデル** をタップして生成に使用するモデルを選択します。

- ピッカーは、同期されたすべてのモデルからテキストから画像へのモデルのように見えるものをフィルタリングして表示するため、リストは短く保たれます。
- 希望のモデルがリストにない場合（珍しい名前の場合）、**すべてのモデルを表示** を有効にして完全なリストから選択します。
- 適切に同期された `Provider:model` エントリのみが有効な選択としてカウントされます。リストが空の場合は、まず **設定 → API プロバイダー** / **モデル管理** でモデルを同期してください。

## デフォルトサイズ

デフォルトの出力寸法を設定します。**幅 × 高さ** をピクセル単位で入力します（例：`1024` × `1024`）。

- デフォルトは `1024 × 1024` です。
- 対応サイズはモデルとプロバイダーによって異なります — モデルがサイズを拒否した場合は、ドキュメントに記載されている値を試してください（一般的なオプションは `1024×1024`、`1024×1792`、`1792×1024`）。

## 仕組み

1. 画像生成を有効にして画像モデルを選択します
2. 会話でアシスタントに画像の作成を依頼します
3. Agora はそのプロバイダーの認証情報を使用して、リクエストを設定された画像モデルにルーティングします
4. 生成された画像が会話に返されます

!!! tip
    プロンプトは具体的に — 被写体、スタイル、構図、照明を説明してください。明確なプロンプトは曖昧なものよりもはるかに良い結果を生み出します。
