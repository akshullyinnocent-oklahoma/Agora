# API プロバイダー

Agora は AI プロバイダーに直接接続します — 仲介者なし、サブスクリプションなし、テレメトリなし。自分の API キーを持ち込み、すべてはデバイスから実行されます。

## 組み込みプロバイダー

| プロバイダー | ベース URL | モデル | 備考 |
|----------|----------|--------|-------|
| **Google** | `https://generativelanguage.googleapis.com/v1beta` | Gemini シリーズ | Google AI Studio 経由で無料枠あり |
| **OpenAI** | `https://api.openai.com/v1` | GPT-4、GPT-4o、o-シリーズ | 推論モデル対応 |
| **Anthropic** | `https://api.anthropic.com/v1` | Claude シリーズ | 拡張思考対応 |
| **DeepSeek** | `https://api.deepseek.com/v1` | DeepSeek-V3、DeepSeek-R1 | 推論モデル対応 |
| **Qwen** | `https://dashscope-intl.aliyuncs.com/compatible-mode/v1` | Qwen シリーズ | Alibaba DashScope 経由 |
| **Ollama** | `http://localhost:11434/v1` | プルした任意のモデル | セルフホスト、API キー不要 |
| **OpenRouter** | `https://openrouter.ai/api/v1` | マルチプロバイダー | 1つの API で多数のモデルにアクセス |
| **ローカル** | N/A | GGUF モデル | llama.cpp 経由でデバイス上実行、完全オフライン |

## プロバイダーの切り替え

設定のプロバイダーセレクターをタップしてプロバイダーを切り替えます。各プロバイダーは以下を個別に保持します：

- API キー
- ベース URL（プロキシ/セルフホスト用に編集可能）
- モデルリスト

---

## API キー

### プロバイダーごとに複数のキー

各プロバイダーは複数の名前付き API キーをサポートしています。これにより以下が可能になります：

- **ローテーション** — 異なる使用量枠に応じてキーを切り替え
- **整理** — 仕事用と個人用を分離
- **フォールバック** — バックアップキーを準備

### キーの管理

1. **設定 → プロバイダー** に移動します
2. プロバイダーを選択します
3. **API キー** の下で **新しいキーを追加** をタップします
4. **名前**（例：「仕事用」「個人用」「チーム共有」）と **キー値** を入力します
5. **追加** をタップします

ラジオボタンをタップしてアクティブなキーを設定します。キーを長押しすると **編集** または **削除** できます。

### キーの安全性

!!! warning
    API キーは暗号化された Room データベースにローカル保存されます。Agora サーバー（存在しません）に送信されることはありません。ただし、`.agora` エクスポートファイルに含めると平文でエクスポートされます。

---

## カスタムプロバイダー

任意の OpenAI 互換 API エンドポイントを追加します：

1. **設定 → プロバイダー** に移動します
2. プロバイダーリストの下部にある **+ カスタムプロバイダーを追加** をタップします
3. 以下を入力します：
    - **プロバイダー名** — 任意の表示名
    - **ベース URL** — API エンドポイント
4. **追加** をタップします

Agora は `{base_url}/v1/models` からモデルリストを取得します。追加後、カスタムプロバイダーは組み込みのものと同様に動作します：API キーの追加、モデルの同期、チャットが可能です。

### ユースケース

- **セルフホスト** — vLLM、LocalAI、text-generation-webui、その他の OpenAI 互換サーバーに接続
- **プロキシ** — 企業プロキシや API ゲートウェイを経由
- **代替エンドポイント** — Azure OpenAI、Cloudflare AI Gateway、その他の互換サービスを使用

### 名前変更または削除

カスタムプロバイダーを長押しして **名前変更** または **削除** します。削除するとプロバイダーとそのすべてのキーが削除されます。

!!! warning
    組み込みプロバイダーは名前変更も削除もできません。

---

## ベース URL の上書き

すべてのプロバイダー（組み込みを含む）には編集可能な **ベース URL** があります。以下の場合に便利です：

- **プロキシ**: `https://my-proxy.example.com/v1` を経由
- **セルフホスト**: 自分のインスタンスを指定
- **リージョンルーティング**: 地域固有のエンドポイントを使用

---

## モデルの同期

API キーを追加した後、モデルリストを同期します：

1. **設定 → モデル** に移動します
2. **すべてのプロバイダーから同期** をタップします
3. Agora は設定されたすべてのプロバイダーから利用可能なモデルを取得します

スナックバーに同期の進捗と結果が表示されます。その後、個々のモデルを有効/無効にし、デフォルトを設定できます。

## プロバイダー固有の注意事項

### Google Gemini

- API キーは [Google AI Studio](https://aistudio.google.com/apikey) から取得
- レート制限付きの無料枠あり
- コード実行と検索グラウンディング（組み込みツール）をサポート

### OpenAI

- API キーは [プラットフォーム](https://platform.openai.com/api-keys) から取得
- 推論モデル（o1、o3）は特定の API アクセスが必要
- ストリーミング、ツール、視覚すべて対応

### Anthropic

- API キーは [コンソール](https://console.anthropic.com/) から取得
- 設定可能なトークン予算による拡張思考
- 並列呼び出しを含むツール使用に対応

### Ollama

- API キー不要（ローカルネットワーク）
- ベース URL は通常 `http://<host>:11434/v1`
- Ollama の API からモデルリストを取得
- Ollama 固有のトラブルシューティングについては [FAQ](faq.md) を参照

### OpenRouter

- 1つの API キーで 200+ モデルにアクセス
- トークン単位の料金はモデルによって異なる
- 個別のプロバイダーアカウントなしで様々なモデルを試すのに便利

### ローカル（llama.cpp）

- ネットワーク不要
- GGUF モデルファイルはデバイスに保存
- セットアップについては [ローカルモデル](local-model.md) を参照
