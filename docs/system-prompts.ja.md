# システムプロンプト

システムプロンプトはモデルのペルソナ、動作、基本ルールを定義します。Agora は命令の組み立て方とモデルへの送信方法をきめ細かく制御できます。

## 3セクションエディタ

各システムプロンプトテンプレートには、独立して編集可能な3つのセクションがあります：

```text
┌─────────────────────────────────┐
│ システムプロンプト              │ ← コア命令（ペルソナ、ルール、トーン）
├─────────────────────────────────┤
│ ユーザープレフィックス          │ ← すべてのユーザーメッセージの前に付加
├─────────────────────────────────┤
│ ユーザーサフィックス            │ ← すべてのユーザーメッセージの後に付加
└─────────────────────────────────┘
```

### システムプロンプト

メインの命令ブロックです。ここで以下を定義します：

- **ペルソナ**: 「あなたはクリーンアーキテクチャを専門とするシニア Python 開発者です。」
- **ルール**: 「常に中国語で回答してください。リストには箇条書きを使用してください。」
- **制約**: 「決して謝らないでください。簡潔に。説明よりもコードを優先してください。」

### ユーザープレフィックス & サフィックス

これらは送信するすべてのメッセージをラップします：

- **ユーザープレフィックス** — メッセージテキストの前に追加されます。リマインダーやコンテキストタグに便利です。
- **ユーザーサフィックス** — メッセージテキストの後に追加されます。締めの指示に便利です。

**例**: プレフィックスが `[コンテキスト: Agora ドキュメント作成中]` でサフィックスが `\n\nマークダウンで返信してください。` の場合、モデルが受け取るのは：

```text
[コンテキスト: Agora ドキュメント作成中]
ウェブ検索の設定方法は？
マークダウンで返信してください。
```

## プロンプトの作成

1. **設定 → システムプロンプト** に移動します
2. **新しいプロンプトを追加** をタップします
3. **タイトル**（例：「翻訳者」「コードレビュアー」「中国語アシスタント」）を入力します
4. 3つのセクションを記入します：
    - **テキストを追加** をタップして静的コンテンツを記述
    - **変数を追加** をタップして動的な値を挿入
5. **保存** をタップします

### 項目の並べ替え

各セクション内で、複数のテキストブロックと変数を持つことができます。項目を長押しすると：

- **上に移動** / **下に移動** — セクション内で並べ替え
- **削除** — 項目を削除

## 変数置換

変数はメッセージ送信時に動的な値に置き換えられます：

| 変数 | 展開先 | 例 | 解決タイミング |
|----------|-----------|---------|---------------|
| `{time}` | 現在時刻（HH:mm:ss） | `14:30:00` | プロンプトコンパイル時 |
| `{date}` | 現在の日付（YYYY-MM-DD） | `2026-05-10` | プロンプトコンパイル時 |
| `{sent_time}` | メッセージ送信時刻（HH:mm） | `10:05` | メッセージごと |
| `{sent_date}` | メッセージ送信日（YYYY-MM-DD） | `2026-05-11` | メッセージごと |
| `{active_memory}` | アクティブメモリの内容 | `[保存されたメモリ内容]` | プロンプトコンパイル時 |
| `{model_id}` | 現在選択されているモデル ID | `gemini-1.5-flash` | プロンプトコンパイル時 |

**メッセージごとの変数**（`{sent_time}`、`{sent_date}`）はメッセージを送信するたびに解決されるため、正確な送信時刻を反映します。**プロンプトレベルの変数**（`{time}`、`{date}`、`{active_memory}`、`{model_id}`）はシステムプロンプトのコンパイル時に解決されます。

!!! tip
    「今日は {sent_date} です。最近の出来事について議論する際は、あなたの知識が古い可能性があることに注意してください。」のような日付依存のプロンプトには `{sent_date}` を使用します。モデルの永続メモリをシステム命令に注入するには `{active_memory}` を使用します。

### 変数の追加

1. エディタの任意のセクションで **変数を追加** をタップします
2. ピッカーから変数を選択します
3. セクション内にピル/チップとして表示されます — ドラッグして再配置できます

## プロンプトの管理

### デフォルトとして設定

プロンプトの横のラジオボタンをタップして **グローバルデフォルト** にします。上書きされない限り、すべての会話でこのプロンプトが使用されます。

### 会話ごとの上書き

各会話で異なるシステムプロンプトを使用できます：

1. 会話を開きます
2. トップバーのオーバーフローメニュー（:material-dots-vertical:）をタップします
3. **会話プロンプト** を選択します
4. リストからプロンプトを選択します

会話ごとの設定は、その会話に対してのみグローバルデフォルトを上書きします。

### 編集または削除

- プロンプトをタップして **編集** します
- 長押しして **削除** を選択して削除します

!!! warning
    システムプロンプトの削除は永続的です。それを使用していた会話はグローバルデフォルトにフォールバックします。

---

## システムプロンプトなし

システムプロンプトが選択されていない場合、モデルは特別な指示を受け取らず — ベーストレーニングに従って動作します。これはテストや、システム命令なしの方がパフォーマンスが良いモデルにとって望ましい場合があります。

プロンプトなしを使用するには、プロンプトリストから **なし** を選択します。

---

## 自動タイトル生成

Agora は最初の応答後に会話タイトルを自動生成できます：

1. **設定 → タイトル生成** に移動します
2. **タイトルを自動生成** を有効にします
3. **タイトルモデル** を選択します：
    - **現在のモデルを使用** — 会話でアクティブなモデルを使用
    - **タイトルモデルを選択** — タイトル生成用に特定の高速/低コストモデルを選択

有効にすると、最初のモデル応答後に「タイトルを生成中...」という簡単なスナックバーが表示され、会話は「無題」から説明的なタイトルに自動的に名前変更されます。

## プロンプトの例

### 翻訳者

```yaml
システムプロンプト: |
  あなたはプロの翻訳者です。ユーザー入力を英語に翻訳してください。
  書式、コードブロック、技術用語は保持してください。説明は追加しないでください。
```

### コードレビュアー

```yaml
システムプロンプト: |
  あなたはシニアコードレビュアーです。コードが提示された場合：
  1. バグとエッジケースを特定する
  2. パフォーマンス改善を提案する
  3. セキュリティ問題をチェックする
  具体的に。可能な限り行番号を参照してください。
```

### 中国語アシスタント

```yaml
システムプロンプト: |
  你是一个乐于助人的中文助手。用简洁、清晰的中文回答问题。
ユーザーサフィックス: |
  \n\n请用中文回答。
```
