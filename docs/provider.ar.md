# مزودو API

يتصل Agora مباشرة بمزودي الذكاء الاصطناعي — بدون وسيط، بدون اشتراك، بدون تتبع. أنت تحضر مفاتيح API الخاصة بك، وكل شيء يعمل من جهازك.

## المزودون المدمجون

| المزود | URL الأساسي | النماذج | ملاحظات |
|----------|----------|--------|-------|
| **Google** | `https://generativelanguage.googleapis.com/v1beta` | سلسلة Gemini | فئة مجانية متاحة عبر Google AI Studio |
| **OpenAI** | `https://api.openai.com/v1` | GPT-4، GPT-4o، سلسلة o | نماذج الاستدلال مدعومة |
| **Anthropic** | `https://api.anthropic.com/v1` | سلسلة Claude | التفكير الموسع مدعوم |
| **DeepSeek** | `https://api.deepseek.com/v1` | DeepSeek-V3، DeepSeek-R1 | نماذج الاستدلال مدعومة |
| **Qwen** | `https://dashscope-intl.aliyuncs.com/compatible-mode/v1` | سلسلة Qwen | عبر Alibaba DashScope |
| **Ollama** | `http://localhost:11434/v1` | أي نموذج مسحوب | مستضاف ذاتيًا، لا حاجة لمفتاح API |
| **OpenRouter** | `https://openrouter.ai/api/v1` | متعدد المزودين | الوصول إلى نماذج متعددة عبر API واحد |
| **محلي** | لا ينطبق | نماذج GGUF | على الجهاز عبر llama.cpp، دون اتصال كامل |

## تبديل المزودين

اضغط محدد المزود في الإعدادات للتبديل بين المزودين. كل مزود يحتفظ بـ:

- مفاتيح API
- URL الأساسي (قابل للتعديل للبروكسيات/المستضاف ذاتيًا)
- قائمة النماذج

## مفاتيح API

### مفاتيح متعددة لكل مزود

كل مزود يدعم عدة مفاتيح API مسماة. هذا يتيح:

- **التدوير** — التبديل بين المفاتيح لمستويات استخدام مختلفة
- **التنظيم** — فصل الاستخدام العملي والشخصي
- **الاحتياط** — الاحتفاظ بمفتاح احتياطي جاهز

### إدارة المفاتيح

1. اذهب إلى **الإعدادات → المزود**
2. اختر مزودًا
3. تحت **مفاتيح API**، اضغط **إضافة مفتاح جديد**
4. أدخل **اسمًا** (مثل "عمل"، "شخصي"، "مشترك للفريق") و**قيمة المفتاح**
5. اضغط **إضافة**

اضغط زر الاختيار لتعيين المفتاح النشط. اضغط مطولاً على مفتاح **لتعديل** أو **حذف**.

### أمان المفاتيح

!!! warning
    مفاتيح API مخزنة محليًا في قاعدة بيانات Room مشفرة. لا تُرسل أبدًا إلى خوادم Agora (لا توجد خوادم). ومع ذلك، تُصدر بنص صريح إذا ضمنتها في ملف تصدير `.agora`.

## المزودون المخصصون

أضف أي نقطة نهاية API متوافقة مع OpenAI:

1. اذهب إلى **الإعدادات → المزود**
2. اضغط **+ إضافة مزود مخصص** في أسفل قائمة المزودين
3. أدخل:
    - **اسم المزود** — أي اسم عرض
    - **URL الأساسي** — نقطة نهاية API
4. اضغط **إضافة**

يجلب Agora قائمة النماذج من `{base_url}/v1/models`. بمجرد الإضافة، يعمل المزودون المخصصون تمامًا مثل المدمجين: أضف مفاتيح API، زامن النماذج، وتحدث.

### حالات الاستخدام

- **مستضاف ذاتيًا** — الاتصال بـ vLLM، LocalAI، text-generation-webui، أو خوادم أخرى متوافقة مع OpenAI
- **البروكسيات** — التوجيه عبر بروكسي مؤسسي أو بوابة API
- **نقاط نهاية بديلة** — استخدام Azure OpenAI، Cloudflare AI Gateway، أو خدمات متوافقة أخرى

### إعادة التسمية أو الحذف

اضغط مطولاً على مزود مخصص **لإعادة التسمية** أو **الحذف**. الحذف يزيل المزود وكل مفاتيحه.

!!! warning
    لا يمكن إعادة تسمية أو حذف المزودين المدمجين.

---

## تجاوز URL الأساسي

كل مزود (بما في ذلك المدمجين) لديه **URL أساسي** قابل للتعديل. هذا مفيد لـ:

- **البروكسيات**: التوجيه عبر `https://my-proxy.example.com/v1`
- **المستضاف ذاتيًا**: التوجيه إلى نسختك الخاصة
- **توجيه المناطق**: استخدام نقاط نهاية خاصة بالمنطقة

---

## مزامنة النماذج

بعد إضافة مفاتيح API، زامن قائمة النماذج:

1. اذهب إلى **الإعدادات → النماذج**
2. اضغط **مزامنة من جميع المزودين**
3. يجلب Agora النماذج المتاحة من كل مزود مضبوط

شريط إشعار يظهر تقدم المزامنة والنتائج. يمكنك بعد ذلك تفعيل/تعطيل النماذج الفردية وتعيين افتراضي.

## ملاحظات خاصة بالمزودين

### Google Gemini

- مفاتيح API من [Google AI Studio](https://aistudio.google.com/apikey)
- فئة مجانية متاحة مع حدود معدل
- يدعم تنفيذ الكود والتأريض البحثي (أدوات مدمجة)

### OpenAI

- مفاتيح API من [المنصة](https://platform.openai.com/api-keys)
- نماذج الاستدلال (o1, o3) تتطلب وصول API محدد
- البث، الأدوات، والرؤية كلها مدعومة

### Anthropic

- مفاتيح API من [Console](https://console.anthropic.com/)
- تفكير موسع مع ميزانيات رموز قابلة للتكوين
- استخدام الأدوات مع استدعاءات متوازية مدعوم

### Ollama

- لا حاجة لمفتاح API (شبكة محلية)
- URL الأساسي عادة `http://<host>:11434/v1`
- قائمة النماذج مجلوبة من API الخاص بـ Ollama
- راجع [الأسئلة الشائعة](faq.md) لاستكشاف مشاكل Ollama

### OpenRouter

- مفتاح API واحد لأكثر من 200 نموذج
- تسعير الدفع لكل رمز يختلف حسب النموذج
- جيد لتجربة نماذج مختلفة بدون حسابات مزودين فردية

### محلي (llama.cpp)

- لا حاجة للشبكة
- ملفات نماذج GGUF مخزنة على الجهاز
- راجع [النماذج المحلية](local-model.md) للإعداد
