# Песочница

Agora может запускать легковесное окружение Alpine Linux локально на вашем устройстве — подключение к интернету не требуется. Песочница позволяет модели устанавливать пакеты и выполнять команды в изолированной корневой файловой системе.

!!! note "Доступность"
    Песочница доступна во всех сборках. Доступ к ней осуществляется из **Настройки → Оболочка → Управление песочницей** или напрямую через **Настройки → Песочница**.

## Как это работает

Песочница использует корневую файловую систему Alpine Linux, развёрнутую в приватном хранилище приложения на устройстве. Минимальный менеджер пакетов на основе `apk` позволяет устанавливать программное обеспечение в это окружение, а команды выполняются внутри контейнера на основе proot.

Это **не** полноценная виртуальная машина — это легковесный контейнер в пользовательском пространстве, использующий ядро хоста. Он обеспечивает достаточную изоляцию для безопасного экспериментирования при низком потреблении ресурсов.

## Влияние VPN — Критически важно

!!! danger "Отключите VPN перед использованием сети песочницы"

    VPN-приложения мешают разрешению DNS в proot. Вот почему:

    **Корневая причина — у PRoot нет изоляции сетевых пространств имён.**

    PRoot использует `ptrace` для перехвата системных вызовов и перенаправления путей к файлам, но он **не** поддерживает `CLONE_NEWNET` (сетевые пространства имён Linux). Все процессы внутри песочницы напрямую разделяют сетевой стек хостовой системы Android. Нет виртуального сетевого интерфейса, нет изолированной таблицы маршрутизации и нет независимой конфигурации DNS.

    **Как VPN на Android ломает DNS внутри proot:**

    1. VPN-приложения Android используют API `VpnService`, который создаёт **TUN-интерфейс** — виртуальное сетевое устройство, перехватывающее **весь** трафик устройства, включая трафик изнутри proot
    2. Для предотвращения утечек DNS за пределы зашифрованного туннеля VPN **перенаправляет весь трафик порта 53 (DNS)** на свои собственные DNS-серверы
    3. Внутри proot, когда приложение вызывает `getaddrinfo()` (стандартный DNS-резолвер libc), запрос проходит через системный резолвер Android — который VPN уже перехватил
    4. На Android 12+ Google переработал DNS-резолвер, сделав `getaddrinfo()` внутри окружений proot особенно хрупким ([termux/proot#215](https://github.com/termux/proot/issues/215))
    5. Маршрутизация TUN VPN и путь DNS системного резолвера конфликтуют внутри proot: резолвер отправляет DNS-запрос, TUN VPN перехватывает его, но ответ никогда не доходит обратно через слой `ptrace` proot

    **Наблюдаемые симптомы:**

    | Операция | Результат |
    |-----------|--------|
    | `ping 1.1.1.1` | ✅ Работает (прямой IP, DNS не нужен) |
    | `ping google.com` | ❌ Ошибка — «Временная ошибка разрешения имени» |
    | `apk add python3` | ❌ Ошибка — не может разрешить `dl-cdn.alpinelinux.org` |
    | `curl https://example.com` | ❌ Ошибка — ошибка разрешения имени |
    | `curl https://1.1.1.1` | ✅ Работает (прямое IP-соединение) |

    **Исправление:** Полностью отключите VPN перед выполнением любых сетевых операций в песочнице (установка пакетов, `curl`, `wget` и т.д.). Вы можете снова включить VPN после завершения сетевых операций.

    Это фундаментальное ограничение архитектуры proot — он не может виртуализировать сетевой стек, когда Android VPN переопределяет DNS-маршрутизацию системы через TUN-интерфейс.

## Настройка

### Установка корневой файловой системы

При первом открытии песочницы вы увидите панель, указывающую, что rootfs не установлена. Нажмите **Установить**, чтобы скачать и распаковать корневую файловую систему Alpine.

!!! info "Использование хранилища"
    Базовая rootfs занимает примерно 100–200 МБ. Установленные пакеты потребляют дополнительное пространство. Общее использование диска отображается на панели.

---

## Управление пакетами

### Установка пакета

1. Введите имя пакета в текстовом поле (например, `python3`)
2. Нажмите **Установить**
3. Следите за выводом терминала для отслеживания прогресса установки

Также можно нажать любой **чип быстрой установки** для распространённых пакетов:

```
python3   git      curl      wget
openssh   nodejs   build-base   htop
```

### Установленные пакеты

Под секцией установки все установленные пакеты перечислены с:

- **Именем** — имя пакета Alpine
- **Версией** — установленная версия
- **Описанием** — краткое резюме (обрезано)

### Удаление пакета

Нажмите значок :material-close: на любом установленном пакете, чтобы удалить его. Перед удалением появляется диалог подтверждения.

---

## Панель управления

Когда песочница готова, панель показывает:

- **Использование диска** — индикатор выполнения и числовое отображение (МБ или ГБ)
- **Количество установленных** — общее количество пакетов

---

## Вывод терминала

При установке или удалении пакетов вывод терминала отображается в тёмном, прокручиваемом моноширинном представлении под полем ввода. Вывод автоматически прокручивается, следуя за последними строками.

Используйте это для:
- Отслеживания прогресса установки
- Отладки неудачных операций с пакетами
- Просмотра файлов, устанавливаемых пакетом

---

## Сброс песочницы

**Опасная зона** внизу содержит опцию **Сбросить песочницу**. Это полностью удаляет корневую файловую систему и все установленные пакеты.

!!! danger "Разрушительное действие"
    Сброс песочницы удаляет всё окружение Alpine. Вам нужно будет переустановить rootfs и все пакеты заново. Диалог подтверждения предотвращает случайные сбросы.
