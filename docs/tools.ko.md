# 에이전트 도구

Agora의 모델은 도구를 자율적으로 사용할 수 있습니다 — 사용자가 각 작업을 수동으로 트리거할 필요 없이 무엇을 검색, 실행, 읽기, 기억할지 스스로 결정합니다. 도구는 **다중 라운드 루프**로 작동합니다: 모델이 도구를 호출하고, 결과를 읽은 다음, 다른 도구를 호출하거나 응답할지 결정할 수 있습니다.

## 도구 호출 작동 방식

1. 메시지 전송
2. 모델이 외부 정보나 작업이 필요하다고 판단
3. **도구 호출** 발행 — 도구 이름과 인자가 포함된 구조화된 요청
4. Agora가 기기 내 또는 원격 서버에서 도구 실행
5. 결과가 모델에 피드백됨
6. 모델이 다른 도구를 호출하거나 최종 응답 생성 가능

이 루프는 단일 메시지 턴 내에서 여러 번 반복될 수 있습니다.

## 사용 가능한 도구

### 웹 검색

인터넷 검색 및 웹 페이지 가져오기. 모델이 최신 정보를 조회하거나, 사실을 확인하거나, 문서를 가져올 수 있습니다.

- **프로바이더**: Brave, Serper, Tavily, SearXNG
- **구성**: 설정 → 웹 검색
- **가이드**: [웹 검색](web-search.ko.md)

### 코드 실행

샌드박스 환경에서 코드 스니펫 실행 (프로바이더 의존). 계산, 데이터 분석, 인라인 로직 테스트에 유용합니다.

- **프로바이더**: Google Gemini (내장 코드 실행)
- **구성**: 설정 불필요 — Gemini 모델에 자동 활성화

### 원격 셸

[Conch](https://github.com/newo-ether/conch) 프로토콜을 통해 원격 머신에서 명령 실행. 모델이 서버 상태를 확인하거나, 파일을 관리하거나, 스크립트를 실행할 수 있습니다.

- **프로토콜**: 종단 간 암호화 (ECDH + AES-256-GCM)
- **구성**: 설정 → 셸
- **가이드**: [원격 셸](shell.ko.md)

### 파일 작업

Conch 프로토콜을 통해 원격 기기에서 파일 읽기, 쓰기, 편집, Glob 검색, grep 검색. 모델이 원격 파일 시스템을 직접 조작할 수 있습니다.

!!! note
    파일 작업은 구성된 Conch 셸 기기가 필요합니다. 설정은 [원격 셸](shell.ko.md)을 참조하세요.

### 메모리

대화 간에 걸친 영구 지식 저장소:

- **활성 메모리** — 모든 API 호출에 항상 포함됨. 모델이 항상 기억해야 할 사실, 선호 사항, 컨텍스트에 사용.
- **저장된 메모리** — 모델이 도구 호출을 통해 검색, 읽기, 쓰기, 편집할 수 있는 이름 지정된 메모리 파일 모음.

자세한 내용은 [메모리 & 캐시](memory.ko.md)를 참조하세요.

### 대화 검색

모델이 키워드 또는 시맨틱(RAG) 방법을 사용하여 과거 대화 기록을 검색할 수 있습니다. 사용자가 수동으로 찾아 공유할 필요 없이 이전 토론을 참조할 수 있습니다.

설정은 [대화 검색](search.ko.md)을 참조하세요.

## 채팅에서의 도구 UI

도구가 호출되면 대화에서 인라인으로 표시됩니다:

<div class="grid cards" markdown>

- **:material-progress-wrench: 도구 호출 배너**

    ---

    도구 이름과 간략한 상태 표시 (예: :material-magnify: "웹에서 '최신 AI 뉴스' 검색 중").

- **:material-check-circle: 도구 결과**

    ---

    실행 후, 형식화된 결과나 요약 표시 (예: "'최신 AI 뉴스'에 대한 5개 결과 찾음").

</div>

### 확장 가능한 세부 정보

도구 호출을 탭하여 확장하고 다음을 확인:

- **인자** — 도구에 전송된 정확한 매개변수
- **결과** — 도구 실행의 원시 출력
- **상태** — 성공, 실패, 또는 부분 결과

### 실패한 호출

도구 호출이 실패하면, 모델이 오류를 통보받고 재시도하거나 조정할 수 있습니다. 오류 메시지와 함께 빨간색 배너가 표시됩니다.

## 도구 권한

모델이 접근할 수 있는 도구를 제어합니다:

| 설정 | 위치 | 기본값 |
|---------|----------|---------|
| 웹 검색 | 설정 → 웹 검색 | 꺼짐 |
| 셸 | 설정 → 셸 | 꺼짐 |
| 메모리 (저장) | 설정 → 메모리 → 저장된 메모리 접근 | 꺼짐 |
| 메모리 (활성) | 설정 → 메모리 → 활성 메모리 접근 | 꺼짐 |
| 과거 대화 | 설정 → 메모리 → 과거 대화 접근 | 꺼짐 |
| 대화 검색 | 설정 → 대화 검색 | 켜짐* |

*모델의 대화 검색 능력은 임베딩 모델이 구성되어 있는지에 따라 달라집니다. 없으면 키워드 검색만 가능합니다.

## 다중 라운드 도구 루프

모델은 여러 도구 호출을 연결할 수 있습니다. 예를 들어:

1. 사용자: "최신 Linux 커널 버전은 무엇이고 내 서버가 이를 실행 중인가요?"
2. 모델이 `web_search("최신 Linux 커널 버전")` 호출
3. 모델이 `shell_execute("uname -r", device="내-서버")` 호출
4. 모델이 결과를 비교하고 응답

각 도구 호출과 결과는 최종 텍스트 응답 전에 대화에서 별도의 인라인 항목으로 표시됩니다.
