# 埋め込み / RAG

埋め込みモデルはテキストを意味を捉えた数値ベクトルに変換します。Agora はこれらのベクトルを会話履歴に対するセマンティック検索（RAG）に使用します — 含まれる単語だけでなく、意味によってメッセージを見つけます。

## 仕組み

1. 各メッセージが埋め込みモデルに送信されます
2. モデルはメッセージの意味を表すベクトル（数値のリスト）を返します
3. 検索時には、クエリも埋め込まれます
4. Agora はクエリベクトルとすべてのメッセージベクトルの間の **コサイン類似度** を計算します
5. 閾値を超える類似度を持つメッセージが一致として返されます

## 対応プロバイダー

| プロバイダー | ベース URL | API キー要否 | 備考 |
|----------|----------|------------------|-------|
| **OpenAI** | `https://api.openai.com/v1` | 要 | `text-embedding-3-small`、`text-embedding-3-large` |
| **Mistral** | `https://api.mistral.ai/v1` | 要 | `mistral-embed` |
| **Voyage AI** | `https://api.voyageai.com/v1` | 要 | `voyage-3`、`voyage-3-lite` |
| **SiliconFlow** | `https://api.siliconflow.cn/v1` | 要 | `BAAI/bge-large-zh-v1.5`（中国語最適化） |
| **Ollama** | `http://localhost:11434/v1` | 不要 | `qwen3-embedding`、`nomic-embed-text` など |
| **カスタム** | 任意 | オプション | 任意の OpenAI 互換埋め込みエンドポイント |
| **ローカル** | N/A | 不要 | llama.cpp 経由の GGUF 埋め込みモデル |

## 埋め込みモデルの追加

### リモート（API）

1. **設定 → 会話検索** に移動します
2. **リモートモデルを追加** をタップします
3. 以下を設定します：

| 項目 | 説明 |
|-------|-------------|
| **プロバイダー** | ドロップダウンから選択（OpenAI、Mistral、Voyage、SiliconFlow、Ollama、カスタム） |
| **モデル名** | 正確なモデル ID（例：`text-embedding-3-small`） |
| **ベース URL** | 既知のプロバイダーは自動入力、プロキシ用に編集可能 |
| **API キー** | 空白のままにしてチャットプロバイダーキーから自動解決、または専用キーを入力 |
| **バッチサイズ** | API リクエストあたりの埋め込みメッセージ数（1～100） |

4. **追加** をタップ — 保存前に接続テストが実行されます

!!! tip
    チャット用に同じプロバイダーを既に設定している場合、API キーフィールドはオプションです。空白のままにすると、Agora がチャット API キーを自動的に解決します。

### ローカル（GGUF）

1. **設定 → 会話検索** に移動します
2. **ローカルモデルを追加** をタップします
3. `.gguf` 埋め込みモデルファイルをインポートします（例：`bge-small-en-v1.5-q4_k.gguf`）
4. 名前を付けます
5. **追加** をタップします

埋め込みモデルは通常チャットモデルよりもはるかに小さく、せいぜい数百 MB です。

### Ollama

1. マシンに Ollama をインストールします
2. 埋め込みモデルをプルします：`ollama pull qwen3-embedding:8b`
3. Agora で、リモートモデルを追加します：
    - プロバイダー: **Ollama**
    - ベース URL: `http://<host>:11434/v1`
    - モデル名: `qwen3-embedding:8b`（`:tag` を含める）
    - API キー: 空白のまま
4. **追加** をタップします

!!! note
    `:8b`、`:latest` などの Ollama サフィックスタグはモデル名の一部です。`ollama list` の正確な名前を使用してください。

## キャッシュ

モデルを追加した後、メッセージをキャッシュ（埋め込みを生成）する必要があります：

1. 埋め込みモデルで **キャッシュ** をタップします
2. Agora はすべての未キャッシュメッセージをバッチで処理します
3. 円形の進捗インジケーターが現在の進捗を表示します
4. 完了: 「すべての N 件のメッセージがキャッシュされました」

### 自動キャッシュ

**自動キャッシュ** を有効にして、新しいメッセージが到着すると自動的に埋め込みます。これにより検索インデックスが常に最新に保たれます。

### 再キャッシュ

**再キャッシュ** をタップして、既存のすべての埋め込みを削除し、ゼロから再構築します。以下の場合に使用します：

- 異なる埋め込みモデルに切り替える場合
- 埋め込み品質が低下していると思われる場合
- キャッシュに不整合がある場合

!!! warning
    再キャッシュは元に戻せず、大規模なメッセージ履歴では長時間かかることがあります。

---

## バッチサイズ

**バッチサイズ** 設定（1～100）は、キャッシュ中に API リクエストごとに送信されるメッセージ数を制御します：

- **高い**: 高速なキャッシュだが、API ペイロードが大きい
- **低い**: 小さなリクエスト、遅いが遅い接続でより信頼性が高い

デフォルトから始めて、タイムアウトが発生する場合は下げ、より高速なキャッシュが必要な場合は上げます。

---

## セットアップのテスト

リモートモデルを追加すると、Agora は自動接続テストを実行します。失敗した場合：

1. モデル名を確認 — Ollama のタグ（`:8b`、`:latest`）を含める
2. ベース URL がデバイスから到達可能であることを確認
3. API キーが有効であることを確認（必要な場合）
4. そのプロバイダーの既知のモデル名を試す

よくあるエラー：
- **「モデル名が間違っています」** — タグを含めて正確なスペルを確認
- **「ベース URL が間違っています」** — エンドポイントが `/v1/embeddings` をサポートしていることを確認
- **「API キーがありません」** — 一部のプロバイダーは認証が必要
- **「ネットワークエラー」** — 接続を確認

## プロバイダーの推奨

| ユースケース | 推奨プロバイダー |
|----------|---------------------|
| **最高品質（英語）** | Voyage AI `voyage-3` |
| **最高品質（中国語）** | SiliconFlow `BAAI/bge-large-zh-v1.5` |
| **無料 / セルフホスト** | Ollama `qwen3-embedding` または `nomic-embed-text` |
| **完全オフライン** | ローカル GGUF `bge-small-en-v1.5` |
| **既に OpenAI を使用中** | OpenAI `text-embedding-3-small`（安価、高速） |
