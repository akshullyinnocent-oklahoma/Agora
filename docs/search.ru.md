# Поиск по диалогам

Agora может искать по всей вашей истории диалогов — либо по совпадению ключевых слов, либо с помощью семантического (смыслового) поиска с использованием моделей эмбеддингов.

## Методы поиска

### Поиск по ключевым словам

Быстрое, точное совпадение текста. Поиск ищет буквальные вхождения вашего запроса в содержимом сообщений.

**Лучше всего для:**
- Поиска конкретной фразы или термина
- Быстрых поисков, когда вы помните точную формулировку
- Нулевой настройки — работает сразу

**Ограничения:**
- Пропускает синонимы и связанные понятия
- Нет понимания смысла

### Семантический поиск (RAG)

Использует модели эмбеддингов для поиска сообщений по **смыслу**, а не по точным словам. Запрос «как настроить базу данных» может найти сообщения о «конфигурации Room», даже если слово «база данных» нигде не встречается.

**Лучше всего для:**
- Поиска диалогов по теме
- Широких запросов, где вы не помните точную формулировку
- Обнаружения связанных обсуждений в разных диалогах

**Требования:**
- Должна быть настроена модель эмбеддингов (см. [Эмбеддинги / RAG](embedding.ru.md))
- Сообщения должны быть закэшированы (эмбеддинги сгенерированы)

## Настройка

### 1. Добавление модели эмбеддингов

См. [Эмбеддинги / RAG](embedding.ru.md) для подробной настройки. Вы можете использовать:
- **Удалённые модели** (OpenAI, Mistral, Voyage, Ollama и др.)
- **Локальные модели** (GGUF файлы, полностью автономно)

### 2. Выбор методов поиска

В **Настройки → Поиск по диалогам**:

| Настройка | Описание |
|---------|-------------|
| **Метод поиска модели** | Как модель ищет, когда вызывает инструмент `search_conversations` |
| **Метод ручного поиска** | Как работает строка поиска в панели диалогов |

Установите каждый в **Ключевые слова** или **Семантический (RAG)**.

### 3. Настройка области поиска

| Настройка | Диапазон | Описание |
|---------|-------|-------------|
| **Контекстных сообщений на результат** | 4–32 | Сколько окружающих сообщений включать с каждым совпадением (шаги: 4, 8, 12, 16, 20, 24, 28, 32) |
| **Макс. результатов поиска** | 5–30 | Максимальное количество возвращаемых совпадений (шаги: 5, 10, 15, 20, 25, 30) |
| **Порог сходства** | 0.0–1.0 | Только RAG: минимальная оценка сходства для совпадения. Выше = строже. По умолчанию: 0.5 |

### 4. Кэширование сообщений

При использовании RAG нажмите **Кэшировать**, чтобы сгенерировать эмбеддинги для всех существующих сообщений. Включите **Авто-кэш**, чтобы индекс обновлялся автоматически.

## Использование поиска

### Ручной поиск (строка поиска)

1. Откройте **панель диалогов** (меню-гамбургер :material-menu: или проведите вправо)
2. Нажмите строку поиска вверху
3. Введите ваш запрос
4. Результаты появятся ниже — нажмите любой результат, чтобы открыть этот диалог на совпавшем сообщении

### Поиск, инициированный моделью

Когда включён **Доступ к прошлым диалогам** (Настройки → Память), модель может автономно искать в вашей истории:

```text
Вы: «Что мы решили насчёт дизайна API на прошлой неделе?»
Модель: [Ищет «решение по дизайну API»]
       «В прошлый вторник мы решили использовать...»
```

Поиск отображается как карточка инструмента в диалоге.

## Порог сходства

Ползунок **Порог сходства** (0.0 до 1.0) управляет тем, насколько близко сообщение должно совпадать, чтобы быть включённым в результаты RAG:

- **Низкий (0.3–0.5)**: Больше результатов, может включать слабо связанное содержимое
- **Средний (0.5–0.7)**: Сбалансированный — хорошее значение по умолчанию
- **Высокий (0.7–0.9)**: Меньше результатов, только очень близкие совпадения

Начните со значения по умолчанию и корректируйте на основе результатов. Если вы получаете слишком много нерелевантных совпадений, повысьте порог. Если пропускаете релевантные диалоги, понизьте его.

## Отображение результатов поиска

В панели диалогов результаты поиска показывают:

- **Заголовок диалога** (или «Без названия»)
- **Совпавшее сообщение** — сообщение пользователя или модели, которое совпало
- **Метка роли** — Пользователь или Модель
- **Контекстные сообщения** — окружающие сообщения для контекста

Нажмите результат, чтобы открыть диалог, прокрученный до совпавшего сообщения.
