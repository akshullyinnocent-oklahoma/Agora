# ローカルモデル

GGUF モデルファイルと llama.cpp を使用して、Android デバイス上で直接 LLM を実行します。ネットワーク不要、API キー不要、完全にプライベートです。

## 仕組み

Agora は Android NDK（CMake）経由で llama.cpp をバンドルしています。GGUF ファイルをインポートすると、モデルは完全にデバイスの CPU 上で実行され — データがデバイスから出ることはありません。

## 要件

- **GGUF 形式** モデルのみ（llama.cpp の標準形式）
- **デバイスメモリ**: モデルは利用可能な RAM に収まる必要があります。目安として：
    - 1～3B パラメータモデル: 4～6 GB RAM
    - 7～8B パラメータモデル: 6～8 GB RAM
- **ストレージ**: GGUF ファイルのサイズは ~500 MB（量子化された小モデル）から 5+ GB まで

!!! warning
    ローカル推論は CPU 集約的で、クラウド API よりもはるかに遅くなります。オフライン使用、プライバシー重視のコンテンツ、実験に最適です — 高速で大量のチャットには向いていません。

---

## チャットモデルのインポート

1. GGUF モデルファイルをデバイスにダウンロードします（下記の推奨ソースを参照）
2. **設定 → プロバイダー** に移動します
3. プロバイダーとして **ローカル** を選択します
4. **GGUF モデルをインポート** をタップします
5. デバイスから `.gguf` ファイルを選択します
6. モデルを設定します：

| パラメータ | 説明 | 例 |
|-----------|-------------|---------|
| **モデル ID** | 小文字の識別子、スペースなし | `qwen3-8b` |
| **エイリアス** | 表示名 | `Qwen 3 8B` |
| **コンテキストサイズ** | トークン単位の最大コンテキストウィンドウ | `4096` |
| **Temperature** | ランダム性（0.0～2.0） | `0.7` |
| **Top P** | Nucleus sampling 閾値（0.0～1.0） | `0.9` |
| **最大トークン数** | 最大生成長 | `2048` |

7. **追加** をタップします

モデルがインポートされ、すぐに使用可能になります。

## 埋め込みモデルのインポート

埋め込みモデルはより小さく、セマンティック検索に使用されます：

1. **設定 → 会話検索** に移動します
2. **ローカルモデルを追加** をタップします
3. `.gguf` 埋め込みモデルファイルを選択します
4. 名前を付けます
5. **追加** をタップします

検索のセットアップについては [埋め込み / RAG](embedding.md) を参照してください。

---

## アクティブモデルの選択

1つ以上のモデルをインポートした後：

1. **設定 → プロバイダー → ローカル** に移動します
2. インポートされたすべてのモデルが表示されます
3. 使用したいモデルの横の **ラジオボタン** をタップします
4. チャットプロバイダーとして **ローカル** が選択されている場合、選択したモデルがアクティブになります

---

## ローカルモデルの管理

### 名前変更

モデルをタップしてエイリアスを変更したり、パラメータ（temperature、コンテキストサイズなど）を調整します。

### 削除

モデルを長押しして **削除** をタップします。これにより Agora からモデルが削除され、ストレージから GGUF ファイルが削除されます。

## 推奨モデル

### チャットモデル

| モデル | サイズ | 必要な RAM | 備考 |
|-------|------|------------|-------|
| Qwen 3 1.7B | ~1 GB | 3～4 GB | サイズの割に良好な品質 |
| Llama 3.2 3B | ~2 GB | 4～5 GB | 堅実なオールラウンダー |
| Qwen 3 8B | ~5 GB | 7～8 GB | 最高品質、高 RAM |

### 埋め込みモデル

| モデル | サイズ | 備考 |
|-------|------|-------|
| BGE Small EN v1.5 | ~130 MB | 良好な英語埋め込み |
| BGE Small ZH v1.5 | ~130 MB | 中国語最適化 |
| Nomic Embed Text v1.5 | ~270 MB | 良好な多言語対応 |

### GGUF ファイルの入手先

- [Hugging Face](https://huggingface.co/models?library=gguf) — "GGUF" で検索
- [bartowski の量子化モデル](https://huggingface.co/bartowski) — 幅広い選択肢、よく整理されています

!!! tip
    チャットモデルには `Q4_K_M` 量子化を探してください — 品質とサイズの最適なトレードオフを提供します。

---

## パフォーマンスのヒント

- **小さいコンテキスト = 高速**: 2048 から始めて、必要な場合のみ増やします
- **低い量子化 = 高速**: Q4_K_M は Q6_K や Q8 よりも高速です
- **他のアプリを閉じる**: ローカル推論はできるだけ多くの RAM を必要とします
- **電源に接続**: 推論は CPU 集約的で、長時間の使用はバッテリーを消耗します
