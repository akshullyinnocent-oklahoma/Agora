# Часто задаваемые вопросы

## API и Провайдеры

### Как получить API-ключ?

- **Google Gemini**: [Google AI Studio](https://aistudio.google.com/apikey) — доступен бесплатный тариф
- **OpenAI**: [Platform API Keys](https://platform.openai.com/api-keys)
- **Anthropic**: [Console API Keys](https://console.anthropic.com/)
- **DeepSeek**: [Platform](https://platform.deepseek.com/)
- **OpenRouter**: [Keys page](https://openrouter.ai/keys)
- **Brave Search**: [Brave Search API](https://api.search.brave.com/)

### Можно ли использовать несколько API-ключей для одного провайдера?

Да. Каждый провайдер поддерживает несколько именованных ключей. Нажмите радиокнопку, чтобы выбрать активный ключ. Полезно для переключения между рабочими и личными ключами или для наличия резервного ключа. См. [API Провайдеры](provider.ru.md#api-keys).

### Как добавить пользовательского провайдера?

Перейдите в Настройки → Провайдер → **+ Добавить пользовательского провайдера**. Введите имя и базовый URL. Подходит любая конечная точка, совместимая с OpenAI. См. [Пользовательские провайдеры](provider.ru.md#custom-providers).

## Локальные модели

### Какие GGUF модели работают?

Agora поддерживает формат GGUF как для чата, так и для эмбеддингов. Чат-модели должны помещаться в памяти устройства (1–8B параметров в зависимости от ОЗУ). Модели эмбеддингов значительно меньше (100–500 МБ). См. [Локальные модели](local-model.ru.md).

### Как запускать модели офлайн?

Импортируйте GGUF чат-модель через Настройки → Провайдер → Локальный → **Импортировать GGUF модель**. Для полностью автономного семантического поиска также импортируйте GGUF модель эмбеддингов. Сетевое подключение не требуется.

### Почему моя локальная модель работает так медленно?

Локальный инференс выполняется на процессоре вашего устройства. Это inherently медленнее, чем облачные API. Советы: используйте меньшие модели (1–3B параметров), более низкую квантизацию (Q4_K_M), более короткие контекстные окна и закрывайте фоновые приложения.

## Эмбеддинги и Поиск

### Почему тест модели эмбеддингов не проходит?

Распространённые причины:

- **Неверное имя модели** — проверьте точное написание, включая теги Ollama (например, `qwen3-embedding:8b`, а не `qwen3-embedding`)
- **Неверный базовый URL** — убедитесь, что конечная точка поддерживает `/v1/embeddings`
- **Отсутствует API-ключ** — некоторые провайдеры требуют аутентификацию даже для эмбеддингов
- **Сеть** — проверьте подключение к конечной точке

### В чём разница между поиском по ключевым словам и RAG?

Поиск по ключевым словам ищет точное совпадение текста. RAG (семантический поиск) ищет по смыслу — запрос «настройка базы данных» может найти «конфигурацию Room», даже если слово «база данных» нигде не встречается. RAG требует модель эмбеддингов и закэшированные сообщения. См. [Поиск по диалогам](search.ru.md).

### Как использовать Ollama для эмбеддингов?

1. Установите Ollama на машину
2. Загрузите модель эмбеддингов: `ollama pull qwen3-embedding:8b`
3. В Agora добавьте удалённую модель эмбеддингов с пресетом **Ollama**
4. Используйте `http://<хост>:11434/v1` как базовый URL
5. Введите точное имя модели, включая тег (например, `qwen3-embedding:8b`)
6. Оставьте API-ключ пустым

## Память

### В чём разница между Активной памятью и Сохранёнными воспоминаниями?

**Активная память** — это единый постоянный контекст, включаемый в каждый API-вызов — модель всегда его видит. **Сохранённые воспоминания** — это коллекция именованных файлов, которые модель ищет и извлекает по требованию. Используйте Активную память для постоянных фактов; используйте Сохранённые воспоминания для справочных материалов. См. [Память и кэш](memory.ru.md).

### Может ли модель изменять мои воспоминания?

Да, если вы включите **Доступ к сохранённым воспоминаниям** и/или **Доступ к активной памяти** в Настройки → Память. Модель может создавать, читать, редактировать и удалять воспоминания через вызовы инструментов. Все разрешения по умолчанию отключены.

## Оболочка и Инструменты

### Как настроить удалённый доступ к оболочке?

Разверните сервер [Conch](https://github.com/newo-ether/conch) на целевой машине, затем добавьте устройство в Настройки → Оболочка, указав его URL и API-ключ. См. [Удалённая оболочка](shell.ru.md).

### Зашифровано ли соединение с оболочкой?

Да. Conch использует обмен ключами ECDH + шифрование AES-256-GCM + подпись HMAC-SHA256. Весь трафик между Agora и сервером Conch сквозным образом зашифрован.

## Данные

### Как создать резервную копию данных?

Перейдите в Настройки → Управление данными → **Экспорт данных**. Выберите категории и экспортируйте в файл `.agora`. Храните его в надёжном месте. См. [Переносимость данных](import-export.ru.md).

### Можно ли импортировать из ChatGPT или Claude?

Да. Экспортируйте ваши данные из ChatGPT или Claude (они предоставляют файлы `.zip`), затем импортируйте в Настройки → Управление данными → **Сторонние**. Поддерживаются стратегии как Слияния, так и Замены. См. [Переносимость данных](import-export.ru.md#third-party-import).

### Включаются ли API-ключи в экспорт?

Могут включаться, но это опционально. Экран экспорта позволяет переключить включение API-ключей. При включении отображается предупреждение. Ключи хранятся в открытом виде внутри файла `.agora`, поэтому включайте их только для полной миграции устройства в доверенные места назначения.

## Общее

### Где хранятся мои данные?

Всё хранится локально на вашем Android-устройстве в базе данных Room. У Agora нет серверов, нет облачной синхронизации, нет телеметрии. Сообщения отправляются напрямую с вашего устройства настроенному вами AI-провайдеру.

### Поддерживает ли Agora несколько языков?

Да. Интерфейс приложения поддерживает **English** и **中文 (китайский)**. Настройки → Язык. После переключения требуется перезапуск.

Создайте issue на [GitHub](https://github.com/newo-ether/Agora/issues). Для участия в разработке см. раздел [Contributing](https://github.com/newo-ether/Agora#contributing) в README.
