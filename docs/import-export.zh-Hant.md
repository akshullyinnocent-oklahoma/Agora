# 資料可攜性

Agora 將您的所有資料儲存在裝置上，並提供完整的匯入/匯出功能。您擁有自己的資料——可以移入、移出、備份。

## 匯出

將您的資料匯出為單一 `.agora` 檔案——一個包含 Agora 儲存的所有內容的可攜式封存檔。

### 匯出內容

您可以選擇要包含的內容：

| 類別 | 內容 |
|----------|----------|
| **對話與訊息** | 所有聊天記錄、訊息樹、分支 |
| **記憶** | 使用中的記憶與所有已儲存的記憶檔案 |
| **系統提示** | 所有自訂系統提示範本 |
| **設定** | 應用程式設定與偏好 |
| **API 金鑰** | 所有已設定的 API 金鑰 |

!!! danger "API 金鑰警告"
    API 金鑰以**明文**匯出。任何擁有 `.agora` 檔案的人都可以讀取您的金鑰。僅在信任目的地且安全處理檔案的情況下，才啟用 API 金鑰匯出。

### 如何匯出

1. 前往 **設定 → 資料控制**
2. 點選 **匯出資料**
3. 選擇要包含的類別
4. 點選 **匯出**
5. 選擇要儲存 `.agora` 檔案的位置

---

## 匯入

從先前的 `.agora` 匯出檔還原資料。

### 匯入策略

匯入時，您可以選擇 Agora 如何處理裝置上已存在的資料：

| 策略 | 行為 |
|----------|----------|
| **合併** | 新增項目，保留現有項目。如果存在相同 ID 的項目，匯入版本會覆寫它。 |
| **取代** | 清除所選類別中的所有現有資料，然後匯入。全新開始。 |
| **略過** | 僅匯入沒有衝突的項目。現有項目保持不變。 |

!!! tip
    大多數情況下使用**合併**——它可以安全地新增資料，同時保留裝置上已有的內容。

### 如何匯入

1. 前往 **設定 → 資料控制**
2. 點選 **匯入資料**
3. 選擇一個 `.agora` 檔案
4. 檢閱匯入預覽——查看檔案內容（匯出日期、版本、內容數量）
5. 選擇匯入策略
6. 點選 **匯入**

!!! danger "API 金鑰警告"
    如果匯出檔案包含 API 金鑰，Agora 會在匯入前警告您。金鑰以明文匯入。僅在信任檔案來源的情況下繼續。

---

## 第三方匯入

從其他 AI 聊天平台匯入對話。

Claude 和 ChatGPT 都會將您的資料匯出為 **`.zip`** 封存檔。Agora 直接匯入該 `.zip` 檔案——無需先解壓縮，且 Agora **不**接受獨立的 `.json` 檔案。

### 從 Claude 匯入

**1. 從 Claude 匯出。** 前往 [Claude](https://claude.ai/) → **設定 → 資料控制 → 匯出資料**。Claude 通常會很快準備好封存檔——一般在 **一小時內**——並透過電子郵件傳送下載連結給您。

!!! warning "請及時下載"
    Claude 的下載連結**很快就會過期**。收到電子郵件後請立即下載 `.zip` 檔案——如果等待太久，連結會失效，您將需要重新申請匯出。

**2. 匯入到 Agora。**

1. 前往 **設定 → 資料控制 → 第三方 → 從 Claude 匯入**
2. 選擇已匯出的 `.zip` 檔案
3. 檢閱預覽——查看對話數量與訊息數量
4. 選擇**合併**或**取代**策略
5. 點選 **匯入**

!!! note
    Agora 直接從 Claude 的 `.zip` 匯出檔讀取對話資料。附件會被偵測並顯示在預覽中，但只會匯入訊息文字——附件檔案本身不會被匯入。

### 從 ChatGPT 匯入

**1. 從 ChatGPT 匯出。** 前往 [ChatGPT](https://chatgpt.com/) → **設定 → 資料控制 → 匯出資料**。ChatGPT 會處理請求，並在準備好後透過電子郵件傳送下載連結給您。

!!! info "請耐心等待"
    ChatGPT 的匯出通常需要 **1 到 2 天**才能送達。這是正常的——請等待電子郵件，而不是重新申請。

**2. 匯入到 Agora。**

1. 前往 **設定 → 資料控制 → 第三方 → 從 ChatGPT 匯入**
2. 選擇已下載的 `.zip` 檔案
3. 檢閱預覽
4. 選擇**合併**或**取代**策略
5. 點選 **匯入**

!!! note
    使用者與助理訊息都會被匯入。訊息角色會保留下來。

---

## 檔案格式

`.agora` 檔案是以 JSON 為基礎的封存檔。如果您有技術背景，可以使用標準工具檢查或處理它。此格式設計為向前與向後相容。

## 自動備份

Agora 可以按排程自動備份您的資料。您不需要記得匯出——Agora 會為您處理。

### 運作方式

- 自動備份使用 Android WorkManager 在背景定期執行
- 當備份時間到時，Agora 會將您選擇的類別匯出到已設定的目錄
- 僅在備份失敗時顯示通知——成功的備份是無聲的
- 舊備份會根據您的保留設定自動刪除

### 設定

1. 前往 **設定 → 資料控制 → 自動備份**
2. 開啟/關閉 **自動備份**
3. 設定 **備份頻率**——選擇 1 天、3 天、5 天、1 週或 1 個月
4. 選擇 **匯出內容**——選取要包含的類別。API 金鑰**可以**包含在內（勾選該方塊時會顯示警告）——僅在備份位置為私密且安全的情況下才啟用。API 金鑰預設**不**包含在內。
5. 設定 **備份位置**——點選以選擇資料夾（預設為 `Download/Agora/Backup`）
6. 開啟/關閉 **自動刪除舊備份**，並設定 **刪除早於** 的期間

!!! info "自動刪除限制"
    刪除週期必須長於備份週期。例如，如果您每週備份一次，則備份可以在 1 個月或 1 年後自動刪除——不能更短。這樣可以防止在建立新備份之前刪除您唯一的備份。

!!! note
    自動備份使用 Android 的 WorkManager 來確保可靠性，即使應用程式關閉或裝置重新啟動也不受影響。在 Doze 模式下，備份可能會稍微延遲以節省電池電量。

---

## 最佳實踐

- **定期匯出**作為備份——將檔案保存在安全的地方
- **啟用自動備份**以獲得免手動的排程保護
- **不要在日常匯出中包含 API 金鑰**——僅在完整裝置遷移時才啟用金鑰匯出
- **對增量匯入使用合併**——取代是破壞性的
- **匯入前先預覽**——檢查匯出日期與內容數量，以確認這是正確的檔案
