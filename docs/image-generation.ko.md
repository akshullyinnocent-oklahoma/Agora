# 이미지 생성

텍스트 프롬프트에서 텍스트-이미지 모델을 사용하여 대화 내에서 바로 이미지를 생성합니다.

## 기능

이미지 생성이 활성화되면, Agora는 전용 텍스트-이미지 모델(DALL·E, GPT-Image, Imagen, FLUX, Stable Diffusion, Seedream, Qwen-Image 등)을 사용하여 프롬프트를 이미지로 변환할 수 있습니다. 생성된 이미지는 대화에 반환되므로 다른 답장과 마찬가지로 반복 작업이 가능합니다.

이미지 생성은 채팅하는 모델과 **독립적인 자체 모델 선택**을 사용합니다 — 하나의 모델로 채팅하면서 다른 모델로 이미지를 생성할 수 있습니다.

## 설정

1. **설정 → 이미지 생성**으로 이동
2. **이미지 생성 활성화** 토글 켜기
3. **모델**을 탭하고 텍스트-이미지 모델 선택
4. 선택적으로 **기본 크기** 설정 (너비 × 높이)

!!! note "프로바이더 자격 증명 재사용"
    이미지 모델은 해당 프로바이더의 API 키와 기본 URL을 재사용합니다 — **설정 → API 프로바이더**에서 이미 구성한 것과 동일한 자격 증명입니다. 별도로 입력할 키가 없습니다. 로컬 모델(적절한 가중치 포함)을 사용하여 생성을 기기 내에서 유지할 수도 있습니다.

## 모델 선택

**모델**을 탭하여 생성에 사용할 모델을 선택합니다.

- 선택기는 동기화된 모든 모델에서 텍스트-이미지 모델로 보이는 것들을 필터링하여 표시하므로 목록이 짧게 유지됩니다.
- 원하는 모델이 목록에 없으면(특이한 이름), **모든 모델 표시**를 활성화하여 전체 목록에서 선택합니다.
- 올바르게 동기화된 `Provider:model` 항목만 유효한 선택으로 간주됩니다. 목록이 비어 있으면 **설정 → API 프로바이더** / **모델 관리**에서 먼저 모델을 동기화하세요.

## 기본 크기

기본 출력 크기를 **너비 × 높이** (픽셀)로 설정합니다 (예: `1024` × `1024`).

- 기본값은 `1024 × 1024`입니다.
- 지원되는 크기는 모델과 프로바이더에 따라 다릅니다 — 모델이 크기를 거부하면 문서화된 값을 시도하세요 (일반적인 옵션은 `1024×1024`, `1024×1792`, `1792×1024`).

## 작동 방식

1. 이미지 생성 활성화 및 이미지 모델 선택
2. 대화에서 어시스턴트에게 이미지 생성 요청
3. Agora가 해당 프로바이더의 자격 증명을 사용하여 구성된 이미지 모델로 요청 전달
4. 생성된 이미지가 대화에 반환됨

!!! tip
    프롬프트를 구체적으로 작성하세요 — 주제, 스타일, 구도, 조명을 설명하세요. 명확한 프롬프트가 모호한 프롬프트보다 훨씬 나은 결과를 만듭니다.
