# Локальные модели

Запускайте LLM прямо на вашем Android-устройстве, используя файлы моделей GGUF и llama.cpp. Без сети, без API-ключей, полностью приватно.

## Как это работает

Agora включает llama.cpp через Android NDK (CMake). Когда вы импортируете файл GGUF, модель работает полностью на процессоре вашего устройства — никакие данные не покидают устройство.

## Требования

- Модели только в **формате GGUF** (стандарт для llama.cpp)
- **Память устройства**: Модель должна помещаться в доступную ОЗУ. Как правило:
    - Модели с 1–3B параметрами: 4–6 ГБ ОЗУ
    - Модели с 7–8B параметрами: 6–8 ГБ ОЗУ
- **Хранилище**: Файлы GGUF варьируются от ~500 МБ (квантизированные малые модели) до 5+ ГБ

!!! warning
    Локальный инференс интенсивно нагружает процессор и значительно медленнее облачных API. Он лучше всего подходит для автономного использования, приватного контента или экспериментов — не для быстрого, высокообъёмного чата.

## Импорт чат-модели

1. Скачайте файл модели GGUF на ваше устройство (см. рекомендуемые источники ниже)
2. Перейдите в **Настройки → Провайдер**
3. Выберите **Локальный** в качестве провайдера
4. Нажмите **Импортировать GGUF модель**
5. Выберите файл `.gguf` на вашем устройстве
6. Настройте модель:

| Параметр | Описание | Пример |
|-----------|-------------|---------|
| **ID модели** | Идентификатор в нижнем регистре, без пробелов | `qwen3-8b` |
| **Псевдоним** | Отображаемое имя | `Qwen 3 8B` |
| **Размер контекста** | Максимальное контекстное окно в токенах | `4096` |
| **Temperature** | Случайность (0.0–2.0) | `0.7` |
| **Top P** | Порог nucleus sampling (0.0–1.0) | `0.9` |
| **Макс. токенов** | Максимальная длина генерации | `2048` |

7. Нажмите **Добавить**

Модель импортирована и готова к использованию немедленно.

## Импорт модели эмбеддингов

Модели эмбеддингов меньше и используются для семантического поиска:

1. Перейдите в **Настройки → Поиск по диалогам**
2. Нажмите **Добавить локальную модель**
3. Выберите файл `.gguf` модели эмбеддингов
4. Дайте ей имя
5. Нажмите **Добавить**

См. [Эмбеддинги / RAG](embedding.ru.md) для настройки поиска.

---

## Выбор активной модели

После импорта одной или нескольких моделей:

1. Перейдите в **Настройки → Провайдер → Локальный**
2. Вы увидите список всех импортированных моделей
3. Нажмите **радиокнопку** рядом с моделью, которую хотите использовать
4. Выбранная модель становится активной, когда выбран провайдер **Локальный** в качестве чат-провайдера

---

## Управление локальными моделями

### Переименование

Нажмите на модель, чтобы изменить её псевдоним или настроить параметры (temperature, размер контекста и т.д.).

### Удаление

Долгое нажатие на модель и нажмите **Удалить**. Это удаляет модель из Agora и удаляет файл GGUF из хранилища.

## Рекомендуемые модели

### Чат-модели

| Модель | Размер | Требуется ОЗУ | Примечания |
|-------|------|------------|-------|
| Qwen 3 1.7B | ~1 ГБ | 3–4 ГБ | Хорошее качество для своего размера |
| Llama 3.2 3B | ~2 ГБ | 4–5 ГБ | Надёжная универсальная |
| Qwen 3 8B | ~5 ГБ | 7–8 ГБ | Лучшее качество, высокие требования к ОЗУ |

### Модели эмбеддингов

| Модель | Размер | Примечания |
|-------|------|-------|
| BGE Small EN v1.5 | ~130 МБ | Хорошие английские эмбеддинги |
| BGE Small ZH v1.5 | ~130 МБ | Оптимизирован для китайского |
| Nomic Embed Text v1.5 | ~270 МБ | Хороший мультиязычный |

### Где взять файлы GGUF

- [Hugging Face](https://huggingface.co/models?library=gguf) — поиск «GGUF»
- [Квантизированные модели bartowski](https://huggingface.co/bartowski) — широкий выбор, хорошо организовано

!!! tip
    Ищите квантизацию `Q4_K_M` — она предлагает наилучший компромисс между качеством и размером для чат-моделей.

## Советы по производительности

- **Меньше контекст = быстрее**: Начните с 2048 и увеличивайте только при необходимости
- **Ниже квантизация = быстрее**: Q4_K_M быстрее, чем Q6_K или Q8
- **Закройте другие приложения**: Локальному инференсу нужно как можно больше ОЗУ
- **Подключите к питанию**: Инференс интенсивно нагружает процессор, и длительное использование разряжает батарею
